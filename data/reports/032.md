# 开源Agent项目学习 #032

## 项目名称
OpenAI Codex CLI

## GitHub链接
https://github.com/openai/codex

## 一句话简介
OpenAI 官方开源（Apache 2.0）的终端编程 Agent，用 Rust 构建，支持 OS 级沙箱隔离、MCP 协议、Skills 系统，可在 CLI / Desktop / IDE / Cloud 四个表面运行。

## 项目架构

Codex 采用四层架构：

```
┌─────────────────────────────────────────────────────────┐
│ SURFACE 层：CLI / Desktop App / IDE 扩展 / Cloud Tasks │
├─────────────────────────────────────────────────────────┤
│ EXTENSION 层：MCP / Skills / Apps / Web Search         │
├─────────────────────────────────────────────────────────┤
│ SECURITY 层：OS 级沙箱 + 审批策略                        │
│ macOS: Seatbelt | Linux: Landlock + seccomp             │
├─────────────────────────────────────────────────────────┤
│ CORE 层：GPT-5.x-Codex 模型 + Shell/Patch/Read/Search  │
└─────────────────────────────────────────────────────────┘
```

- **Core 层**：GPT-5.x-Codex 模型家族驱动，工具包括 Shell 执行、文件 Patch、文件读取、Web 搜索。上下文用满时自动 compact。
- **Security 层**：OS 内核级沙箱（非容器），macOS 用 Apple Seatbelt，Linux 用 Landlock + seccomp。配合四级审批策略（auto-approve → suggest → ask → never）。
- **Extension 层**：MCP 连接外部服务（GitHub、Figma、Sentry 等）；Skills 打包可复用工作流；Web Search 提供实时互联网上下文。
- **Surface 层**：CLI（终端自动化）、Desktop（多工作树并行）、IDE 扩展（编辑-编译-测试循环）、Cloud（异步长任务）。

关键配置文件：
- `config.toml`：全局/项目级配置（模型、MCP server、审批策略）
- `AGENTS.md`：跨工具项目指令（兼容 Codex / Cursor / Copilot / Amp 等 60,000+ 开源项目）
- `codex.json`：项目级 Codex 配置

## 技术亮点

- **Rust 构建的高性能 CLI**：原生二进制，跨平台（macOS/Linux/Windows），npm/brew/winget 多种安装方式
- **OS 级沙箱隔离**：不依赖容器，直接在内核层面限制文件系统和网络访问，安全性远超 Docker 方案
- **AGENTS.md 跨工具标准**：一份项目指令文件被 60,000+ 开源项目采用，已成为事实上的 Agent 配置标准（类似 `.editorconfig`）
- **Skills 系统**：打包可复用的领域专业知识，按需加载，类似 OpenClaw 的 Skills 但面向编码场景
- **多 Surface 统一智能**：CLI、Desktop、IDE、Cloud 四个入口共享同一个模型能力，用户按场景切换
- **1M 上下文窗口**：GPT-5.4 支持 100 万 token 上下文，配合 `/compact` 命令和 `@file` 引用管理 token 预算
- **灵活的审批策略**：四级从 auto-approve 到 never，按工具类型（shell、file write、mcp）分别配置
- **Hooks 引擎**：在 Agent 操作前后注入自定义逻辑（如 lint、格式化、通知）

## 适用场景

- **日常编码**：在终端中直接与代码库交互，执行重构、添加功能、修复 bug
- **CI/CD 集成**：GitHub Action 自动化代码审查和修复
- **团队协作**：AGENTS.md 统一团队 AI 编码规范，跨工具兼容
- **大型项目重构**：1M 上下文 + 自动 compact 支持超大型代码库
- **多任务并行**：Desktop App 支持多个工作树同时进行不同任务
- **企业部署**：支持自定义审批策略、私有 MCP server、合规要求

## 对我们的启发

1. **OS 级沙箱 > 容器沙箱**：OpenClaw 目前用 exec 工具执行命令，安全依赖 elevated/approval 机制。Codex 的 Landlock/seccomp 方案更底层更安全，值得参考。对于楠楠这种有完整文件系统访问权的 Agent，内核级沙箱是终极安全方案。

2. **AGENTS.md 标准化**：这个文件格式已经被行业广泛采用。OpenClaw 的 AGENTS.md 本身也遵循类似理念，说明这条路是对的。可以进一步对齐 AGENTS.md 的格式规范，让 OpenClaw 项目更易被其他工具识别。

3. **Skills 系统对标**：Codex 的 Skills 和 OpenClaw 的 Skills 概念几乎一样——都是按需加载的领域知识包。Codex 面向编码，OpenClaw 面向通用 Agent，但核心设计理念一致。Codex 的 Skills 打包方式（可分享、可版本化）值得学习。

4. **多 Surface 统一**：Codex 的 CLI/IDE/Cloud/桌面 四个表面共享一个核心的做法很好。OpenClaw 已经有 Web、Discord、飞书等多通道，核心 Agent 智能是统一的，这个架构方向正确。

5. **Context 管理的实战经验**：Codex 的 `/compact`、`@file` 引用、token 预算管理等经验，对 OpenClaw 优化长对话记忆管理有直接参考价值。
