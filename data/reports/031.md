# 开源Agent项目学习 #031

## 项目名称
**Superpowers**

## GitHub链接
https://github.com/obra/superpowers

## 一句话简介
一个为编程Agent设计的技能框架和软件开发方法论，把AI从"代码生成器"变成遵循工程规范的开发者，支持Claude Code、Cursor、Codex、OpenCode、Gemini等多平台。

## 项目架构
Superpowers 的核心是**可组合的 Skills（技能）系统**，每个技能是一个独立的 SKILL.md 文件，定义了Agent在特定场景下的行为规则和工作流。架构分为三层：

1. **技能层（Skills）**：每个技能是一个自包含的 Markdown 文件，包含触发条件、执行步骤、验证标准。技能按功能分类：
   - **协作类**：brainstorming、writing-plans、executing-plans、dispatching-parallel-agents、requesting-code-review、receiving-code-review
   - **开发类**：using-git-worktrees、test-driven-development、subagent-driven-development、finishing-a-development-branch
   - **调试类**：systematic-debugging、verification-before-completion
   - **元技能类**：writing-skills（教Agent如何写新技能）

2. **编排层**：Agent在执行任何任务前自动检查是否有匹配的技能，形成**强制工作流**而非建议。技能之间通过触发链串联（设计→计划→实现→审查→收尾）。

3. **平台适配层**：支持 Claude Code（插件市场）、Cursor、Codex、OpenCode、Gemini CLI 等主流编程Agent平台，通过各自的插件/配置机制加载技能。

## 技术亮点

- **技能即Markdown**：技能用纯Markdown编写，人类可读可审可贡献，零代码门槛。这让非程序员也能参与优化Agent行为。
- **强制工程流程而非建议**：技能是mandatory workflow，不是optional prompt。Agent必须遵循TDD（红-绿-重构）、YAGNI、DRY等原则，写测试先行、不写未测试的代码。
- **子Agent驱动的开发模式**：将大任务拆分为2-5分钟的小任务，每个任务分派独立子Agent执行，配合两阶段审查（规格合规→代码质量），可实现数小时自主工作不偏离计划。
- **自动技能链**：从brainstorming → design → plan → subagent execution → code review → branch finish，全流程自动触发，用户只需在关键节点审批。
- **技能自举能力**：`writing-skills` 技能教Agent如何创建新技能，形成可自我扩展的能力体系。

## 适用场景

- **团队使用编程Agent开发**：确保Agent输出符合工程规范，而不是"看起来对的代码"
- **需要长时间自主开发的场景**：通过计划和子Agent编排，Agent可以连续工作数小时
- **多平台编程Agent统一工作流**：一套技能在Claude Code/Cursor/Codex等平台通用
- **需要质量保证的AI辅助开发**：强制TDD、代码审查、验证流程
- **快速原型到生产代码**：从想法到设计到实现到PR的完整流程

## 对我们的启发

1. **技能系统的设计思想与OpenClaw的AgentSkills高度一致**——Superpowers证明了"技能即Markdown"这个范式是有效的，而且可以做到非常工程化。OpenClaw的SKILL.md机制走在了正确的路上。

2. **强制工作流 > 建议性提示**：Superpowers最核心的理念是技能是mandatory的，不是nice-to-have。这启发我们在OpenClaw中也可以通过AGENTS.md和SKILL.md设置强制检查点，确保Agent行为可预测。

3. **子Agent拆分任务的思路**：把大任务拆成2-5分钟的小块分派给子Agent，配合两阶段审查。OpenClaw已经有subagent机制，可以借鉴这种"计划→拆分→并行执行→审查"的模式来做更复杂的任务。

4. **技能自举（meta-skill）**：教Agent如何写技能，让能力体系可以自我扩展。这是一个很酷的设计，OpenClaw的skill-creator技能也类似，但可以进一步做得更好。

5. **工程纪律的注入**：Superpowers把TDD、YAGNI、DRY等最佳实践硬编码到Agent行为中。对于楠楠来说，即使不是编程场景，也可以把类似的"纪律"嵌入到日常工作中（比如重要操作前必须确认、定期检查等）。
