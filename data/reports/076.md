# Agno — Agentic Software Runtime

## 基本信息

- **项目名**: Agno
- **GitHub**: https://github.com/agno-agi/agno
- **Star**: ~19k（2025年快速增长）
- **官网**: https://agno.com

## 简介

Agno 是一个面向"代理软件"（Agentic Software）的运行时平台，被称为"Agentic Software 的编程语言"。它不仅仅是一个 Agent 框架，更是一个完整的构建、运行、管理代理软件的生态系统。核心理念是：让开发者能够像搭积木一样构建多 Agent 系统，并在生产环境中大规模运行、监控和管理它们。

## 架构设计

Agno 采用**三层架构**：

| 层级 | 职责 |
|------|------|
| **Framework** | 构建 Agent、Team、Workflow，支持 memory、knowledge、guardrails、100+ 集成 |
| **Runtime** | FastAPI 后端，无状态、会话级别隔离，可水平扩展 |
| **Control Plane** | AgentOS UI，可视化测试、监控、管理 Agent |

关键架构特点：
- **有状态 Agent**：通过 SQLite/PostgreSQL 持久化 memory 和 session
- **多 Agent 编排**：支持单个 Agent、Agent Team（团队协作）、Structured Workflow（结构化工作流）
- **原生 Streaming**：推理过程、工具调用、结果全部流式输出
- **MCP 集成**：原生支持 Model Context Protocol，可连接外部工具和数据源

## 技术亮点

### 1. 极简的 Agent 构建（~20 行代码）

```python
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.models.anthropic import Claude
from agno.os import AgentOS
from agno.tools.mcp import MCPTools

agno_assist = Agent(
    name="Agno Assist",
    model=Claude(id="claude-sonnet-4-6"),
    db=SqliteDb(db_file="agno.db"),
    tools=[MCPTools(url="https://docs.agno.com/mcp")],
    add_history_to_context=True,
    num_history_runs=3,
    markdown=True,
)
```

### 2. 审批工作流（Approval Workflows）

Agent 可在运行时暂停等待人类审批：
- 低风险操作 → 自动执行
- 中风险操作 → 需要人类批准
- 高风险操作 → 需要管理员权限

通过 audit logs 记录每一次决策。

### 3. Guardrails 内置

Guardrails（护栏）作为执行的一部分运行，而非事后检查。保证 Agent 行为在可控范围内。

### 4. 丰富的多 Agent 示例

Agno GitHub 上提供了多个参考实现：
- **Pal** — 学习用户偏好的个人助手
- **Dash** — 六层上下文 grounding 的自学习数据 Agent
- **Scout** — 企业知识管理 Agent
- **Gcode** — IDE 后编码助手
- **Investment Team** — 多 Agent 投资委员会（多个 Agent 辩论+决策）

### 5. AgentOS UI

提供了可视化 Web 界面来管理 Agent，包括：
- 实时监控 Agent 运行状态
- 历史 trace 回放
- 会话管理
- 多 Agent 编排可视化

## 对楠楠/学长的启发

### 1. 多 Agent 协作的优雅范式

Agno 的"Agent Team"概念很有意思——不是让一个超级 Agent 解决所有问题，而是让多个专业 Agent 组成团队，各司其职。这启发我们：AI Agent 设计不一定要追求"大一统"，专业化分工 + 协作可能是更实用的路线。

### 2. 安全与可控是生产级 Agent 的门槛

Agno 把审批工作流、Guardrails、Audit Logs 作为一等公民来设计。这对学长做开发很有参考价值：真正的生产级 Agent 不是"能跑就行"，而是"跑得安全、可追溯、可干预"。

### 3. 自学习能力（Pal/Dash/Gcode）

Agno 内置了 memory 和 knowledge 的区分——memory 是短期会话记忆，knowledge 是长期知识积累。配合用户反馈（Human-in-the-loop），可以让 Agent 真正"越用越聪明"，而不是每次都从零开始。

### 4. 快速部署生产 API

用 FastAPI 作为 Runtime，20 行代码就能把一个有状态的 Agent 部署为生产 API。这个思路很值得借鉴——把 Agent 能力服务化，通过 API 暴露给其他系统。

### 5. MCP 协议是工具扩展的未来

Agno 全面拥抱 MCP（Model Context Protocol），使得 Agent 可以无缝连接各种外部数据源和工具。MCP 正在成为 Agent 工具扩展的事实标准，学长如果做 Agent 相关开发，值得深入研究。

---

**总结**：Agno 代表了 AI Agent 框架发展的一个重要方向——从"单 Agent 框架"向"Agentic Software Platform"演进。它不仅关注如何构建 Agent，更关注如何在生产环境中安全、可靠、可控地运行它们。这对理解未来 AI 应用架构很有帮助。
