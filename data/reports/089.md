## 简介

Browser-Use 是一个开源的 Python 库，让 AI Agent 能够控制浏览器完成各种网页操作。它用自然语言指令替代了传统的 Selenium/Playwright 脚本，把"找按钮然后点击"这种脆弱的 XPath 匹配变成了 AI 理解页面后自主决策。

简单说：**你告诉 AI 要做什么，AI 自己操作浏览器完成**。

---

## 架构设计

核心是一个**感知-决策-执行循环**：

```
用户指令
  ↓
LLM 处理层（理解任务 + 当前页面状态）
  ↓
DOM 提取 + 视觉分析（双通道）
  ↓
行动规划（决定下一步做什么）
  ↓
Playwright 执行（点击、输入、导航）
  ↓
状态更新 → 反馈循环，直到任务完成
```

三层核心组件：
1. **LLM 集成层**：支持 OpenAI、Claude、Gemini、本地 Ollama（隐私/省钱）
2. **浏览器控制引擎**：底层用 Playwright，通过 WebSocket 通信
3. **视觉理解系统**：同时分析 DOM + 截图，hybrid 方式，比纯 DOM 更抗网站变化

---

## 技术亮点

### 1. DOM + Vision 双通道感知
传统自动化靠 XPath/CSS 选择器，网站改版就挂。Browser-Use 把截图也发给 LLM 视觉分析，即使按钮没有 ID，也能靠外观识别出来。

### 2. MCP（Model Context Protocol）支持
Agent Settings Tab 可以连接外部 MCP 服务，打通各种工具生态。

### 3. 多 LLM 后端兼容
不绑死某一家——OpenAI、Claude、Gemini、本地 Ollama 都能用。想省钱跑本地？一条命令切换。

### 4. Web UI + CLI 双入口
既有命令行方式编程调用，也有 Gradio 做的 Web 界面，非程序员也能用。

### 5. 行动历史记录
每一步操作都被记录，方便回溯和调试。

## 对楠楠/学长的启发

### 1. 双通道感知思路值得借鉴
楠楠做 AI 助手，经常需要"理解"用户给的内容。类比 Browser-Use 的 DOM+Vision 双通道，我们可以多模态理解——不只看文字，也看图片、上下文，交叉验证更准确。

### 2. 开放生态（支持 MCP）让项目生命力更强
Browser-Use 不自己造所有轮子，而是通过 MCP 接入外部工具。这种"被集成"的能力让它生态越来越大。学长做开发时也可以考虑让自己的工具支持标准协议，方便被其他系统接入。

### 3. 本地运行是重要差异化
Browser-Use 支持 Ollama 本地模型，不用把所有数据都发给 OpenAI。学长对数据隐私敏感的话，这种架构思路很有参考价值——**关键能力本地化，通用能力云端化**。

### 4. "渐进式自动化"理念
Browser-Use 的反馈循环是逐步探索式的，每一步都看结果再决定下一步。这比一次性规划更robust。学长的 AI 辅助开发也可以借鉴——不要一次性生成整个项目，而是逐步、反馈式地推进。

---

## 总结

Browser-Use 是 2024-2025 年最火的 AI Agent 开源项目之一，它成功地把 LLM 的理解能力与浏览器的执行能力结合在一起。架构清晰（感知-决策-执行循环）、多模态融合（DOM+视觉）、开放生态（MCP）三个特点让它从众多浏览器自动化工具中脱颖而出。对于学习 AI Agent 架构设计，特别是"如何让 AI 控制真实世界界面"，Browser-Use 是一个非常值得研究的案例。
