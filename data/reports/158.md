# AI Agent 学习报告 #158 — Mastra

**GitHub:** https://github.com/mastra-ai/mastra

## 一句话简介
Mastra 是一个 TypeScript 原生的全栈 AI Agent 框架，专注模型编排、检索工作流和可观测性，让 TypeScript 开发者能快速构建生产级 AI 应用。

## 核心架构

Mastra 采用**模块化分层设计**：

```
┌─────────────────────────────────────┐
│         Agent Layer                 │  ← 核心Agent定义
├─────────────────────────────────────┤
│     Tools / RAG / Workflows         │  ← 工具和编排
├─────────────────────────────────────┤
│      Observability Layer             │  ← 可观测性（追踪、日志）
├─────────────────────────────────────┤
│    Model Provider Abstraction       │  ← 40+模型统一接入
└─────────────────────────────────────┘
```

**关键组件：**
- **Mastra Agents** — 定义 Agent 行为，支持工具绑定、记忆和系统提示词
- **Workflows** — 声明式工作流编排，支持条件分支和循环
- **RAG Pipeline** — 内置检索增强生成管道，支持多向量库
- **Observability** — 内置分布式追踪（OpenTelemetry），生产环境可观测
- **Mastra SDK** — 统一的 TypeScript API，类型安全

## 技术亮点

- **TypeScript 原生**：类型安全的 Agent 定义，IDE 自动补全，开发体验好
- **40+ 模型支持**：OpenAI、Anthropic、Google、Cohere、Mistral 等统一抽象
- **内置 RAG**：开箱即用的检索管道，支持 PDF、Markdown、URL 等多格式
- **生产级可观测性**：OpenTelemetry 集成，追踪每一步工具调用和模型响应
- **Tools as Functions**：将工具定义为带类型的 TypeScript 函数，简洁直观
- **Sync/Async 双模式**：支持流式输出和标准响应两种交互模式

## 对楠楠的启发

1. **TypeScript-first 的工程价值**：Mastra 证明了 TS 类型系统对 Agent 开发的重要性。楠楠在写 OpenClaw 脚本时，多用类型标注能让错误提前暴露，工具调用更可靠。

2. **内置 RAG 管道的必要性**：楠楠目前的知识管理靠文件 + 上下文注入，没有专门的检索层。如果要构建更系统的知识库，Mastra 的 RAG Pipeline 模式值得借鉴。

3. **可观测性内嵌设计**：OpenClaw 目前缺乏内置追踪，Mastra 把 OpenTelemetry 作为核心组件而非第三方集成。这个思路对调试复杂 Agent 行为至关重要。

4. **工作流即代码**：Mastra 的 Workflows 用声明式 TypeScript 代码而非 JSON/YAML 配置，更容易版本控制和调试。楠楠的 AGENTS.md 任务流可以朝这个方向演进。
