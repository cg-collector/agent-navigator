# 开源Agent项目学习 #050

## 项目名称
ReMe (Remember Me, Refine Me)

## GitHub链接
https://github.com/agentscope-ai/ReMe

## 一句话简介
专为 AI Agent 设计的记忆管理工具包，用 Markdown 文件实现可读、可编辑、可迁移的持久化记忆系统。

## 项目架构

ReMe 来自 AgentScope 团队（阿里达摩院），提供两套记忆系统：

**1. ReMeLight — 文件型记忆系统（核心）**
- `working_dir/MEMORY.md` — 长期记忆，持久化用户偏好和重要信息
- `working_dir/memory/YYYY-MM-DD.md` — 每日日志，每次对话后自动写入
- `working_dir/dialog/YYYY-MM-DD.jsonl` — 原始对话记录（压缩前）
- `working_dir/tool_result/<uuid>.txt` — 长工具输出缓存（自动过期清理）

核心组件：
- **ContextChecker** — 检查上下文是否超限，按阈值切分消息
- **Compactor** — ReActAgent，将对话历史压缩为结构化摘要
- **ToolResultCompactor** — 截断长工具输出，存文件并在消息中保留引用
- **Summarizer** — ReActAgent + 文件读写工具，自动提取重要记忆写入文件
- **MemorySearch** — 向量 + BM25 混合检索，语义搜索记忆文件

**2. 向量型记忆系统**
- 传统的向量数据库方案，支持 Qdrant/Milvus/Chroma 等后端
- 配合文件型系统使用，提供更精确的语义检索

**关键流程 — pre_reasoning_hook（推理前钩子）：**
自动串联：压缩工具输出 → 检查上下文 → 压缩历史 → 异步生成摘要，一步到位。

## 技术亮点

- **Memory as Files**：把记忆存为 Markdown 文件而非黑盒数据库，人类可直接阅读、编辑、用 Git 版本管理。这与 OpenClaw 的 `MEMORY.md` + `memory/YYYY-MM-DD.md` 设计理念完全一致！
- **用 Agent 管理记忆**：Compactor 和 Summarizer 本身就是 ReActAgent，用 LLM 来决定什么该记、什么该忘、怎么总结，而不是硬编码规则
- **混合检索**：向量语义搜索 + BM25 精确匹配，兼顾语义相似和关键词命中
- **工具输出自动压缩**：Agent 调用工具产生长输出时，自动截断并存为文件引用，防止上下文爆炸
- **SOTA 基准表现**：在 LoCoMo 和 HaluMem 记忆基准测试上达到最佳效果
- **异步摘要**：summary_memory 在后台异步执行，不阻塞主对话流程

## 适用场景

- 个人 AI 助手（长期记住用户偏好和聊天历史）
- 编程助手（跨会话保持代码风格和项目上下文）
- 客服机器人（追踪用户问题历史，提供个性化服务）
- 任务自动化（从历史任务中学习成功/失败模式）
- 多轮对话系统（自动压缩长对话，在有限上下文窗口中保留关键信息）

## 对我们的启发

1. **验证了我们的设计方向**：ReMe 的文件型记忆结构与楠楠现有的 `MEMORY.md` + `memory/YYYY-MM-DD.md` 几乎一模一样！这说明我们的方案不是拍脑袋想出来的，而是业界正在趋同的最佳实践。

2. **我们可以借鉴的点**：
   - **自动摘要写入**：ReMe 的 Summarizer 用 Agent 自动提取对话要点写入记忆文件，比我们现在靠心跳手动整理 MEMORY.md 更自动化
   - **工具输出压缩**：长工具结果自动截断存文件+留引用，OpenClaw 在处理长网页/搜索结果时可以用类似机制
   - **pre_reasoning_hook 模式**：在每次推理前自动执行记忆管理流程，确保上下文窗口的高效利用

3. **我们的优势**：ReMe 是纯 Python 库，需要开发者手动集成。而 OpenClaw 已经内置了类似的文件记忆系统 + 心跳维护机制 + 子 Agent 架构，开箱即用。我们还可以进一步强化自动摘要功能。
