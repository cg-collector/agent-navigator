# 📚 AI Agent 项目研究报告 #261 — OS-Copilot：面向通用计算机智能体的自我进化框架

## 一、项目背景与问题定义

**论文信息：** OS-Copilot: Towards Generalist Computer Agents with Self-Improvement (arXiv:2402.07456, 2024年2月)

**作者团队：** 上海AI Lab、华东师范大学、普林斯顿大学、香港大学

**核心问题：** 当前绝大多数AI Agent被设计为与**单一领域**交互——要么只能操作特定软件，要么只能在某个网站内完成任务。这种"窄域"设计严重限制了Agent在通用计算机场景中的实用性。人类使用电脑时需要同时操作浏览器、终端、文件系统、Office套件和各种第三方应用，而现有Agent无法跨越这些边界。

**OS-Copilot要回答的根本问题是：** 如何构建一个能与操作系统全面交互的通用型计算机智能体？更进一步，这个智能体能否像人一样，通过自我学习和经验积累，不断掌握新应用的操作能力？

## 二、核心架构深度剖析

OS-Copilot是一个**模块化框架**，用于构建操作系统级别的语言智能体。其架构分为三个核心组件：

### 2.1 Planner（规划器）

规划器接收用户请求，将复杂任务分解为更简单的子任务。关键在于它必须理解Agent自身的能力边界，以生成**粒度恰当**的子任务规划。为此，规划器会检索Agent的能力信息（内置工具、操作系统环境等）来辅助决策。

### 2.2 Configurator（配置器）— 三种记忆的灵感

这是OS-Copilot最具创新性的组件，其设计灵感来自人脑的三种记忆类型：

- **工作记忆（Working Memory）：** 存储当前子任务的上下文和中间状态
- **陈述性记忆（Declarative Memory）：** 存储关于用户偏好、OS版本、系统信息的语义知识
- **程序性记忆（Procedural Memory）：** 存储**可复用的工具代码库**（如Python脚本、AppleScript），这是Agent"技能"的物理载体

Configurator通过**Configuration Tracker**（基于密集检索从长期记忆中召回相关信息）来组装完整的执行提示词。当现有工具无法满足需求时，会激活**Tool Generator**自动生成针对目标应用的新工具。

### 2.3 Actor（执行器）— 双阶段执行

Actor采用两阶段设计：

1. **可执行动作落地（Action Grounding）：** Executor根据配置提示生成可执行动作（如bash命令、API调用），并在真实OS环境中执行
2. **自批评与修正（Self-Criticism）：** Critic模块评估执行结果，判断子任务是否成功完成：
   - 成功 → 对生成的工具打分（0-10分），高分工具（>8分）存入程序性记忆
   - 失败 → Refiner收集反馈，对动作/工具/子任务进行自我修正，最多迭代3次

### 2.4 FRIDAY：基于OS-Copilot构建的自我进化Agent

FRIDAY是OS-Copilot框架的具体实现实例，其核心设计原则是**最大化通用性**——通过自我精炼（Self-Refinement）和自主学习（Self-Directed Learning）来持续扩展能力边界。

完整运行流程示例（以"切换系统到深色模式"为例）：
```
用户请求 → Planner分解子任务 → Configurator检索记忆(无匹配工具) 
→ Tool Generator生成AppleScript工具 → Executor执行 
→ Critic评估结果 → 成功则工具入库(评分>8)/失败则Refiner修正重试
```

## 三、技术亮点

### 亮点1：受神经科学启发的三记忆架构

**原理说明：** OS-Copilot将认知科学中经典的记忆分类理论直接映射到Agent工程实践中：
- 陈述性记忆 = "知道什么"（用户画像、系统信息）
- 程序性记忆 = "知道怎么做"（可执行的代码工具）
- 工作记忆 = "正在做什么"（当前任务上下文）

这种设计的精妙之处在于**程序性记忆的自动积累机制**：每次成功执行后，Critic用LLM给工具打分，高质量工具自动沉淀为长期技能。这意味着FRIDAY用得越多，能力越强——形成了一个正反馈循环。这与人类通过练习形成肌肉记忆的过程高度同构。

### 亮点2：基于LLM的工具自动生成与筛选机制

**原理说明：** 当Configuration Tracker在程序性记忆中找不到匹配工具时，Tool Generator会被激活，利用LLM**即时生成**针对目标应用的新工具代码。例如面对"切换深色模式"的需求，它会直接生成一段调用macOS AppleScript的Python类。

关键创新点在于**质量门控**：不是所有生成的工具都会被保留。Critic用LLM作为"裁判"，从正确性、通用性、复用潜力三个维度给工具打0-10分，只有8分以上的工具才能进入程序性记忆库。这有效防止了错误或低质量工具污染知识库，保证了系统的可靠性不会随时间退化。

### 亮点3：自主课程学习（Self-Directed Curriculum Learning）

**原理说明：** 这是FRIDAY最惊艳的能力。给定一个学习目标（如"掌握Excel操作"），FRIDAY能自主设计一套**由易到难的任务课程表**，然后通过试错方式逐个攻克：

1. FRIDAY自主提出一系列与目标相关的连续任务（从简单格式化到复杂数据透视表）
2. 按难度递增顺序执行每个任务
3. 执行过程中积累新工具和语义知识
4. 无需人工标注数据或演示，完全自主探索

实验证明，经过自主学习后，FRIDAY在SheetCopilot-20电子表格基准上的Pass@1率显著提升，且能成功完成复杂的PowerPoint幻灯片制作任务（包括字体控制、图片定位等精细操作）。这证明了**纯自主探索足以让Agent掌握全新应用**。

## 四、基准测试表现

### GAIA通用AI助手基准测试

GAIA包含466个极具挑战性的问答任务，需要Agent具备计算、网页浏览、多模态处理、文件操作等综合能力。

| 方法 | GAIA得分 |
|------|---------|
| GPT-4 (纯推理) | 24.8% |
| ReAct + GPT-4 | 28.0% |
| DEPS + GPT-4 | 35.6% |
| **FRIDAY (OS-Copilot)** | **47.9%** |
| 人类表现 | ~62% |

**FRIDAY相比之前最佳方法提升35%（35.6%→47.9%），展现了强大的跨应用泛化能力。**

### SheetCopilot-20 电子表格专项测试

| Agent | Pass@1 |
|-------|--------|
| 基线（无自主学习） | 较低 |
| FRIDAY（经自主学习后） | 显著提升 |

定性分析显示：未经过自主学习的FRIDAY无法有效控制PPT中的字体、字号和图片位置；经过自主学习后，它逐步掌握了文本框配置、颜色调整、行距修改、图片大小定位等精细化操作能力。

## 五、与同类项目横向对比

| 维度 | OS-Copilot/FRIDAY | Computer Use (Anthropic) | OpenHands | AutoGPT |
|------|-------------------|------------------------|-----------|---------|
| **交互范围** | 全OS（Web+终端+文件+GUI应用） | 主要屏幕截图+点击 | 终端+浏览器为主 | API调用为主 |
| **学习能力** | ✅ 自主课程学习+工具积累 | ❌ 无显式学习机制 | ❌ 无 | ❌ 有限 |
| **记忆系统** | 三种记忆（工作/陈述/程序性） | 简单上下文窗口 | 文件系统持久化 | 向量记忆 |
| **GUI交互能力** | ✅ 通过脚本操控原生应用 | ✅ 像素级点击模拟 | ⚠️ 有限 | ❌ 不支持 |
| **工具生成** | ✅ LLM按需生成+自动入库 | 预定义工具集 | 预定义工具 | 预定义工具 |
| **开源情况** | ✅ 开源 | ❌ 闭源API | ✅ 开源 | ✅ 开源 |
| **核心优势** | 自我进化、跨应用泛化 | 易用性强、生态好 | 软件工程能力强 | 社区活跃 |

**核心差异化：** OS-Copilot的独特之处不在于单次任务执行能力，而在于其**可持续进化的能力体系**——工具自动生成→质量筛选→记忆沉淀→跨任务复用的完整闭环。大多数竞品是"静态能力"的Agent，而FRIDAY是"动态成长"的Agent。

## 六、对实际工作的启发

### 1. Agent应该有"技能库"而非只有"提示词"

OS-Copilot的程序性记忆设计启示我们：一个真正有用的Agent不应该每次都靠Prompt重新推导解决方案，而应该维护一个**经过验证的高质量工具/代码库**。在实际工作中，可以考虑为Agent建立类似的"技能沉淀"机制——每次成功完成任务后，提取可复用的模式存入知识库。

### 2. 自主学习比预训练更重要

FRIDAY证明了**无需大量预训练数据**，Agent可以通过自主探索掌握新应用。这对实际工作的启示是：与其花大量时间做few-shot示例工程，不如设计一套让Agent自主试错学习的流程，让它在真实环境中积累经验。

### 3. 记忆分层设计值得借鉴

工作记忆/陈述性记忆/程序性记忆的三分法具有很好的工程指导意义。当前很多Agent系统把所有状态都塞进上下文窗口，既浪费token又容易溢出。参考OS-Copilot的设计，可以将：临时上下文放工作记忆、用户偏好放陈述性记忆、验证过的代码模板放程序性记忆。

### 4. 质量门控是防止能力退化的关键

工具自动生成很酷，但如果没有严格的质量筛选，Agent的知识库很快会被低质量内容污染。OS-Copilot的"8分门槛+Critic评审"机制是一个简单但有效的工程实践，值得在任何具备自我修改能力的Agent系统中采纳。
