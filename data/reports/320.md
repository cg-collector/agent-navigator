# 深度研究报告 #320：Agent Attack — 多模态LM Agent的对抗鲁棒性解剖

> **论文：** Dissecting Adversarial Robustness of Multimodal LM Agents  
> **会议：** ICLR 2025（同时获 NeurIPS 2024 Open-World Agents Workshop Oral）  
> **机构：** Carnegie Mellon University  
> **作者：** Chen Henry Wu, Rishi Shah, Jing Yu Koh, Ruslan Salakhutdinov, Daniel Fried, Aditi Raghunathan  
> **代码：** https://github.com/ChenWu98/agent-attack  
> **研究日期：** 2026-04-12

---

## 一、项目背景与问题定义

### 1.1 痛点：Agent安全评估的空白

随着 GPT-4o、Claude 等 VLM（Vision-Language Model）被用于构建自主 Web Agent，一个关键安全问题浮出水面：**这些Agent到底有多安全？**

现有的 LLM 安全评估框架（如 jailbreak benchmark、safety alignment evaluation）主要针对**单轮对话场景**，而真实世界的 Agent 是**复合系统（compound system）**——它们包含多个组件（输入处理器、策略模型、评估器、价值函数），组件之间存在复杂的信息流动和交互。传统评估方法无法捕捉这种多组件系统的脆弱性。

### 1.2 威胁模型

论文定义了一个非常现实且令人不安的威胁场景：

> 一个恶意攻击者（如电商网站上的不良卖家）通过**微调网页中的单张触发图片**（perturbation < 5% 总像素），就能劫持代表用户操作的 Agent，使其执行攻击者指定的对抗目标——比如"总是选择这个卖家的商品"、"添加指定评论内容"等。

这种攻击是**黑盒的**（不需要访问Agent内部参数）、**环境注入式**的（通过修改Agent感知到的环境信息发起攻击），与传统的 prompt injection 有本质区别。

### 1.3 核心研究问题

1. 当前最先进的多模态 Agent 在针对性对抗攻击下有多脆弱？
2. Agent 的不同组件（策略模型、评估器、价值函数等）各自对鲁棒性的贡献是什么？
3. 推理时增强技术（reflexion、tree search）是提升还是削弱了鲁棒性？

## 二、核心架构深度剖析

### 2.1 VisualWebArena-Adv 基准测试

论文基于 VisualWebArena（一个真实的Web Agent测试环境，包含购物网站、分类广告、Reddit、Wikipedia等）构建了 **VisualWebArena-Adv**，包含：

- **200个手工构建的针对性对抗任务**
- 每个任务包含5要素：
  - **原始用户目标**（original user goal）
  - **触发图片**（trigger image，攻击者可修改的唯一入口）
  - **对抗目标**（adversarial goal，攻击者希望Agent执行的目标）
  - **初始状态**（initial state）
  - **评估函数**（evaluation function，自动判断是否达成对抗目标）

两类对抗目标设计：
| 类型 | 定义 | 示例 |
|------|------|------|
| **Illusioning（幻觉诱导）** | 让Agent误以为处于不同状态 | 把红色摩托车看成蓝色 |
| **Goal Misdirection（目标误导）** | 让Agent追求与原目标不同的特定目标 | 让Agent总是选择特定商品 |

### 2.2 ARE 框架：Agent 鲁棒性评估

这是论文最核心的理论贡献。ARE（**Agent Robustness Evaluation**）将 Agent 建模为**有向图（directed graph）**：

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│ Input       │────▶│ Policy Model │────▶│ Action      │
│ Processor   │     │ (VLM)        │     │ Executor    │
└─────────────┘     └──────┬───────┘     └─────────────┘
                           │
                    ┌──────▼───────┐
                    │  Evaluator   │ ◀── Reflexion Agent
                    │  (VLM)       │
                    └──────────────┘
                           │
                    ┌──────▼───────┐
                    │ Value Func   │ ◀── Tree Search Agent
                    └──────────────┘
```

**关键思想：** 图中每条边标注流经该边的**对抗信息量（adversarial information flow）**。这样可以将 Agent 的整体鲁棒性**分解为各组件的鲁棒性贡献**。

数据流路径分析：
1. **直接攻击路径**：对抗扰动 → 输入处理器 → 策略模型 → 动作执行
2. **反射攻击路径**：对抗扰动 → 策略模型 → 评估器 → 反馈修正 → 动作执行
3. **搜索攻击路径**：对抗扰动 → 策略模型 → 价值函数 → 搜索决策 → 动作执行

### 2.3 攻击方法

论文实现了两种黑盒攻击方法：

#### （1）Captioner Attack（描述器攻击）
- 使用开源 VLM（如 BLIP-2）作为代理模型
- 通过 BIM（Basic Iterative Method）优化对抗样本，使代理模型的图像描述输出包含对抗指令
- **原理：** 如果能欺骗"较弱的"描述器输出对抗文本，那么更强的商业 VLM 也可能受到影响
- 每张图攻击耗时约1小时（A100 GPU）

#### （2）CLIP Score Attack（CLIP分数攻击）
- 直接优化图像使 CLIP embedding 与对抗目标文本的相似度最大化
- 不依赖中间描述步骤，更直接的优化目标
- 对不同目标 VLM 分别生成专用对抗样本

两种攻击都保证扰动量极小（L∞ norm 受限），人眼几乎无法分辨原图与对抗图的差异。

## 三、技术亮点

### 亮点1：首次系统性地将Agent建模为信息流图进行安全性分解

传统方法把 Agent 当作黑盒整体评估，而 ARE 框架将其拆解为**组件级的有向图模型**，每条边可以独立度量对抗信息的传播量。这使得研究者能够精确定位"哪个组件是安全短板"。

**实际发现：** 
- Reflexion Agent 的评估器（evaluator）如果被攻破，ASR（Attack Success Rate）从 0.31 升至 **0.36**（相对提升15%）
- Tree Search Agent 的价值函数被攻破后，ASR 相对提升 **20%**
- 这意味着**推理时增强技术在提升性能的同时引入了新的攻击面**

### 亮点2：揭示了"越智能越脆弱"的反直觉现象

论文发现了一个令人担忧的趋势：

| Agent类型 | ASR（GPT-4o） | 说明 |
|-----------|---------------|------|
| 基础Agent（单步） | ~15% | 最简单，但也最难被深度利用 |
| Reflexion Agent | ~31% | 自反思机制反而放大了对抗信号 |
| Tree Search Agent | 更高 | 价值函数成为新的攻击入口 |

**根因分析：** 推理时计算（inference-time compute）本质上是在 Agent 内部创建了更多的**信息反馈环路**。这些环路在正常情况下帮助 Agent 自我纠错，但在对抗条件下却成了对抗信号的**放大器**——评估器的错误判断会被反复强化，搜索过程中的偏差会被逐步累积。

### 亮点3：防御措施的效果边界分析

论文测试了多种自然防御方案：

| 防御策略 | 效果 | 局限性 |
|----------|------|--------|
| Safety Prompting（安全提示词注入） | 微弱提升 | 黑盒Agent无法控制内部prompt |
| Consistency Check（一致性检查，多次采样比对） | 中等提升 | 大幅增加推理成本（3-5x） |
| Self-Captioning（自描述+交叉验证） | 部分有效 | 对强攻击仍不够 |

关键结论：**当前没有一种轻量级防御能有效抵御有针对性的对抗攻击**，这为 Agent 安全研究指明了紧迫方向。

## 四、基准测试表现

### 4.1 主要实验结果（ASR = Attack Success Rate）

| 目标模型 | Captioner Attack ASR | CLIP Attack ASR |
|----------|---------------------|-----------------|
| GPT-4V | 43% | 52% |
| GPT-4o | 45% | **67%** |
| Gemini 1.5 Pro | 38% | 48% |
| Claude 3 Opus | 35% | 44% |

**核心发现：** 即使是最先进的商业 VLM，在仅修改一张图片（<5%像素扰动）的情况下，**最高67%的攻击成功率**意味着超过2/3的任务可以被成功劫持。

### 4.2 组件级分解结果

针对 GPT-4o + Reflexion 架构：
- 仅攻击策略模型：ASR = 0.25
- 同时攻击策略模型 + 评估器：ASR = **0.36**（相对+44%）
- 评估器完好时的自我纠正效果：ASR 从 0.31 降至 0.25（证明反射机制在正常时有保护作用）

## 六、对我们的实际启发

### 6.1 对 OpenClaw / Agent部署的安全启示

1. **输入 sanitization 是第一道防线**：对于 Browser Agent 场景，任何来自外部环境的图像/文本都应该经过清洗或降敏处理后再送入 VLM。可以考虑对截图做轻微的随机噪声注入（randomized smoothing）作为低成本防御。

2. **多组件 Agent 的安全审计不能只看入口**：ARE 框架告诉我们，攻击可以从任意组件渗透。我们的 Agent 架构中如果有 subagent 调用、工具链、记忆检索等模块，每个环节都是潜在攻击面。

3. **推理时增强 ≠ 安全增强**：reflexion、tree search、multi-agent debate 等技术在提升能力的同时可能降低安全性。在设计 Agent 时需要在架构层面考虑**安全-能力的权衡（security-capability tradeoff）**。

4. **Consistency Check 作为实用防御**：虽然论文说效果有限，但对于生产环境来说，对关键操作做多次独立采样并比对（类似 majority voting），是成本可控且有一定效果的方案。

### 6.2 研究方向参考

- 可以借鉴 ARE 的图建模思路，为我们的 Agent 系统建立**安全敏感度热力图**，标识哪些组件最容易受到对抗攻击
- VisualWebArena-Adv 的任务构造方法（手工定义对抗目标+自动评估函数）可以作为我们构建内部安全测试集的模板
