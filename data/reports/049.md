# 开源Agent项目学习 #049

## 项目名称
OpenAI Agents SDK

## GitHub链接
https://github.com/openai/openai-agents-python

## 一句话简介
OpenAI 官方出品的轻量级多智能体工作流框架，用最少的抽象原语实现生产级 Agent 应用，是此前实验性项目 Swarm 的正式升级版。

## 项目架构
SDK 围绕极简核心原语构建：

- **Agent**：LLM + 指令 + 工具 + 护栏 + 交接规则的封装单元
- **Runner**：内置 Agent Loop，自动处理工具调用、结果回传、循环直到任务完成
- **Handoffs（交接）**：Agent 可将控制权转交给其他 Agent，实现多 Agent 委派
- **Agents as Tools**：Agent 可作为工具暴露给其他 Agent 调用，构建层次化协作
- **Guardrails（护栏）**：输入/输出验证层，与 Agent 执行并行运行，快速失败
- **Sessions**：持久化会话记忆层，跨轮次保持上下文
- **Tracing**：内置追踪系统，支持工作流可视化、调试和监控

设计原则：功能足够有用，但原语足够少，学习成本低。Python-first，利用语言特性而非自创 DSL。

## 技术亮点
- **极简主义设计**：仅 Agent、Handoff、Guardrail 三个核心原语，对比 LangChain 的庞大抽象体系，上手极快。从 Swarm（纯实验）进化为生产就绪，证明了"少即是多"的架构哲学
- **Handoff + Agents as Tools 双模式编排**：既支持 Agent 间控制权转移（handoff），也支持将 Agent 包装为工具调用，灵活覆盖"接力"和"管理者"两种多 Agent 模式
- **原生 MCP 支持**：内置 MCP Server 工具集成，MCP 工具与函数工具使用方式一致，与 OpenClaw 的 MCP 生态直接兼容
- **Realtime Agents**：支持 gpt-realtime-1.5 构建低延迟语音 Agent，具备自动打断检测、上下文管理、护栏等完整功能
- **内置 Tracing + 评估闭环**：不仅追踪执行流程，还衔接 OpenAI 的评估、微调和蒸馏工具链，形成"开发→调试→评估→优化"完整闭环
- **Provider-agnostic**：虽然叫 OpenAI Agents SDK，但支持 Responses API、Chat Completions API 以及 100+ 其他 LLM（通过 LiteLLM）

## 适用场景
- 快速构建多 Agent 协作应用（客服系统、研究助手、工作流自动化）
- 需要人机协作（Human-in-the-Loop）的 Agent 流程
- 语音 Agent 开发（Realtime Agents 模式）
- 从原型到生产的 Agent 项目（Guardrails + Tracing 提供生产保障）
- MCP 工具生态集成场景

## 对我们的启发
1. **极简原语的力量**：OpenClaw 的 skill 系统（SKILL.md + 工具列表）也是一种极简设计。可以借鉴 Agent SDK 的思路，进一步精简 OpenClaw 的 Agent 配置，减少不必要的抽象层
2. **Handoff 模式值得学习**：OpenClaw 的 subagent 模式类似 Handoff，但缺乏 Agent-as-Tool 的嵌套能力。未来可以考虑让楠楠将特定 skill Agent 暴露为可被其他 Agent 调用的"工具"
3. **Guardrails 并行验证**：OpenClaw 目前主要靠 system prompt 做安全控制，可以引入类似 Guardrails 的并行验证层，在工具调用前/后自动检查安全性
4. **Tracing 闭环**：OpenClaw 的 heartbeat 和 memory 系统是基础版追踪，但缺乏可视化。内置 Tracing 的思路可以借鉴——让楠楠的每一步操作都可追溯、可调试
5. **Provider-agnostic 哲学**：OpenClaw 已经支持多模型切换（智谱/MiniMax），这与 SDK 的设计理念一致。继续保持这种灵活性很重要
