# 开源Agent项目学习 #041

## 项目名称
Hermes Agent

## GitHub链接
https://github.com/nousresearch/hermes-agent

## 一句话简介
由 Nous Research 构建的自改进 AI Agent——内置封闭学习循环，能从经验中自动创建技能、持续改进，跨会话持久记忆。

## 项目架构
Hermes Agent 采用模块化子系统架构，核心组件包括：

```
hermes-agent/
├── run_agent.py          # AIAgent 核心循环（同步编排引擎）
├── cli.py                # 交互式终端 UI
├── model_tools.py        # 工具发现/编排
├── hermes_state.py       # SQLite 会话/状态数据库
├── batch_runner.py       # 批量轨迹生成
├── agent/                # prompt 构建、压缩、缓存、元数据、轨迹
├── tools/                # 工具实现和终端环境（6种后端）
├── gateway/              # 消息网关、会话路由、投递、配对
├── cron/                 # 定时任务存储和调度
├── honcho_integration/   # Honcho 记忆集成（用户建模）
├── acp_adapter/          # ACP 编辑器集成服务
├── skills/               # 内置技能（40+）
└── environments/         # RL/基准测试环境框架
```

**核心子系统：**
- **Agent Loop**：`run_agent.py` 中的 AIAgent 负责模型选择、prompt 构建、工具执行、重试回退、压缩持久化
- **Prompt System**：分拆为 prompt_builder、prompt_caching、context_compressor，支持上下文压缩和 prompt 缓存
- **Tooling Runtime**：工具注册表、toolsets 分组、6种终端后端（local/Docker/SSH/Daytona/Singularity/Modal）
- **Session Persistence**：SQLite 存储，压缩分割后保留历史血缘
- **Messaging Gateway**：长运行编排层，支持 Telegram/Discord/Slack/WhatsApp/Signal/Email/CLI
- **Cron**：定时任务作为一等 Agent 任务实现，而非简单 shell 任务
- **Skills System**：兼容 agentskills.io 开放标准，支持社区共享

## 技术亮点
- **封闭学习循环（Closed Learning Loop）**：Agent 自主编排记忆，定期自我提醒（nudges），自主创建技能，使用时自动改进技能，FTS5 跨会话检索 + LLM 摘要，集成 Honcho 方言式用户建模
- **真正的 Serverless 部署**：6种终端后端（local/Docker/SSH/Daytona/Singularity/Modal），其中 Daytona 和 Modal 支持休眠，空闲时几乎零成本
- **Programmatic Tool Calling**：通过 `execute_code` 将多步骤工具管道折叠为单次推理调用，大幅降低延迟和 token 消耗
- **容器安全隔离**：只读根文件系统、降权 capabilities、namespace 隔离，适合生产环境部署
- **RL 研究就绪**：批量轨迹生成、Atropos RL 训练集成、ShareGPT 导出 + 轨迹压缩，由模型训练团队（Nous Research）构建
- **Subagent 并行委托**：隔离的子 Agent 拥有独立会话、终端和 Python RPC 脚本，零上下文成本流水线

## 适用场景
- **个人 AI 助手**：通过 Telegram/Discord 等平台与用户持续交互，跨会话记忆
- **DevOps / MLOps 自动化**：40+ 内置技能覆盖 GitHub 工作流、研究、部署等场景
- **定时自动化**：自然语言 cron 调度，无人值守执行报告、备份、简报
- **RL 数据生成**：批量轨迹采集和导出，用于模型微调
- **多环境开发**：从 $5 VPS 到 GPU 集群到 serverless，灵活部署

## 对我们的启发
1. **封闭学习循环理念与 OpenClaw 的 Skill 系统高度契合**：Hermes 的"Agent 从经验中创建技能"和我们楠楠的 proactive learning（SELF-IMPROVING skill）思路一致。可以考虑让楠楠在完成任务后自动提取 reusable pattern 写成新 skill
2. **Honcho 用户建模值得借鉴**：通过 dialectic 方法深度建模用户画像，比我们的 USER.md 静态文件更动态。可以考虑集成类似机制，让楠楠自动从对话中提炼对学长偏好的理解
3. **Programmatic Tool Calling 很聪明**：将多步骤管道折叠为单次调用，这对降低延迟和成本非常有效。OpenClaw 的子 Agent 机制可以借鉴这种思路，减少不必要的多轮交互
4. **Serverless 休眠部署**：Modal/Daytona 的 idle hibernation 模式对低成本运行 Agent 很有价值，值得关注
5. **SOUL.md 和 Context Files 概念**：Hermes 也用 SOUL.md 定义 Agent 人格——说明这个模式正在成为行业共识，我们的架构走在正确方向上
