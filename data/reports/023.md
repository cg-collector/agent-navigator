All-Hands AI 是一个开源社区，专注于为所有人构建 AI 软件开发 Agent。OpenHands 是其核心产品，定位为**云端编码 Agent 的开放平台**，能让 AI Agent 像人类开发者一样修改代码、运行命令、浏览网页、调用 API 等。

---

## 2. 核心定位

> _"An Open Platform for AI Software Developers as Generalist Agents"_

OpenHands 的核心愿景是：**让 AI Agent 成为一个通才型软件开发者**，能够在真实计算环境中完成复杂、多步骤的软件工程任务。与专注于代码补全的 Copilot 类工具不同，OpenHands 追求的是**自主、异步、长时间执行**的 Agent 能力（类似 Cognition AI 的 Devin、Anthropic 的 Claude Code）。

## 3. 架构演进：从 V0 到 V1

### 3.1 V0：单体架构的问题

OpenHands 最初采用单体架构（V0），将 Agent 逻辑、评估、应用全放在一个代码库中，并以沙箱执行为核心假设设计。这种设计在快速原型阶段非常高效，但随着项目规模扩大，暴露了严重问题：

- **紧耦合**：Agent 逻辑与沙箱、执行环境深度绑定
- **僵化的沙箱假设**：本地 CLI 执行需要特殊处理，导致代码重复
- **配置蔓延**：大量可变配置难以管理
- **研究与生产耦合**：难以区分实验性功能和生产级稳定性

### 3.2 V1：模块化 SDK 架构

OpenHands V1 进行了完整的架构重构，解耦为 **四个独立包**：

```
┌──────────────────────────────────────────────────────┐
│              Applications (CLI, Web UI, GitHub App) │
├──────────────────────────────────────────────────────┤
│         OpenHands Software Agent SDK                │
│  (Agent Core · Event-Sourced State · Config · Types) │
├────────────────┬─────────────────┬──────────────────┤
│   Tool Package │ Workspace Package│  Server Package  │
│  (MCP · Tools) │ (Local · Remote  │ (REST/WebSocket) │
│                │  Sandboxed)      │                  │
└────────────────┴─────────────────┴──────────────────┘
```

#### 四个设计原则：

1. **Optional Isolation（可选隔离）**：Agent 默认本地运行，需要安全管控时可切换到沙箱隔离环境
2. **Stateless by Default + Single Source of Truth**：所有组件（Agent、Tools、LLM）在构造时验证不可变；单一对话状态对象记录所有可变上下文，支持确定性回放和会话恢复
3. **Strict Separation of Concerns（关注点严格分离）**：下游应用（CLI、Web UI、GitHub App）共享 SDK 而非复制逻辑
4. **Two-Layer Composability（两层可组合性）**：开发者可以独立组合 SDK、Tools、Workspace、Server 四层包，并通过添加或替换类型化组件（Tools、Agent）安全地扩展

## 4. Agent-Computer Interface（ACI）

OpenHands 的一大核心创新是 **ACI（Agent-Computer Interface）**，最初由 SWE-agent 提出，OpenHands 在其基础上进一步发展。

ACI 的核心思想：**为软件工程任务专门设计 Agent 与计算机系统的交互界面**，而不是让 Agent 直接使用原始的 shell 文件系统 API。一个设计良好的 ACI 应该：

- **约束动作空间**：将 Agent 的操作限制在安全、合理、可控的范围内
- **状态可视化**：让 Agent 清楚当前工作区状态
- **错误可恢复**：操作失败后 Agent 能理解错误并重试
- **最小化令牌开销**：高效表达操作意图

OpenHands 的 ACI 包含：文件编辑、代码搜索、Bash 命令执行、浏览器自动化、API 调用等针对软件开发的专业工具集。

## 5. 技术亮点

### 5.1 模型无关的多 LLM 路由（Model-Agnostic Multi-LLM Routing）
支持 **100+ LLM 提供商**的路由切换，不绑定特定模型商。这是目前主流 Agent SDK 中覆盖面最广的。

### 5.2 内置安全分析（Built-in Security Analyzer）
对 Agent 的每一步操作进行安全分析，防止危险命令的执行。

### 5.3 灵活的生命周期控制
- **Pause/Resume（暂停/恢复）**：支持中途停止和继续执行
- **Sub-Agent Delegation（子 Agent 委托）**：将复杂任务分解给专门的子 Agent
- **History Restore（历史恢复）**：恢复历史会话状态
- **Deterministic Replay（确定性回放）**：基于事件溯源模型，支持精确回放 Agent 执行过程

### 5.4 本地到远程执行的可移植性
一套代码，可以本地运行（适合快速原型），也可以一键部署到远程沙箱（适合生产安全隔离），无需改代码。

### 5.5 交互式工作区界面（Built-in Workspace Interfaces）
OpenHands 不仅仅是一个 SDK，还自带完整的交互界面：
- **浏览器内 VSCode IDE**：在 Web 界面中查看和编辑代码
- **VNC 桌面**：完整的远程桌面体验
- **持久化 Chromium 浏览器**：支持 Web 自动化任务

### 5.6 REST/WebSocket 服务器
内置生产级服务器，支持远程 Agent 部署和实时交互，有别于纯库式 SDK。

### 5.7 Benchmark 表现
在 **SWE-Bench Verified** 和 **GAIA** 基准上展示了强性能。

## 6. 对 OpenClaw 的启发

### 6.1 事件溯源+确定性回放 → 调试能力提升
OpenHands 的事件溯源模型非常值得借鉴。OpenClaw 如果引入**会话状态快照和历史回放**机制，将极大提升 Agent 调试能力——开发者可以精确重现 Agent 每一步决策，找到问题所在。

### 6.2 可选隔离模型 → 安全性与灵活性的平衡
OpenClaw 在执行敏感操作（文件系统、系统命令）时，可以借鉴 OpenHands 的**可选沙箱隔离**策略：
- 普通任务 → 本地快速执行
- 高风险操作 → 自动切换到隔离环境

### 6.3 模型无关路由 → 多模型支持
OpenHands 的多 LLM 路由架构（100+ 提供商）证明了一条可行路径：Agent 层面不应该与特定模型强绑定。OpenClaw 可以进一步探索**任务自适应模型选择**——简单任务用便宜模型，复杂任务动态升级到更强模型。

### 6.4 ACI 理念 → 工具设计的专业化和约束化
OpenHands 的 ACI 启发我们：**工具设计应该是领域特定的、经过精心约束的**，而非提供原始的底层 API。一个设计良好的工具接口能显著提升 Agent 的可靠性，降低幻觉和错误操作的概率。

### 6.5 双层可组合性 → OpenClaw 扩展性的参照
OpenHands 的 SDK/Tools/Workspace/Server 四层分离，对 OpenClaw 的插件架构有直接参考价值——将 Agent 核心、工具实现、执行环境、网络服务解耦，让不同场景（本地调试、生产部署）可以复用同一套 Agent 逻辑。

## 7. 总结

OpenHands 是当前最具影响力的开源 AI 软件开发 Agent 平台之一，64k+ GitHub Stars 的体量证明了其社区认可度。其 V1 架构从单体到模块化的演进过程本身就是一部优秀的架构重构教科书。**ACI 设计理念**、**事件溯源状态管理**、**可选隔离执行模型**、**多 LLM 路由**等技术亮点，对 OpenClaw 在 Agent 调试可靠性、安全执行、多模型支持等方向有直接的启发价值。
