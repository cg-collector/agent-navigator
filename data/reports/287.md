# AI Agent 项目研究报告 #287：UGround — 像人类一样导航数字世界的通用视觉定位模型

## 1. 项目背景与问题定义

**UGround**（全称 *Navigating the Digital World as Humans Do: Universal Visual Grounding for GUI Agents*）由俄亥俄州立大学 OSU-NLP Group 与 Orby AI 联合提出，发表于 **ICLR 2025 Oral**（接收率仅 1.8%），是 GUI Agent 领域最具影响力的工作之一。

### 核心痛点

当前主流 GUI Agent（如 Mind2Web、SeeAct 等）严重依赖 **文本化 UI 表示**——HTML 文档或无障碍树（Accessibility Tree）来完成元素定位和操作。这种范式存在三大根本缺陷：

1. **噪声与不完整性**：2024 年对全球前 100 万网站的调研显示，95.9% 的首页存在无障碍合规错误，平均每页 56.8 个错误。HTML 更是充斥着与用户无关的冗余信息。
2. **计算开销膨胀**：HTML 编码消耗的 token 数可达对应视觉表示的 **10 倍以上**，在长 horizon 任务中延迟和成本被逐步放大。
3. **平台依赖性强**：获取 a11y tree 在桌面/移动端环境本身耗时且不稳定。

UGround 的核心命题是：**GUI Agent 能否像人类一样，仅通过视觉感知界面并执行像素级操作？** 答案是肯定的——关键在于一个足够强大的**视觉定位（Visual Grounding）模型**。

## 2. 核心架构深度剖析

### 2.1 SeeAct-V 框架：纯视觉的 Agent 架构

UGround 并非一个完整的 Agent，而是作为核心模块嵌入 **SeeAct-V 框架**中运行：

```
┌─────────────────────────────────────────────┐
│               SeeAct-V Pipeline              │
│                                              │
│  ┌──────────┐    ┌──────────────────────┐    │
│  │ 截图输入  │───▶│ MLLM 规划器 (GPT-4o) │    │
│  │ (Screenshot)   │ 生成文本动作计划       │    │
│  └──────────┘    └────────┬─────────────┘    │
│                            │                  │
│                     "点击搜索按钮"             │
│                            ▼                  │
│                   ┌──────────────┐            │
│                   │  UGround     │            │
│                   │ (视觉定位模型) │            │
│                   │              │            │
│                   │ 输出: (x, y)  │            │
│                   │   像素坐标     │            │
│                   └──────┬───────┘            │
│                          ▼                    │
│                   执行点击/输入等像素级操作      │
└─────────────────────────────────────────────┘
```

**数据流**：截图 → MLLM 生成文本动作描述 → UGround 将描述映射为屏幕坐标 → 执行器完成操作。注意 "Click" 在 "Type" 前自动插入。

### 2.2 UGround 模型架构

UGround 基于 **LLaVA 架构**做轻量适配：

- **视觉编码器**：保留 LLaVA 的视觉 tokenizer，将截图编码为视觉 token 序列
- **语言解码器**：在 LLM 输出层增加一个回归头，直接输出归一化的 (x, y) 坐标
- **训练目标**：给定截图 + 指称表达式（Referring Expression），预测目标 UI 元素的中心点坐标

关键设计决策：
- **分辨率自适应**：支持任意分辨率的 GUI 截图输入（桌面横屏、移动端竖屏、Web 各种尺寸）
- **模块化解耦**：定位模型与规划 MLLM 完全解耦，可即插即用替换不同后端
- **多尺度训练**：数据中包含从手机到 4K 显示器的各种分辨率

### 2.3 数据合成管线

这是 UGround 最精巧的部分。团队构建了**迄今最大规模的 GUI 视觉定位数据集**：

| 维度 | 规模 |
|------|------|
| 总元素数 | **1000 万** GUI 元素 |
| 覆盖截图 | **130 万** 张 |
| 主要来源 | Web 页面（~95%） |
| 平台泛化 | Web → Desktop / Mobile 零样本迁移 |

**合成策略**完全基于 Web 页面的双表示特性：

1. **爬取真实网页** → 同时获得 HTML DOM 和渲染截图
2. **利用 HTML 元素元数据**自动生成三类指称表达式：
   - **视觉型 RE**："蓝色的提交按钮"、"顶部导航栏中的 Logo"
   - **位置型 RE**："右上角的关闭按钮"、"页面中间的搜索框"
   - **功能型 RE**："用于登录的按钮"、"下载文件的链接"
3. **坐标标注**：通过 HTML 元素到渲染像素的精确映射获得 ground-truth bounding box，取中心点

**核心洞察**：虽然训练数据 95% 来自 Web，但 GUI 设计语言跨平台高度共享（按钮、输入框、列表等的视觉模式一致），因此模型能零样本迁移到桌面和移动端。

## 3. 技术亮点

### 亮点一：纯视觉范式的可行性证明

UGround 首次系统性地证明：**仅用视觉输入的 GUI Agent 可以达到甚至超越使用额外文本信息的 SOTA Agent**。这在之前被认为是不可行的——业界普遍认为视觉-only 会丢失太多语义信息。

具体证据：
- 在 **Mind2Web-Live**（在线 Web Agent 基准）上，GPT-4o + UGround 达到 **50.8% 完成率**，超越 GPT-4o + Choice（文本基线）的 47.6%
- 在 **AndroidWorld**（在线移动 Agent 基准）上，GPT-4o + UGround-V1-7B 达到 **44.0% 任务成功率**，远超 GPT-4 + SoM（图文混合）的 25.4%

这意味着 HTML/a11y tree 不是必需品，而是拐杖。去掉它们反而减少了噪声、降低了延迟。

### 亮点二：Web 合成数据的跨平台零样本泛化

仅用 Web 数据训练的 UGround，在 Desktop 和 Mobile benchmark 上表现优异：

| Benchmark | 平台 | UGround-V1-7B | 最佳竞品 |
|----------|------|---------------|---------|
| ScreenSpot | Mobile Text | 93.0% | Claude CU 98.2% |
| ScreenSpot | Desktop Icon | 76.4% | AGUVIS-72B 77.9% |
| ScreenSpot | Web Icon | 84.0% | UGround自身最佳 |
| AndroidControl | High | 49.8% | 竞品最高 55.0% |

7B 参数量的 UGround-V1-7B 在多个维度上逼近或超过 72B 级别的通用模型（如 Molmo-72B、AGUVIS-72B），证明**领域专用小模型 > 通用大模型**的路线在 GUI 定位任务上成立。

### 亮点三：模块化解耦设计 — 规划与执行分离

UGround 采用**规划-定位分离**架构，而非端到端联合训练：

- **优势 A**：定位模型可以独立迭代升级，无需重新训练昂贵的 MLLM 规划器
- **优势 B**：同一个 UGround 可以搭配 GPT-4、GPT-4o、Claude 等任意 MLLM 后端
- **优势 C**：推理时定位模型仅需处理短文本+单张图，成本极低（7B 模型即可）

对比 OmniParser、ShowUI 等端到端方案，UGround 的模块化设计更具工程实用性。

## 4. 基准测试表现

### 4.1 视觉定位基准 — ScreenSpot

ScreenSpot 是最权威的 GUI 元素定位 benchmark，覆盖三个平台六类元素：

| 模型 | Mobile Avg | Desktop Avg | Web Avg | **总体** |
|------|-----------|-------------|---------|---------|
| SeeClick (前SOTA) | 65.0 | 51.1 | 44.1 | 53.4 |
| OmniParser | 75.5 | 77.5 | 66.2 | 73.0 |
| ShowUI | 79.2 | 68.7 | 72.7 | 75.1 |
| **UGround-V1-7B** | **86.5** | **85.1** | **87.5** | **86.3** |
| **UGround-V1-72B** | **88.8** | **90.3** | **89.2** | **89.4** |

UGround-V1-7B 相比前 SOTA ShowUI 提升 **11.2 个百分点**，72B 版本更是接近 Claude Computer Use（92.0%）的水平。

### 4.2 离线 Agent 基准

| 基准 | 指标 | GPT-4o+Choice(文本) | GPT-4o+UGround | 提升 |
|------|------|---------------------|----------------|------|
| Multimodal-Mind2Web | Element Acc. | ~32.9% | **46.8%** | +13.9 |
| AndroidControl | Step Acc (High) | 42.1% | **48.4%** | +6.3 |
| OmniACT | Action Score | 17.0% | **32.8%** | +15.8 |

### 4.3 在线 Agent 基准

| 基准 | 指标 | 最佳文本基线 | UGround最佳 |
|------|------|------------|-----------|
| Mind2Web-Live | Success Rate | 22.1% | 23.1% |
| AndroidWorld | Success Rate | 30.6% | **44.0%** |

AndroidWorld 上 **44.0%** 的成绩尤为亮眼——这是纯视觉 Agent 首次在该 benchmark 上大幅反超文本基线。

## 5. 与同类项目横向对比

| 维度 | **UGround / SeeAct-V** | **OmniParser** | **ShowUI** | **OS-Atlas** | **Anthropic CU** |
|------|----------------------|---------------|-----------|-------------|-----------------|
| **范式** | 纯视觉 + 模块化解耦 | 纯视觉图标检测 | 端到端 VLA | 纯视觉定位 | 纯视觉（闭源） |
| **输入** | 仅截图 | 仅截图 | 仅截图 | 仅截图 | 仅截图 |
| **定位方式** | 回归坐标 | 检测框候选 | 端到端生成 | 回归坐标 | API内置 |
| **开源** | ✅ 模型+数据全开源 | ✅ | ✅ | ✅ | ❌ |
| **参数量** | 2B/7B/72B | 可变 | 可变 | 4B/7B | 未知 |
| **ScreenSpot** | 🥇 **89.4** (72B) | 73.0 | 75.1 | 81.0 | 82.9 |
| **AndroidWorld** | 🥇 **44.0%** | 未报告 | 未报告 | 未报告 | 较低 |
| **模块化** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | N/A |
| **跨平台** | Web+Desktop+Mobile | 主要 Desktop/Web | 多平台 | 多平台 | 多平台 |

**核心优势**：
- 相比 OmniParser/ShowUI：更高的定位精度，且模块化设计更易集成
- 相比 OS-Atlas：更强的跨平台泛化能力，且数据集完全开源
- 相比 Anthropic CU：开源可控，且在部分 benchmark 上性能接近

**劣势**：
- 仍依赖外部 MLLM 做高层规划，不是端到端方案
- Web 合成数据虽能迁移，但在极端异构 UI（如专业软件、游戏界面）上可能退化
- 定位精度仍有提升空间（vs 人类水平差距明显）

## 6. 对我们的实际启发

### 启发一：视觉优先的 Agent 架构值得投入

OpenClaw 的 browser 工具当前主要依赖 snapshot（DOM 结构）进行元素定位。UGround 证明了**纯视觉定位路线的可行性和高效性**。可以考虑：
- 在 browser 工具链中加入视觉定位备选路径：当 DOM 解析失败或过于嘈杂时，fallback 到 screenshot + 定位模型
- 对于飞书/企业内部工具等 a11y tree 不完善的应用场景，视觉路线可能是唯一可靠选择

### 启发二：小模型专用路线的性价比极高

7B 的 UGround-V1-7B 在多数场景下已经够用，推理成本低廉。这提示我们：
- GUI 定位不需要通用大模型，专用小模型即可
- 可以考虑部署轻量级 UGround 作为 OpenClaw 的本地 grounding 服务

### 启发三：数据合成的杠杆效应

UGround 用纯 Web 合成数据就实现了跨平台泛化，这个方法论可以复用：
- 如果我们要训练特定领域的 GUI Agent（如飞书专属、某 ERP 系统），可以用类似思路从目标系统的 UI 自动合成训练数据
- 关键是利用结构化元数据（DOM/控件树）来自动生成多样化的指称表达式
