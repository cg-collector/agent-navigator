## 简介

TaskWeaver 是微软推出的**代码优先（Code-First）AI Agent 框架**，专注于**数据分析和复杂任务规划执行**。与传统的纯文本对话式 Agent 不同，TaskWeaver 通过代码片段来理解和执行用户请求，能够处理 pandas DataFrame 等复杂数据结构，实现"生成代码 → 执行代码 → 分析结果"的闭环。

核心理念：**让 LLM 像专业数据分析师一样思考和工作**，用户只需用自然语言描述需求，TaskWeaver 自动完成从规划到执行的全流程。

## 架构设计

```
用户请求（自然语言）
       ↓
   Planner（规划器）
       ↓  分解任务
   Code Interpreter（代码解释器）
       ↓  执行代码
   Plugin System（插件系统）
       ↓  调用工具/函数
   代码执行环境（Container/Sandbox）
       ↓
   返回结果 + 状态更新
       ↓
   循环迭代直到任务完成
```

### 核心组件

1. **Planner（规划器）**: 理解用户意图，将复杂任务分解为可执行的子步骤
2. **Code Interpreter（代码解释器）**: 将规划结果转化为可执行的 Python 代码
3. **Plugin System（插件系统）**: 以函数形式封装各种工具（如数据库查询、RAG、Web搜索等）
4. **State Management（状态管理）**: 维护聊天历史 **和代码执行历史**，保留内存中的数据状态
5. **Experience System**: 支持将历史对话转化为持久化的经验，加速后续类似任务处理

### 状态管理 — 核心差异点

TaskWeaver 区别于其他框架的关键设计：**同时保留聊天历史和代码执行历史**，包括内存中的数据。这使其特别适合处理**高维表格数据（如 DataFrame）** 的复杂分析任务。

## 技术亮点

### 1. 代码优先执行
- 不是让 LLM 直接输出文字，而是生成并执行 Python 代码
- 支持 pandas、numpy 等数据分析库原生操作
- 代码在沙箱容器（Container Mode）中安全执行

### 2. 多角色协作
- 支持自定义角色（Role），如 Recepta（推理增强）、Planner、CodeInterpreter
- 可以为不同角色配置不同的 LLM（Multi-LLM 支持）
- 角色之间通过共享内存（Shared Memory）通信

### 3. 插件系统
- 以 Python 函数形式定义插件
- 支持 RAG、数据查询、Web 搜索、视觉理解等丰富插件
- 2025年新增 Vision Input 支持和 Recepta 推理角色

### 4. 经验学习（Experience）
- 将历史成功案例转化为可复用的经验
- 支持静态/动态经验选择
- 可将聊天历史个性化地转化为 Agent 的"记忆"

### 5. 安全执行
- 默认切换到 Container 模式执行代码
- 与 AgentOps 集成提供可观测性和监控
- 支持 Docker 一键部署

### 6. 多模型支持
- 支持 OpenAI、Azure OpenAI、LiteLLM、Ollama、Gemini、Qwen 等多种 LLM 后端
- 也支持本地模型

## 对楠楠/学长的启发

### 对楠楠的启发 👧
1. **代码能力是 AI Agent 的硬道理**：TaskWeaver 的"代码优先"理念告诉我们，光会聊天不够，真正能执行复杂任务的 Agent 必须能生成和执行代码。楠楠如果要深入 AI 开发，学好 Python 和数据分析很重要～
2. **状态管理让 Agent 更聪明**：TaskWeaver 同时维护对话历史和执行历史，这让 Agent 能"记住"中间结果。对楠楠来说，这也是一种学习方法的启示——不只是记答案，还要记过程！

### 对学长的启发 🧑‍💻
1. **数据分析场景的 Agent 落地**：TaskWeaver 展示了如何用 Agent 做数据分析（从数据库拉取数据 → 检测异常 → 可视化），这对学长做数据类项目很有参考价值
2. **Multi-LLM 架构**：为不同角色配置不同 LLM 的设计很实用，可以针对规划用 GPT-4，针对代码执行用更快的模型，节省成本
3. **安全沙箱很重要**：做能执行代码的 Agent 时，安全隔离是必须考虑的，TaskWeaver 的 Container Mode 是个好示范

## 总结

TaskWeaver 是一个**专注于数据分析场景**的代码优先 Agent 框架，它的设计哲学（代码执行 + 状态管理 + 插件化）在众多 Agent 框架中独树一帜。虽然它不像 LangGraph 那样通用，但在数据分析领域做得非常深入。对于想构建**能真正执行代码的 AI 助手**的开发者来说，TaskWeaver 的架构思路和源码都值得研究。
