# 开源Agent项目学习 #043

## 项目名称
OpenSage (Self-programming Agent Generation Engine)

## GitHub链接
https://github.com/opensage-agent/OpenSage

## 一句话简介
首个让 LLM 自动创建子Agent拓扑和工具集的 Agent 开发套件（ADK），由 UC Berkeley Dawn Song 团队提出，实现了从"人设计Agent"到"AI自己编程Agent"的范式转变。

## 项目架构
OpenSage 围绕 Agent Development Kit（ADK）的三大核心组件构建：

1. **自生成拓扑（Self-generated Topology）**：Agent 可以在运行时动态创建、执行、终止子 Agent，自主决定多 Agent 系统的结构和依赖关系，无需人工预设。

2. **自生成工具集（Self-generated Toolkits）**：Agent 能根据任务需求自动创建和管理工具，突破固定工具集的限制。内置了针对软件工程任务的专用工具包。

3. **层次化图记忆系统（Hierarchical Graph-based Memory）**：采用图结构的层级记忆，支持高效的记忆存储、检索和管理，让 Agent 能从历史交互中学习。

核心产品是 **SageAgent**——基于 OpenSage 构建的顶级 Agent，在多个基准测试上超越现有 SOTA。

## 技术亮点
- **首个"自编程"ADK**：不依赖人类手动设计拓扑和工具，LLM 自主生成，类似现代 ML 从手工特征到端到端学习的跨越
- **动态拓扑管理**：运行时按需创建/销毁子 Agent，不同任务使用不同结构，泛化能力强
- **图记忆系统**：层次化的 graph-based memory，比简单的对话历史或 KV 存储更高效，支持复杂任务的上下文管理
- **三大基准验证**：在三个 SOTA agentic benchmark 上超越 Google ADK、OpenAI ADK、Claude ADK、LangChain 等主流框架
- **严格消融实验**：每个组件（拓扑、工具、记忆）的贡献都有量化验证

## 适用场景
- **复杂软件工程任务**：代码生成、调试、重构等需要多步推理和多工具协作的场景
- **需要动态任务分解的场景**：不同任务需要不同的 Agent 结构，无法预设固定流水线
- **多 Agent 协作系统**：需要子 Agent 自主组建团队、分配角色的场景
- **研究平台**：作为 ADK 研究的基础设施，探索"AI设计AI"的新范式

## 对我们的启发
OpenSage 的核心思想——**让 Agent 自己决定如何组织自己**——对 OpenClaw 有直接启发：

1. **动态子 Agent 生成**：OpenClaw 目前的 subagent 是主 Agent 根据任务预定义生成的。如果能让 Agent 自主判断"这个任务需要几个子Agent、分别负责什么"，会更智能。比如复杂分析任务可以自动拆分成"搜索Agent + 写作Agent + 审查Agent"。

2. **自生成工具**：楠楠目前用固定的 SKILL.md 描述工具。如果 Agent 能根据场景动态组合工具或生成新工具链（比如"爬取这个网站，我来自定义一个 scraper"），处理未知任务的能力会大幅提升。

3. **图记忆**：OpenClaw 的 MEMORY.md + daily notes 是扁平文本。图结构的记忆系统可以更好地捕捉实体关系（比如"学长→喜欢→冰壶"、"项目A→依赖→项目B"），让 Agent 真正"理解"而非"检索"。

4. **从人设计到AI设计**：这是最深刻的启发。目前 Skills 都是人工编写的 SKILL.md，未来如果能实现 Skill 的自动生成和迭代（Agent 自己发现需要什么能力，自己写 Skill，自己测试改进），就接近 OpenSage 的愿景了。
