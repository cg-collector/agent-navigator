## 📝 项目简介

AutoGen 是微软研究院开发的开源编程框架，专门用于构建**多智能体（Multi-Agent）AI 应用系统**。它的核心理念是让多个 AI Agent 可以自主行动或与人类协同工作，通过对话和工具调用来解决复杂任务。

AutoGen 的定位类似于"AI Agent 领域的 PyTorch"——提供底层抽象和构建块，让开发者可以灵活地组合出各种多智能体工作流。项目还提供了 **AutoGen Studio** 无代码 GUI 工具，方便快速原型验证。

> ⚠️ 注意：微软已推出新的 [Microsoft Agent Framework](https://github.com/microsoft/agent-framework) 作为继任者，但 AutoGen 仍持续维护并接收安全补丁。

## 🏗️ 架构设计

### 核心分层架构

AutoGen 采用**分层可扩展设计**，每层职责清晰，上层依赖下层：

```
┌─────────────────────────────────────┐
│         应用层 (Applications)        │  ← 用户构建的具体 Agent 应用
├─────────────────────────────────────┤
│       编排层 (Orchestration)         │  ← Agent 间对话路由、工作流管理
│   - Sequential / RoundRobin / etc.  │
├─────────────────────────────────────┤
│    AgentChat 层 (Agent Runtime)      │  ← Agent 定义、工具绑定、消息处理
│   - AssistantAgent                  │
│   - ToolUseAgent                    │
│   - CodingAgent                     │
│   - AgentTool (Agent as Tool)       │
├─────────────────────────────────────┤
│     扩展层 (Extensions)              │  ← 模型客户端、MCP、代码执行沙箱
│   - OpenAI / Azure / Anthropic      │
│   - MCP Server 集成                 │
│   - Docker Code Execution           │
└─────────────────────────────────────┘
```

### 核心模块

1. **AgentChat (`autogen-agentchat`)** — 核心 Agent 运行时
   - `AssistantAgent`：通用对话 Agent，支持 LLM + 工具调用
   - `CodingAgent`：专用代码编写 Agent，带沙箱执行
   - `ToolUseAgent`：纯工具调用 Agent（无 LLM）
   - `AgentTool`：将一个 Agent 包装为另一个 Agent 的工具（**Agent 嵌套/层级调用**）

2. **模型客户端抽象 (`autogen-ext.models`)** — 统一的 LLM 接口
   - 支持 OpenAI、Azure OpenAI、Anthropic 等多种模型后端
   - 异步设计，原生支持流式输出

3. **MCP 集成 (`autogen-ext.tools.mcp`)** — 原生支持 Model Context Protocol
   - 通过 StdioServerParams 连接任意 MCP Server
   - McpWorkbench 管理多个 MCP 服务器的生命周期
   - 示例：集成 Playwright MCP 实现网页浏览能力

4. **代码执行沙箱** — Docker 隔离的代码执行环境
   - 安全运行 Agent 生成的代码
   - 支持 Python、Bash 等多种语言

5. **AutoGen Studio** — 可视化无代码界面
   - 拖拽式搭建多 Agent 工作流
   - 快速原型验证和演示

### 关键设计模式

- **Conversation-Based Programming**：Agent 之间通过结构化消息对话来协作
- **Agent-as-Tool 模式**：专家 Agent 可以被包装成通用 Agent 的工具，实现层级分工
- **事件驱动架构**：基于异步事件的消息传递机制
- **Tool-Use Loop**：内置工具调用循环，自动处理 LLM → 工具 → 结果 → LLM 的迭代

## ✨ 技术亮点

### 1️⃣ Agent-as-Tool：优雅的 Agent 层级嵌套

这是 AutoGen 最具创新性的设计之一。通过 `AgentTool` 类，可以将任意 Agent（如数学专家 Agent）包装为另一个 Agent 可调用的工具：

```python
math_agent = AssistantAgent("math_expert", system_message="You are a math expert.")
math_agent_tool = AgentTool(math_agent)

general_agent = AssistantAgent("assistant", tools=[math_agent_tool, chemistry_agent_tool])
```

这意味着：
- **专家 Agent 可以作为"技能"被复用**，而不是简单的函数调用
- 每个"工具"本身是一个完整的推理循环（LLM + 思考 + 输出），而非固定逻辑
- 天然支持**深层嵌套**——Agent 调用 Agent 调用 Agent

**对比我们的启发：** OpenClaw 的 subagent 机制本质上也是类似思路，但 AutoGen 将其形式化为一级抽象，更通用。

### 2️⃣ 原生 MCP（Model Context Protocol）集成

AutoGen 是最早原生支持 MCP 协议的主流框架之一。通过 `McpWorkbench` 可以无缝连接任何 MCP Server：

```python
async with McpWorkbench(StdioServerParams(command="npx", args=["@playwright/mcp"])) as mcp:
    agent = AssistantAgent("browser", workbench=mcp)
```

这使 Agent 的能力可以通过**标准协议**动态扩展，无需修改框架核心代码。MCP 正在成为 AI Agent 工具生态的事实标准。

### 3️⃣ 分层解耦 + 扩展点设计

AutoGen 的四层架构（应用→编排→Agent→扩展）实现了极佳的关注点分离：
- **模型无关**：切换 LLM 只需换 ModelClient，不影响业务逻辑
- **工具无关**：工具来源可以是 Python 函数、MCP Server、或其他 Agent
- **编排灵活**：支持顺序执行、轮询、自定义图等多种编排策略

这种设计使得框架核心保持轻量，同时扩展性极强。

## 💡 对楠楠的启发

### 我们可以学到什么？

1. **Agent 嵌套调用的形式化**
   - AutoGen 的 `AgentTool` 模式值得借鉴。我们目前的 subagent 是一次性派生，但如果能将常用能力（如"写研究报告"、"分析股票"）封装为可复用的 Agent-Tool，就能实现更高层的编排效率。
   - **具体想法：** 可以考虑为 OpenClaw 设计"Skill Agent"概念——把复杂 skill 封装为一个有独立推理能力的子 Agent，主 Agent 通过标准化接口调用。

2. **MCP 作为工具扩展的标准协议**
   - AutoGen 对 MCP 的深度集成说明这是行业趋势。我们在设计工具系统时，应该优先考虑兼容 MCP 协议，这样未来可以无缝接入整个 MCP 生态系统（数据库、浏览器、API 等）。
   - **具体想法：** 研究 OpenClaw 是否已有 MCP 支持，如果没有可以作为 feature request 提出。

3. **分层架构的可维护性经验**
   - AutoGen 从 v0.2 到 v0.4 的大规模重构（从群聊式到分层式）说明：**早期快速迭代很重要，但及时重构更重要**。当 Agent 数量和交互模式变复杂时，必须引入清晰的分层。
   - 这对我们自己的 skill 系统和 memory 系统有参考价值——当前的设计是否足够模块化？是否需要更清晰的层次划分？

4. **无代码 GUI 的用户触达策略**
   - AutoGen Studio 让非开发者也能体验多 Agent 系统。这对降低 AI Agent 的使用门槛非常有效。
   - **具体想法：** 如果我们未来做面向普通用户的 Agent 产品，可视化编排界面几乎是必备的。
