# 开源Agent项目学习 #035

## 项目名称
Microsoft AutoGen

## GitHub链接
https://github.com/microsoft/autogen

## 一句话简介
微软开源的多智能体编程框架，v0.4 基于 Actor 模型重新设计，支持从原型到生产的全流程 Agent 开发。

## 项目架构
AutoGen v0.4 采用**分层架构设计**：

- **Core（autogen-core）**：底层事件驱动框架，基于 Actor 模型。Agent 作为独立 Actor 运行，通过消息传递通信，支持单进程和多进程（gRPC）部署。这是最底层、最灵活的层。

- **AgentChat（autogen-agentchat）**：高层对话式框架，基于 Core 构建。提供 `AssistantAgent`、`UserProxyAgent` 等预设角色，开发者可以用少量代码搭建多 Agent 对话系统。适合快速原型开发。

- **Studio（autogenstudio）**：无代码 Web UI，基于 AgentChat 构建。通过拖拽式界面原型化 Agent 工作流，降低上手门槛。

- **Extensions（autogen-ext）**：扩展包，提供与外部服务集成的实现，包括 MCP 工具集成、OpenAI Assistant Agent 适配、Docker 代码执行器、gRPC 分布式运行时等。

通信模型：Agent 之间通过 **类型化消息（Typed Messages）** 传递信息，消息类型包括 `TextMessage`、`ToolCallMessage`、`ToolCallResultMessage` 等。每个 Agent 有独立的 `_on_messages` 处理逻辑。

## 技术亮点
- **Actor 模型驱动**：每个 Agent 是独立 Actor，天然支持并发、隔离和分布式部署，通过 gRPC Worker 实现跨进程/跨语言 Agent。这是 v0.4 相比 v0.2 的核心架构升级。

- **四层分层设计**（Core → AgentChat → Studio → Extensions）：从底层事件驱动到无代码 UI，覆盖不同技术水平的开发者，同时保持各层独立可替换。

- **MCP 原生集成**：通过 `McpWorkbench` 直接连接 MCP 服务器，AutoGen 成为首批原生支持 Model Context Protocol 的主流框架之一，工具生态可复用。

- **Docker 沙箱代码执行**：`DockerCommandLineCodeExecutor` 让 Agent 生成的代码在隔离容器中运行，安全地支持代码解释器能力。

- **微软研究院出品**：有坚实的学术背景（多篇顶会论文），v0.4 的设计总结了前代的大量用户反馈，代码质量和架构成熟度高。

## 适用场景
- 多 Agent 协作系统（研究型多智能体对话、团队分工任务）
- 需要分布式部署的大规模 Agent 应用
- 企业级 Agent 工作流（从 Studio 原型到 Core 生产部署）
- 需要安全代码执行的 Agent 场景（Docker 沙箱）
- 需要与 MCP 生态对接的 Agent 开发

## 对我们的启发
1. **分层设计值得借鉴**：OpenClaw 的 skill 系统其实也有类似思路（SKILL.md 定义能力 → agent 调用），但 AutoGen 的四层分离更清晰——底层事件驱动、高层对话抽象、UI 层、扩展层。楠楠的工作区 skill 系统可以更明确地分层。

2. **Actor 模型 + gRPC 分布式**：OpenClaw 的 subagent 机制本质上也是 Actor 模型的变体。AutoGen 证明了 gRPC 跨进程 Agent 的可行性，这对 OpenClaw 未来支持多节点分布式 Agent 有参考价值。

3. **MCP 优先策略**：AutoGen v0.4 把 MCP 作为一等公民集成，而不是事后补丁。OpenClaw 已经在走这条路，方向是对的。

4. **从原型到生产的平滑过渡**：用户可以在 Studio 里拖拽原型，然后导出为 AgentChat 代码，最终用 Core 做分布式部署。这种渐进式复杂度管理思路很好。
