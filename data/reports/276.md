# 深度研究报告 #276：BJudge — 通过行为叙事实现 Computer Use Agent 的宽幅扩展

> **论文**: Scaling Agents for Computer Use  
> **作者**: Vincent Tu, Chih-Lun Lee, Jiachen Yang, Ang Li, Xin Eric Wang（Simular Research）  
> **发表**: ICML 2025 / arXiv:2510.02250v2（2026年2月更新）  
> **链接**: https://arxiv.org/abs/2510.02250  
> **研究日期**: 2026-04-11

---

## 一、项目背景

Computer Use Agent（CUA）是当前 AI Agent 领域最热门的方向之一——让 AI 像人一样操作电脑（点击、打字、浏览），完成真实世界的数字任务。然而，CUA 在长 horizon、复杂任务上的表现始终不稳定：单次执行非常脆弱，小错误会随时间累积放大，导致同一 agent 在相同任务上时好时坏，方差极高。

此前业界尝试了两种扩展路径：
1. **深度扩展（Deep Scaling）**：在单次 rollout 内做 step-wise Best-of-N（如 Agent S2 的 BoN），每步生成 K 个候选动作再选最优。但这种方法会过早锁定一条路径，一旦走上困难路线就无法回退。
2. **模型/提示工程优化**：通过更好的 prompt 或更强的基座模型提升单次成功率。收益有限且边际递减。

**BJudge 开创性地提出第三条路——宽幅扩展（Wide Scaling）**：并行运行多个 agent 实例生成多条完整轨迹，然后智能选择最佳轨迹。核心洞察是：不同 agent 实例在**互补的子任务子集上成功**（论文 Figure 2 清楚展示了三个实例的成功集合高度不重叠），只要能准确选出最好的那条轨迹，整体覆盖率就能大幅提升。

关键瓶颈在于：**如何评估和比较长 horizon 轨迹？** 轨迹信息密集、多模态、大部分细节与任务无关，且很多任务存在多种合法解法。BJudge 用"行为叙事（Behavior Narrative）"解决这个问题。

## 二、架构深度剖析

### 2.1 整体框架

BJudge 框架由三大模块组成：

```
用户指令 I
    │
    ▼
┌─────────────────────────────────┐
│  多实例并行执行引擎              │
│  （基于改进版 Agent S2）         │
│  N 个 agent 并行 → N 条轨迹      │
└─────────────┬───────────────────┘
              │ N 条轨迹 (screenshot + action 序列)
              ▼
┌─────────────────────────────────┐
│  行为叙事生成器 (BNG)            │
│  输入: (s_t, a_t, s_{t+1}) 三元组 │
│  输出: 行为事实 φ_t              │
│  → 将密集轨迹压缩为紧凑叙事       │
└─────────────┬───────────────────┘
              │ N 个行为叙事
              ▼
┌─────────────────────────────────┐
│  比较行为评估器 (CBE)            │
│  LLM-as-Judge: 两两比较/排序     │
│  选出最可能完成任务的轨迹         │
└─────────────┬───────────────────┘
              │ 最佳轨迹
              ▼
        返回执行结果
```

### 2.2 改进的 CUA 基线

BJudge 基于 Agent S2（当前开源 SOTA 的 agentic 框架）做了以下改进：

- **结构化动作空间**：将自由格式文本输出规范化为 `click(x,y)`, `type(text)`, `key(key_name)`, `scroll(direction)` 等结构化动作，减少解析误差
- **多策略提示**：使用 action plan + memory summary 双轨推理，agent 先制定高层计划再逐步执行
- **截图增强**：在每次操作前后都截取屏幕状态，为行为叙事提供 before/after 对比素材
- **错误恢复机制**：检测到异常状态（弹窗、白屏、加载超时时自动触发恢复策略）

### 2.3 行为叙事生成器（BNG）

这是 BJudge 最核心的创新。对于轨迹中的每个步骤 `(s_i, a_i, s_{i+1})`：

1. **输入**：操作前截图 s_i、agent 动作 a_i、操作后截图 s_{i+1}
2. **VLM 提取事实**：调用 VLM（如 GPT-4o）识别"这个动作实际产生了什么效果"，输出结构化事实 φ_i
3. **过滤压缩**：只保留任务相关的 action-effect 对，丢弃无关视觉细节
4. **组装叙事**：将所有步骤的事实串联成自然语言行为叙事

例如，原始轨迹可能有 100 张截图 × 每张几 MB，但行为叙事可能只有一段简洁的文字描述："打开了文件管理器 → 导航到 Downloads → 右键点击 file.pdf → 选择'打开方式' → 选择 Chrome 浏览器 → 文件在浏览器中打开"。

这种表示既保留了判断任务成败所需的关键信息，又大幅降低了下游 judge 的处理负担。

### 2.4 比较行为评估器（CBE）

CBE 接收多个候选行为叙事，执行两两比较排序：

- **输入**：原始指令 I + N 个候选行为叙事
- **方法**：LLM-as-Judge，对每对叙事进行比较，判断哪个更可能完成任务
- **输出**：排名最高的轨迹作为最终结果

论文实验了多种 judge 策略：
- **独立评分**：给每个叙事打分（效果较差，绝对评分难校准）
- **两两比较 + 排序**（最佳）：逐对比较后汇总排名
- **直接选优**：一次性从所有候选中选出最好（适合 N 较小时）

## 三、技术亮点

### 亮点 1：行为叙事——轨迹表示的新范式

传统 CUA 评估依赖两种方式：(1) 原始截图序列——信息过载、token 消耗巨大；(2) 最终状态检查——无法区分部分成功的轨迹。BJudge 的行为叙事是中间路线：**语义级压缩但保留因果链**。

这类似于人类复盘时的思维方式——我们不会逐帧回忆每个像素变化，而是用"我做了什么导致了什么"的高层叙述来理解自己的行为。这种表示使得：
- Judge 的上下文窗口需求从 O(截图数×分辨率) 降到 O(叙事长度)
- 不同长度的轨迹可以公平比较
- 多种解法的轨迹都能被正确评估（不硬编码唯一正确路径）

### 亮点 2：宽幅扩展 vs 深度扩展的范式转换

这是论文最重要的理论贡献。Step-wise BoN（深度扩展）的问题在于**局部最优不等于全局最优**——某一步的"最好动作"可能把 agent 引向一条更难的路径。而宽幅扩展让每条轨迹独立探索完整的解空间，最后再做全局选择。

论文的消融实验清楚证明了这一点：
- 单次 Agent S2 baseline: ~40% OSWorld
- Step-wise BoN (K=4): ~45%（+5%，边际收益低）
- Wide Scaling N=3 无 judge 随机选: ~48%
- **Wide Scaling N=3 + BJudge: ~65%（+25%）**
- **N=10 + BJudge: 72.6%**

这意味着 judge 的价值远大于单纯增加 rollout 数——**有选择的扩展才是真扩展**。

### 亮点 3：跨操作系统零样本泛化

BJudge 在 OSWorld（Ubuntu/Linux）上训练/调优后，**零样本迁移到 WindowsAgentArena 和 AndroidWorld** 都取得了强结果：
- WindowsAgentArena: 显著超越之前 SOTA
- AndroidWorld: 同样取得竞争力表现

这说明行为叙事是一种**平台无关的抽象表示**——无论底层 GUI 是 GTK、Win32 还是 Android View，"点击了什么按钮导致什么界面变化"这一层语义是通用的。这对构建跨平台的通用 CUA 具有重要启示。

## 四、基准测试表现

### OSWorld 主榜（100 步限制）

| 方法 | 成功率 | 说明 |
|------|--------|------|
| 人类表现 | 72.36% | 人类参考水平 |
| **BJudge (N=10)** | **72.6%** | **🏆 新 SOTA，首次超越人类** |
| 前任 SOTA (Agent S2+) | 63.4% | BJudge 前的最佳 |
| Claude Opus 4.5 CUA | ~61.4% | Anthropic 官方 CUA |
| GPT-5.3 Codex | ~64.7% | OpenAI Codex |
| Agent S2 (单次) | ~39.8% | 开源 baseline |

**关键数字解读**：
- 相比前 SOTA 提升 **9.2 个百分点**（绝对提升）
- 相比单次 baseline 提升 **~32.8 个百分点**
- **首次有方法在 OSWorld 上超越人类表现**
- 使用 GPT-5 作为 base model，10 个并行 rollout

### 消融实验要点

- **Rollout 数量 N**：从 1→3→5→10，收益率递减但持续正向。N=5 时已达 ~68%，性价比最高
- **Judge 策略**：两两比较 > 直接选优 > 独立评分
- **叙事粒度**：action-effect 级别事实 > 原始截图 > 仅动作序列
- **Base model**：更强模型作为 executor 和 judge 都带来一致提升

## 五、横向对比

| 维度 | BJudge | Agent S2 (前SOTA) | Anthropic CUA | OpenHands |
|------|--------|-------------------|---------------|-----------|
| **核心思路** | 多轨迹+行为judge | 单轨迹+step BoN | 单轨迹+API工具 | 单轨迹+代码执行 |
| **OSWorld得分** | **72.6%** | 63.4% | ~61.4% | ~15-20% |
| **计算成本** | 高(N倍rollout+judge) | 中(K倍step采样) | 中 | 低 |
| **是否开源** | 论文已发，代码待发布 | ✅ 开源 | ❌ 商业闭源 | ✅ 开源 |
| **跨平台** | ✅ Linux/Win/Android | 主要 Linux | 主要 Web/Mac | 主要代码 |
| **扩展性** | 投入算力即可换性能 | 受限于单轨迹 | 受限于模型能力 | 受限于沙箱 |
| **最大局限** | 成本高(N倍API调用) | 局部最优陷阱 | 平台受限 | GUI支持弱 |

**核心差异总结**：BJudge 是目前唯一一个通过"系统化多轨迹选择"而非"单模型能力提升"来突破性能天花板的方法。它证明了一个重要观点：**当前 CUA 的瓶颈不在感知或推理能力，而在执行的可靠性——通过冗余+选择可以大幅弥补**。

## 六、启发与思考

### 6.1 对 Agent 架构设计的启发

1. **冗余即正确性**：在不稳定的开放环境中（GUI 操作、网页交互），与其花巨大代价让单次执行完美，不如低成本多次执行加一个靠谱的 selector。这与分布式系统中的冗余设计思想一脉相承。

2. **轨迹抽象层的价值**：行为叙事本质上是在原始观察和最终奖励之间插入了一个**可解释的中间表示**。这对任何需要评估长序列决策的系统都有借鉴意义——不只是 GUI Agent，也包括代码生成（比较多个代码版本的执行轨迹）、机器人操作等。

3. **Judge > Executor 的杠杆效应**：论文隐含的一个重要发现是：投资于"评估能力"的 ROI 远高于投资于"执行能力"。一个中等水平的 executor 配合优秀的 judge，可以打败顶级 executor 无 judge 的方案。

### 6.2 对 OpenClaw / 实际工程的启发

- **多实例重试 + 结果选择**：OpenClaw 可以考虑对复杂浏览器任务实现轻量版 BJudge——跑 2-3 次，用 VLM 判断哪次成功了
- **行为日志标准化**：将 agent 执行过程记录为结构化的 action-event 对而非原始截图流，便于后续审查和调试
- **成本控制**：N=3-5 已经能获得大部分收益（~65-68%），不需要冲到 N=10。实际部署中可以用更小的 N 配合更快的小模型做 judge

### 6.3 局限性与未来方向

- **成本问题**：72.6% 的成绩需要 10 个 GPT-5 rollout + judge API 调用，单任务成本可观。实际落地需要探索：更小的 N、更便宜的 executor/judge 模型组合、缓存复用
- **评估上界**：即使 N→∞，如果所有 rollout 都在某类任务上失败，judge 也无力回天。宽幅扩展的增益受限于 executor 的覆盖范围
- **延迟敏感场景不适用**：需要实时响应的任务无法承受多 rollout 的等待时间
- **代码尚未开源**：论文发表于 ICML 2025，截至 2026 年 4 月代码库尚未公开（或社区复现有限），这在一定程度上限制了影响力

---

## 七、总结

BJudge 是 Computer Use Agent 领域的一个里程碑式工作。它没有发明新的感知模型或推理算法，而是通过一个优雅的架构创新——**行为叙事 + 多轨迹选择**——将 OSWorld 性能推到了超越人类的高度。这项工作的深层意义在于：它揭示了 CUA 领域的下一个突破口可能不是"更强的模型"，而是"更聪明的执行策略"。对于关注 GUI Agent 和自动化方向的从业者来说，BJudge 提供的方法论框架具有极高的参考价值和实践潜力。
