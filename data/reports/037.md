# 开源Agent项目学习 #037

## 项目名称
Semantic Kernel

## GitHub链接
https://github.com/microsoft/semantic-kernel

## 一句话简介
微软开源的企业级AI Agent编排SDK，支持构建单体Agent和多Agent协作系统，提供Python/.NET/Java三语言支持。

## 项目架构

Semantic Kernel 采用分层架构设计：

- **Kernel 核心**：轻量级依赖注入容器，管理AI服务、插件、记忆和Prompt模板
- **Agent Framework**：`ChatCompletionAgent` 是基础Agent类型，支持instructions、tools/plugins、memory
- **Multi-Agent System**：通过 `AgentGroupChat` 等编排模式实现多Agent协作，支持轮转、扇出等策略
- **Plugin System**：四种插件来源——原生代码函数（`@kernel_function`装饰器）、Prompt模板、OpenAPI规范、MCP协议
- **Process Framework**：结构化工作流引擎，用于建模复杂业务流程
- **连接层**：统一抽象多种LLM后端（OpenAI、Azure OpenAI、HuggingFace、Ollama、ONNX等）
- **向量存储抽象**：支持Azure AI Search、Elasticsearch、Chroma等向量数据库

## 技术亮点
- **多语言企业级支持**：Python + .NET + Java 三语言SDK同步迭代，这在Agent框架中非常罕见，适合大型企业技术栈统一
- **MCP原生集成**：直接支持Model Context Protocol作为插件来源，与OpenClaw的MCP架构理念一致，无需额外适配层
- **Process Framework**：提供比简单Agent链更强大的业务流程编排能力，可以用声明式方式定义复杂多步骤工作流，类似BPM但面向AI
- **模型无关设计**：通过Connector抽象层，同一套Agent代码可切换不同LLM后端，对模型锁定的防护做得好
- **结构化输出**：原生支持Pydantic/JSON Schema约束的structured output，直接映射到强类型对象

## 适用场景
- **企业内部AI应用**：微软生态（Azure、Teams、Office）的天然集成优势
- **需要多语言支持的大型项目**：后端用.NET/Java、AI团队用Python，统一框架
- **复杂业务流程编排**：Process Framework适合需要精确控制流程的企业场景
- **需要模型无关性的生产环境**：不希望被单一LLM供应商锁定

## 对我们的启发
- **MCP作为插件标准**：Semantic Kernel直接用MCP做插件协议，验证了MCP是Agent插件化的正确方向，OpenClaw的skill系统可以更深入地拥抱MCP
- **多Agent编排策略**：`AgentGroupChat`的轮转（RoundRobin）和扇出策略值得参考，OpenClaw的subagent调度可以借鉴类似的编排模式
- **Process Framework思路**：对于复杂任务（如多步骤研究、报告生成），用声明式流程而非纯对话链会更可靠，这可以启发OpenClaw的skill pipeline设计
- **模型无关是刚需**：在MiniMax和智谱之间切换时，SK的connector抽象值得学习，OpenClaw也可以更彻底地实现provider切换
