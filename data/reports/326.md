## 2. 项目简介

传统 RAG 系统将检索与生成分离：检索模块负责召回文档，生成模块负责回答。这种架构的根本缺陷在于——检索策略是预定义的、静态的，无法根据问题复杂度动态调整检索深度。对于一个需要多跳推理（multi-hop reasoning）的医学问题，传统 RAG 可能只做一次检索就匆忙作答，导致错误率极高。

Agentic RAG-R1 的核心创新在于：**将 RAG 的检索决策过程建模为可学习的策略，用 GRPO（Generalized Relevance Policy Optimization）强化学习算法直接优化模型的检索-推理行为**。模型不再被动接收检索结果，而是自主决定何时检索、检索什么、何时回溯、何时终止——形成完整的"思考-搜索-反思"闭环。

项目灵感来自 DeepSeek-R1 的思维链推理范式和 TC-RAG 的 Agent 记忆栈架构。实验表明，在 Qwen 2.5-7B-Instruct 基座上，经过仅 200 步 GRPO 训练后，MedQA 测试集的格式准确率从 39% 提升至 92%，答案准确率从 84%（无搜索）/ 79%（有搜索但未微调）提升至 87%。

## 3. 核心架构深度剖析

### 3.1 Agent 记忆栈（Agent Memory Stack）

架构灵感来自 TC-RAG，核心是一个**有状态的 Agent 记忆栈**，维护整个推理-检索的 deliberation loop。Agent 在每一步从以下动作中选择：

```
┌─────────────────────────────────┐
│         Agent Memory Stack       │
├─────────────────────────────────┤
│  Plan（规划）      — 当前未启用  │
│  Reasoning（推理）  — 核心动作   │
│  Backtrack（回溯）  — 自我纠错   │
│  Summary（摘要）    — 信息压缩   │
│  Tool Observation   — 外部检索   │
│  Conclusion（结论）  — 终止推理   │
└─────────────────────────────────┘
         ↕ 交互
┌─────────────────────────────────┐
│  ArtSearch (Wikipedia Search)   │
│  可扩展：文档/知识图谱/自定义    │
└─────────────────────────────────┘
```

**数据流**：Query → Agent 进入 Reasoning → 判断是否需要信息 → 调用 Tool（搜索）→ 获取 Observation → 继续推理或 Backtrack → 最终输出 Conclusion。整个过程形成一个**树状搜索空间**，Backtrack 机制允许 Agent 放弃错误路径。

### 3.2 GRPO 训练策略

训练流程分为两阶段：

1. **Rollout Generation**：用当前策略模型生成多条推理-检索轨迹（trajectories），每条轨迹包含完整的动作序列和中间状态。
2. **Reward 计算 & 策略更新**：对每条轨迹计算奖励信号，通过 GRPO 更新策略，强化"好的检索-推理行为"。

关键设计：**奖励不需要人工标注**，而是自动计算：
- **准确率奖励**（Accuracy Reward）：最终答案是否正确
- **格式奖励**（Format Reward）：输出是否符合预设格式
- **RAG 准确率奖励**（RAG Reward）：基于 RAGAS 框架评估检索质量

总奖励：`r_total = r_accuracy + r_format + r_rag`

## 4. 技术亮点

### 亮点一：检索行为作为策略而非规则

传统 RAG 的检索次数、检索时机都是硬编码的超参数。Agentic RAG-R1 将"是否检索"建模为 Agent 的**决策动作**——模型通过强化学习学会判断：这个子问题我是否有足够知识回答？需要多少次检索？是否需要换关键词重新检索？

这种设计的优雅之处在于：**简单问题少检索（省 token、降延迟），复杂问题多检索多回溯（保准确率）**，完全由模型自适应。

### 亮点二：Backtrack 机制实现自我纠错

大多数 RAG 系统是单向的——检索一次就往下游推进。Agentic RAG-R1 的 Backtrack 动作允许 Agent 识别到"当前推理路径走入死胡同"，主动回退到之前的推理节点，更换检索策略重新开始。这本质上是将**蒙特卡洛树搜索（MCTS）的思想融入 LLM 的推理过程**，只是用强化学习隐式学到了"剪枝"能力而非显式搜索。

### 亮点三：三重奖励信号解耦

将准确率、格式、检索质量三个维度解耦为独立奖励，避免了常见的 reward hacking 问题。例如模型可能学会生成格式完美但内容错误的回答——单独的 accuracy reward 可以抑制这种行为。RAG reward 基于 RAGAS 框架（评估上下文相关性、答案忠实度等），确保模型不仅答对，而且检索过程本身也是高质量的。

## 5. 基准测试表现

### MedQA 测试集（Qwen 2.5-7B-Instruct）

| 配置 | 格式准确率 | 答案准确率 |
|------|-----------|-----------|
| 基座模型（无搜索） | 39% | 84% |
| 基座模型 + 搜索（未微调） | 56% | 79% |
| **Agentic RAG-R1（200步训练 + 搜索）** | **92%** | **87%** |

**关键发现**：引入搜索但未微调时，答案准确率反而从 84% 下降到 79%——这验证了一个重要假设：**给模型工具但不训练它使用，反而可能有害**。Agentic RAG-R1 通过 RL 训练解决了这个问题，不仅恢复了准确率还提升了格式规范性。

### 资源效率

- 支持 DeepSpeed Zero-2 和 Zero-3 分布式训练
- 32B 参数模型仅需 2 张 A100（配合量化）
- 支持 LoRA 微调，降低训练成本

## 6. 与同类项目横向对比

| 维度 | Agentic RAG-R1 | Self-RAG (2023) | ReAct (2022) | Graph-R1 (2025) |
|------|---------------|-----------------|-------------|-----------------|
| **核心方法** | GRPO 强化学习 | SFT + 自反判别 | Prompt Engineering | GRPO + 图结构 |
| **检索决策** | 模型自主决策 | 模型自主决策 | 模型自主决策 | 模型自主决策 |
| **训练方式** | RL（端到端） | 监督微调 | 无训练（ICL） | RL（端到端） |
| **回溯能力** | ✅ Backtrack | ❌ | ❌ | ✅ 图遍历 |
| **可扩展工具** | ✅ 自定义工具 | ❌ 固定检索器 | ✅ 通用工具 | ❌ 知识图谱 |
| **开源程度** | 完全开源 | 模型开源 | 框架开源 | 论文阶段 |
| **主要局限** | 仅测试 MedQA | 泛化性一般 | 无训练增益 | 偏学术验证 |

**优势**：端到端 RL 训练、Backtrack 机制、自定义工具扩展、资源效率高。
**劣势**：仅验证了单一数据集（MedQA），泛化性待验证；无正式论文同行评审；Plan 动作尚未实现。

## 7. 对我们的实际启发

1. **"给工具不训练 = 给枪不会用"**：这个 79% → 87% 的发现对 OpenClaw 架构有直接启示——我们的 Agent 接入了大量工具（搜索、文档、代码），但从未对"何时调用哪个工具"做任何优化。可以考虑用 RL 或至少用 good/bad trajectory 的对比学习来优化工具调用策略。

2. **Backtrack 机制可集成到 Agent 循环**：当前 Agent 遇到错误只能从头开始。如果引入显式的回溯机制——记录决策节点，失败时可回退到上一个可靠状态重新推理——可以显著提升复杂任务的成功率。

3. **三重奖励解耦的设计模式**：在评估 Agent 工作质量时，可以借鉴"准确率 + 格式 + 过程质量"的解耦思路，避免单一指标导致的 reward hacking。
