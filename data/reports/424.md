**报告编号：** 424  
**研究方向：** 长期记忆（Long-Term Memory）  
**论文：** A-MEM: Agentic Memory for LLM Agents

**代码：** https://github.com/WujiangXu/A-mem （评测）、https://github.com/WujiangXu/A-mem-sys （生产级系统）

## 一、项目背景与动机

### 1.1 问题：现有记忆系统的根本缺陷

LLM Agent 在执行复杂任务时依赖记忆系统来利用历史经验，但当前主流的记忆系统（Mem0、MemGPT、SCM 等）存在三个核心问题：

| 缺陷 | 说明 |
|------|------|
| **结构固化** | 需要开发者预先定义存储结构、存储时机和检索时机 |
| **链接僵化** | 基于图数据库的系统依赖预设 schema，无法动态发现新关联 |
| **演化缺失** | 新记忆注入后，现有记忆的内容和属性不会自适应更新 |
| **泛化不足** | 固定工作流导致在新环境中表现急剧下降 |

这些缺陷导致 Agent 在开放式、长周期任务中很快"迷失"——无法把新学的知识与历史经验有机融合。

### 1.2 核心思想：Zettelkasten 方法论

论文从**Zettelkasten（卡片盒笔记法）**汲取灵感。该方法由德国社会学家 Niklas Luhmann 发明，核心理念：

- **原子性笔记**：每张卡片记录一条独立的知识点
- **动态链接**：通过语义相似性而非预设分类建立卡片间联系
- **知识演化**：新卡片加入时会引发旧卡片的关联更新，形成更高阶的认知结构

A-Mem 将这套知识管理哲学工程化，实现为 LLM Agent 的记忆子系统。

## 二、技术架构

### 2.1 整体架构：三层核心模块

```
┌─────────────────────────────────────────────────────────┐
│                   Agent 交互环境                          │
│              (用户对话 / 工具调用 / 任务执行)                 │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│              记忆写入流程 (Memory Write)                   │
│  ┌──────────┐    ┌──────────────┐    ┌───────────────┐  │
│  │ Note     │──▶ │ Link         │──▶ │ Memory        │  │
│  │ Construction│   │ Generation   │    │ Evolution     │  │
│  └──────────┘    └──────────────┘    └───────────────┘  │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│              记忆检索流程 (Memory Retrieval)               │
│     Query ──▶ Embedding ──▶ Vector Search ──▶ Top-k       │
│                                    (含 Link传播)           │
└─────────────────────────────────────────────────────────┘
```

### 2.2 记忆note的数据结构

每条记忆被表示为一个**七元组**：

```
m_i = { c_i, t_i, K_i, G_i, X_i, e_i, L_i }
```

| 属性 | 含义 | 来源 |
|------|------|------|
| `c_i` | 原始交互内容 | 用户输入 / Agent 输出 |
| `t_i` | 时间戳 | 系统时间 |
| `K_i` | LLM生成的关键词 | LLM分析`c_i + t_i`生成 |
| `G_i` | LLM生成的标签 | LLM分析`c_i + t_i`生成 |
| `X_i` | LLM生成的上下文描述 | LLM分析`c_i + t_i`生成 |
| `e_i` | 向量embedding | 文本编码器生成，用于相似度检索 |
| `L_i` | 关联记忆集合 | 链接生成阶段动态建立 |

### 2.3 Note Construction（笔记构建）

**输入**：新的交互内容 `c_i` + 时间戳 `t_i`  
**触发**：Agent 每完成一次与环境交互后自动触发（无需预设时机）

**流程**：
```
LLM( c_i ‖ t_i ‖ Prompt_Template ) → {K_i, G_i, X_i}
```

Prompt 模板引导 LLM 输出 JSON 格式的结构化属性，涵盖：
- **Keywords (K_i)**：捕捉内容核心概念
- **Tags (G_i)**：用于分类的类别标签
- **Contextual Description (X_i)**：提供丰富语义理解的摘要描述

然后使用文本编码器（如 BERT-based encoder）为 `c_i` 生成 dense 向量 `e_i`，用于后续相似度计算。

**原子性原则**：每条笔记记录一个独立完整的交互单元，符合 Zettelkasten 的原子性要求。

## 三、核心机制详解

### 3.1 Link Generation（链接生成）

**目的**：在已有记忆库中发现与新记忆语义相关的历史记忆，建立动态关联。

**流程**：

1. **相似度检索**：计算新记忆 `m_n` 的 embedding `e_n` 与历史记忆 `m_j` 的 `e_j` 的余弦相似度，取 Top-k 近邻
2. **LLM判断链接**：将新记忆 `m_n` 的完整属性与候选近邻 `m_j` 的完整属性一起输入 LLM
   ```
   LLM( m_n的所有属性 ‖ m_j的所有属性 ‖ "判断是否应建立链接" )
   ```
3. **建立链接**：对判定为"有关联"的记忆对，互相将对方加入各自的 `L_i` 集合

**关键设计**：
- 链接是**双向的**，双向确认确保关联质量
- 一个记忆可以同时存在于多个"盒子"（box）中——对应 Zettelkasten 中一张卡片可与多张卡片建立关联的特性
- 链接的建立是**完全动态**的，不依赖任何预设的关系类型

### 3.2 Memory Evolution（记忆演化）

**目的**：新记忆的注入能够触发历史记忆的上下文表征和属性自发更新。

**触发时机**：Link Generation 完成后，对每个近邻历史记忆 `m_j` 执行演化评估。

**流程**：
```
LLM( m_j的所有属性 ‖ m_n的所有属性 ‖ "评估是否应更新m_j" )
  → 决定是否修改 X_j（上下文描述）、K_j（关键词）、G_j（标签）
```

**演化类型**：
- **属性细化**：旧记忆的关键词/标签随新记忆变得更精确
- **上下文深化**：旧记忆的上下文描述因新相关记忆而获得更丰富的语义层次
- **涌现模式**：随着时间推移，高阶模式和抽象概念从多次关联中自发涌现

**意义**：这是 A-Mem 与所有现有 Agent 记忆系统的根本区别——记忆不再静态存储，而是随交互历史持续"生长"。

### 3.3 Memory Retrieval（记忆检索）

**目的**：给定当前查询 context，检索最相关的历史记忆供 Agent 使用。

**流程**：
1. 对查询 `q` 生成 embedding `e_q`
2. 通过向量相似度搜索 Top-k 最相关记忆
3. **Link传播**：检索到的每条记忆的 `L_i` 中的关联记忆也一并被提取（传递闭包扩展）

**优势**：
- 不仅返回语义直接相关的记忆，还通过链接传播覆盖了"同一知识簇"中的其他相关记忆
- 实现了类比人类"由一个记忆触发一整串相关回忆"的认知模式

## 四、代码实现分析

### 4.1 仓库结构

两个代码库：
- **AgenticMemory** (`github.com/WujiangXu/AgenticMemory`)：评测代码，Benchmark 对比实现
- **A-mem-sys** (`github.com/WujiangXu/A-mem-sys`)：生产级记忆系统实现

### 4.2 核心数据结构（伪代码）

```python
class MemoryNote:
    def __init__(self, content: str, timestamp: str):
        self.c = content          # 原始内容
        self.t = timestamp        # 时间戳
        self.k = None             # LLM生成的关键词
        self.g = None             # LLM生成的标签
        self.x = None             # LLM生成的上下文描述
        self.e = None             # embedding向量
        self.l = set()            # 关联记忆ID集合

class AgenticMemory:
    def __init__(self, llm, encoder):
        self.notes: Dict[str, MemoryNote] = {}
        self.llm = llm            # LLM调用接口
        self.encoder = encoder    # 文本编码器

    def add_interaction(self, content: str, timestamp: str):
        # Step 1: Note Construction
        note = MemoryNote(content, timestamp)
        note.k, note.g, note.x = self.llm.generate_attributes(
            content, timestamp, PROMPT_TEMPLATE_NOTE_CONSTRUCTION
        )
        note.e = self.encoder.encode(content)

        # Step 2: Link Generation
        neighbors = self.vector_search(note.e, top_k=K)
        for neighbor_id in neighbors:
            neighbor = self.notes[neighbor_id]
            should_link = self.llm.judge_link(note, neighbor, PROMPT_LINK_JUDGE)
            if should_link:
                note.l.add(neighbor_id)
                neighbor.l.add(note.id)

        # Step 3: Memory Evolution
        for neighbor_id in neighbors:
            neighbor = self.notes[neighbor_id]
            updated = self.llm.evolve(neighbor, note, PROMPT_EVOLVE)
            if updated:
                neighbor.k, neighbor.g, neighbor.x = updated

        self.notes[note.id] = note

    def retrieve(self, query: str, top_k: int = 5):
        q_emb = self.encoder.encode(query)
        candidates = self.vector_search(q_emb, top_k=top_k)
        # Link propagation
        expanded = set(candidates)
        for cid in candidates:
            expanded |= self.notes[cid].l
        return [self.notes[cid] for cid in expanded]
```

### 4.3 向量检索层

论文使用 cosine similarity 进行高效的向量检索。在实际部署中，可替换为 FAISS / Milvus / ChromaDB 等向量数据库，支持大规模记忆的快速 Top-k 查询。

## 五、实验评估

### 5.1 评测数据集

**LoCoMo（Long-Context Multi-Turn Conversational dataset）**：专为评测 Agent 长期对话记忆能力设计的多轮对话数据集，模拟真实的长程交互场景。

### 5.2 评估指标

使用 **6 种评估指标**：
- **F1 Score**（主要指标）：精确率和召回率的调和平均
- **BLEU-1**：词汇级 n-gram 相似度
- **ROUGE**：召回导向的摘要评估
- **METEOR**：多维度翻译评估
- 以及其他指标（涵盖语义相似度和上下文连贯性）

### 5.3 主要结果

```
Baseline对比（LoCoMo F1 Score）：
- 基线方法 LoCoMo:        2.55
- A-Mem:                  3.45
- 提升幅度:               +35%

对比SOTA：
- 相比不使用记忆演化(仅Base Memory): 显著领先
- 6种基础模型上均超越所有baseline
```

### 5.4 消融实验（Ablation）关键发现

| 模块 | 作用 |
|------|------|
| Link Generation | 建立记忆间动态关联，贡献显著 |
| Memory Evolution | 触发历史记忆属性自适应更新，贡献显著 |
| 两者结合 | 产生协同效应，最终性能最优 |

消融结果有力证明了 Link Generation 和 Memory Evolution 两个创新点各自独立有效，且组合后效果进一步增强。

## 六、优缺点分析

### 6.1 优势

| 优势 | 说明 |
|------|------|
| **动态自组织** | 不需要预设存储结构或检索时机，完全由 LLM 驱动动态决策 |
| **记忆自演化** | 独有创新——记忆网络随交互持续生长，而非静态存储 |
| **灵活链接** | 基于语义相似性动态建链，而非预设分类体系 |
| **广泛适用** | 在 6 种基础模型上均有效，方法通用性强 |
| **可解释性强** | 记忆的关键词/标签/链接可被人类审查和理解 |
| **NeurIPS 2025 认证** | 经过顶会 peer review 验证 |

### 6.2 局限性

| 局限 | 说明 |
|------|------|
| **LLM 调用开销** | 每条新记忆需要 2-3 次额外 LLM 调用（属性生成 + 链接判断 + 演化评估），计算成本较高 |
| **检索时延** | Link 传播机制在大型记忆库中可能带来检索延迟增长 |
| **精度依赖 LLM 能力** | 链接判断和演化评估的质量直接依赖底层 LLM 的推理能力 |
| **评估数据集单一** | LoCoMo 为对话场景，其他类型任务（代码生成、任务规划）的记忆效果有待验证 |
| **没有处理记忆遗忘** | 系统只增不减，长期运行后存储规模持续增长 |
| **向量数据库依赖** | 高效检索依赖外部向量数据库，增加了系统复杂度 |

## 七、核心启示

### 7.1 对 Agent 系统设计的启示

1. **记忆需要"能动性"（Agency）**：不是被动存储仓库，而是能主动生成上下文、建立关联、自我演化 的智能体。这代表了从 "RAG静态知识库" 到 "Agentic Memory动态知识网络" 的范式转变。

2. **Zettelkasten 方法论适用于 AI**：人类知识管理的优秀方法论（原子性笔记 + 动态链接 + 持续演化）可以成功迁移到 AI Agent 系统设计中。

3. **记忆演化优于记忆检索**：Agentic Memory 的核心创新在于记忆不是只写不用，而是能随新经验持续更新。这比单纯优化检索策略更接近人类记忆的运作方式。

### 7.2 对 Long-Term Memory 方向的启示

- **从"存多少"到"怎么组织"的转向**：之前的工作大多关注如何在有限 context 中塞入更多记忆，A-Mem 指出了**记忆的组织结构**可能比记忆的绝对数量更重要。
- **Link + Evolution 是值得追随的研究方向**：这条路线被顶会认可后，预计会涌现大量 follow-up 工作。
- **多模态记忆**：当前 A-Mem 只处理文本，后续可扩展到图像、视频等多模态记忆的 Zettelkasten 组织。

### 7.3 对工程实践的启示

- **生产级实现可用**：A-mem-sys 提供了可直接集成的生产级代码，值得尝试接入实际 Agent 项目
- **可插拔设计**：A-Mem 的记忆系统可作为独立模块叠加到现有 Agent 框架（如 LangChain、AutoGen）上
- **性能权衡**：每条记忆增加 2-3 次 LLM 调用的 overhead，在高并发场景需要评估成本

---

## 八、总结

A-MEM 是 NeurIPS 2025 提出的**革命性 Agent 记忆架构**，通过将 Zettelkasten 方法论引入 LLM Agent，实现了：

> **"记忆不仅存储经验，还能从经验中生长出新的理解。"**

核心创新在于**动态链接生成**和**记忆自演化**两大机制，使记忆网络从静态仓库进化为动态自组织知识网络。在 6 种基础模型、6 种评测指标上均显著超越 SOTA 基线，消融实验验证了各模块的独立有效性。

A-MEM 代表了 **Agent Memory 领域从"存储"到"认知"的关键一跃**，对未来的长期记忆研究方向具有重要的参考价值。
