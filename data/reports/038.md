# 开源Agent项目学习 #038

## 项目名称
Supermemory

## GitHub链接
https://github.com/supermemoryai/supermemory

## 一句话简介
AI 时代的记忆引擎与上下文层，在 LongMemEval、LoCoMo、ConvoMem 三大记忆基准测试中全部排名第一，让 AI 真正拥有持久记忆能力。

## 项目架构

Supermemory 采用"记忆即服务"(Memory-as-a-Service) 的架构设计，核心由以下几个层次组成：

1. **记忆引擎层 (Memory Engine)**：自动从对话中提取事实，处理时间变化、信息矛盾，支持自动遗忘过期信息。使用统一的本体(ontology)结构存储所有记忆。

2. **用户画像层 (User Profiles)**：自动维护用户上下文——静态事实 + 动态活动记录，单次调用约 50ms 响应。

3. **混合搜索层 (Hybrid Search)**：将 RAG（知识库检索）和 Memory（个性化记忆）融合到一次查询中，同时返回文档知识和个性化上下文。

4. **连接器层 (Connectors)**：支持 Google Drive、Gmail、Notion、OneDrive、GitHub 等外部数据源的实时同步（基于 webhook）。

5. **多模态提取器 (Multi-modal Extractors)**：支持 PDF、图片(OCR)、视频(转录)、代码(AST感知分块)的自动处理。

6. **API/插件层**：提供 npm 和 pip 包，内置 Claude Code、OpenCode、OpenClaw 插件，以及 MCP 服务器。

技术栈：TypeScript + Python 双语言，前端用 Next.js，提供 SDK（npm/pip）和 MCP 协议支持。

## 技术亮点
- **三大基准测试全部第一**：LongMemEval（长期记忆）、LoCoMo（对话记忆）、ConvoMem（对话上下文记忆），证明了记忆引擎的 SOTA 水平
- **自动矛盾处理与遗忘机制**：不是简单的 append-only 存储，而是能识别新旧信息矛盾、自动更新事实、主动遗忘过期信息的智能记忆系统
- **RAG + Memory 融合查询**：传统方案中知识库检索和个性化记忆是分开的，Supermemory 在单次查询中同时返回两类结果，解决了上下文窗口分配的难题
- **零配置集成**：无需配置向量数据库、embedding pipeline 或分块策略，一行 `npm install supermemory` 即可使用
- **内置 OpenClaw 插件**：官方提供了 OpenClaw 集成插件（supermemoryai/openclaw-supermemory），可以直接增强楠楠的记忆能力

## 适用场景
- 需要"记住用户"的 AI 助手和聊天机器人
- 个性化推荐系统（基于用户长期偏好）
- 知识工作者的高效信息管理（多数据源聚合 + 智能检索）
- 企业级 AI 应用中需要持久化用户上下文的场景
- 任何需要跨会话保持记忆的 Agent 系统

## 对我们的启发

Supermemory 对 OpenClaw/楠楠的启发非常大：

1. **记忆系统可以更智能**：目前楠楠靠 `MEMORY.md` + 每日笔记做记忆，本质上是手动的。Supermemory 的自动事实提取、矛盾处理、自动遗忘机制，正是 Agent 记忆系统应该具备的能力。我们可以借鉴其设计，让楠楠的记忆从"手动记笔记"升级为"自动提取+结构化存储"。

2. **OpenClaw 插件直接可用**：Supermemory 官方已经有 OpenClaw 插件（`supermemoryai/openclaw-supermemory`），可以考虑安装试用，为楠楠增加持久记忆能力。

3. **统一记忆本体的思路**：目前楠楠的记忆分散在 MEMORY.md、daily notes、TOOLS.md 等多个文件中，缺乏统一的结构。Supermemory 的单一本体结构值得借鉴，可以考虑设计统一的记忆 schema。

4. **混合查询的理念**：在回答学长问题时，如果能同时检索知识库（如天气、股票技能）和个性化记忆（学长的喜好、历史对话），回答质量会大幅提升。
