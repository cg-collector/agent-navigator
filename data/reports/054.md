# 开源Agent项目学习 #054

## 项目名称
Kimi Code CLI

## GitHub链接
https://github.com/MoonshotAI/kimi-cli

## 一句话简介
月之暗面（Moonshot AI）出品的终端 AI 编程 Agent，支持代码读写、Shell 执行、网页搜索，可通过 ACP 协议集成到 IDE。

## 项目架构

- **CLI 交互层**: `kimi` 命令启动交互式终端，支持自然语言+Shell混合操作
- **Web UI 层**: `kimi web` 在本地浏览器打开图形界面，支持会话管理、文件引用、代码高亮
- **ACP 集成层**: `kimi acp` 以服务模式运行，通过 Agent Client Protocol (agentclientprotocol.com) 集成到 IDE 和其他 Agent 客户端
- **工具链**: 代码读写、Shell 命令执行、网页搜索和抓取
- **模型层**: 默认使用 Kimi（月之暗面）模型，也支持配置其他平台 API
- **配置层**: AGENTS.md 项目配置文件，`/init` 自动生成；支持 `/login` OAuth 授权

技术栈：Python 3.12-3.14，通过 uv 包管理器安装分发

## 技术亮点

1. **ACP 协议原生支持**: 和 OpenClaw 支持的 ACP (Agent Client Protocol) 是同一个协议，可以直接集成到支持 ACP 的 IDE
2. **三种运行模式**: CLI / Web / ACP Service，覆盖终端党、浏览器党、IDE 集成党
3. **AGENTS.md 自动生成**: `/init` 命令分析项目自动生成配置文件，和 OpenClaw 的 AGENTS.md 理念完全一致
4. **OAuth 授权流程**: 不用手动管理 API Key，浏览器一键授权
5. **国产模型生态**: 默认 Kimi 模型，中文场景优化，对国内开发者友好

## 对楠楠/学长的启发

- **ACP 协议是行业趋势**: Kimi CLI、OpenClaw 都支持 ACP，未来 Agent 互联互通是方向
- **多模式运行值得学习**: CLI + Web + Service 三模式设计，覆盖不同使用场景
- **AGENTS.md 已经是行业标准**: 从 Claude → OpenClaw → Kimi CLI，项目级配置文件已成共识
- **国产 Agent 工具崛起**: 月之暗面加入 CLI Agent 竞争，说明市场在快速升温
