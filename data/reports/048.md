# 开源Agent项目学习 #048

## 项目名称
mcp-agent

## GitHub链接
https://github.com/lastmile-ai/mcp-agent

## 一句话简介
基于 MCP（Model Context Protocol）构建 Agent 的轻量级框架，用简单可组合的模式实现高鲁棒性的 Agent 应用。

## 项目架构
mcp-agent 的核心设计哲学是 **"MCP is all you need"**，架构围绕三个层次展开：

1. **MCP 基础层**：完整实现 MCP 协议（Tools、Resources、Prompts、Notifications），负责 MCP Server 的生命周期管理（连接、断开、重连）
2. **Agent 模式层**：实现了 Anthropic《Building Effective Agents》中描述的所有经典模式，包括：
   - **Augmented LLM**：将任意 LLM（OpenAI、Anthropic 等）连接到 MCP Server
   - **Orchestrator**：多 Agent 编排协调
   - **Router**：根据请求路由到不同 Agent
   - **Map-Reduce**：并行处理 + 结果聚合
   - **Evaluator-Optimizer**：自我评估与迭代优化
3. **持久化运行时**：基于 Temporal 工作流引擎，支持 Agent 的暂停、恢复和故障恢复，无需修改 API 代码

Python 编写，支持 FastMCP 兼容 API，Agent 本身也可作为 MCP Server 暴露。

## 技术亮点
- **MCP-Native 设计**：不自己造工具系统，直接用 MCP 协议连接工具，天然享受 MCP 生态的丰富工具集
- **模式可组合**：Agent 模式是独立模块，可以像搭积木一样自由组合（如 Router + Map-Reduce + Evaluator）
- **Temporal 持久化**：长时间运行的 Agent 工作流通过 Temporal 保障可靠性，支持断点续跑和状态恢复
- **Agent as MCP Server**：Agent 可以将自身能力暴露为 MCP Server，形成 Agent 的嵌套组合
- **简洁的 API 设计**：`async with` 上下文管理器风格，代码量极少即可构建功能完整的 Agent

## 适用场景
- 需要快速接入大量 MCP 工具的 Agent 应用
- 多步骤、长时间运行的工作流（需要暂停/恢复能力）
- 多 Agent 协作场景（编排器模式）
- 将现有 Agent 服务化为 MCP Server 供其他系统调用

## 对我们的启发
1. **MCP 生态是趋势**：mcp-agent 证明了围绕 MCP 协议构建工具层的可行性。OpenClaw 已经在支持 MCP，这条路是对的。
2. **简单模式 > 复杂架构**：Anthropic 的那篇文章说"简单模式组合起来比复杂框架更鲁棒"，这个理念跟我们日常看到的过度工程化的 Agent 框架形成鲜明对比。楠楠的 AGENTS.md 里的"速度原则"也是这个思路——简单任务不要绕弯子。
3. **Agent as Server** 的模式很值得借鉴——Agent 不只是被调用的客户端，还可以把自身能力暴露出去。这跟 OpenClaw 的 subagent/heartbeat 机制有异曲同工之妙。
4. **Temporal 做持久化**是个好思路，对于需要长时间运行的任务（比如楠楠帮学长做大项目），断点续跑能力能大幅提升体验。
