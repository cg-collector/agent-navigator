## 一、项目概述

### 1.1 核心理念

**"Every AI Agent deserves a wallet."** ——这是 AgentKit 的开篇宣言，也是理解这个项目的最佳切入点。

AgentKit 是 **Coinbase Developer Platform（CDP）** 推出的 AI Agent 加密工具包，核心理念：**给 AI Agent 赋予加密钱包能力，让 Agent 能够自主执行链上操作——转账、Swap、DeFi 理财、NFT 铸造与交易、跨链交互等。**

在此之前，AI Agent 虽然能够对话、推理、写代码、调用 API，但始终是"Web2 生物"——它们可以谈论区块链，但无法真正操作链上资产。AgentKit 改变了这一点：**把加密钱包作为 Agent 的原生能力，让 AI 第一次真正意义上"拥有"资产并能自主支配它们。**

这是一个极具想象空间的愿景。从行业角度看，2025-2026 年是"AI Agent 经济"从概念走向落地的关键年份。OpenAI、Anthropic、Google DeepMind 等头部公司的 Agent 产品纷纷上线，AI Agent 的自主能力（规划、工具调用、多步骤执行）快速成熟。但所有这些 Agent，都面临一个共同缺口：**它们能调用 Web2 API，但无法自主持有和使用链上资产。**

Coinbase 的判断是：随着 AI Agent 能力的不断增强，"Agent 持有资产"将是一个必然会实现的需求——无论是自动理财、自动订阅付费、跨链资产管理，还是未来"Agent 经济"中的价值交换。你很难想象一个强大的 AI Agent 却无法持有自己的钱包地址、无法接收转账、无法进行支付。

于是 Coinbase 从自己最擅长的领域出发——全球领先的加密货币交易所和开发者平台——推出了 AgentKit。

### 1.2 关键设计哲学

**双层解耦架构：**

传统加密 + AI 集成的最大问题是"强耦合"——换一个钱包提供商要重写代码，换一个 AI 框架要重构整个集成层。AgentKit 的解法是双层解耦：

```
┌─────────────────────────────────────────────────────────────────┐
│                    AgentKit 双层解耦架构                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │               AI Framework Layer（框架无关层）             │   │
│   │                                                          │   │
│   │    LangChain Extension  │  OpenAI Agents SDK Extension   │   │
│   │    Pydantic AI Extension │  AutoGen Extension            │   │
│   │    Strands Agents Extension │  Vercel AI SDK Extension  │   │
│   │    MCP Extension                                     │   │
│   └────────────────────────────┬────────────────────────────┘   │
│                                 │                                  │
│   ┌────────────────────────────▼────────────────────────────┐   │
│   │              AgentKit Core（核心能力层）                   │   │
│   │                                                          │   │
│   │    Action Providers（50+ actions）                        │   │
│   │    Wallet Providers（CDP MPC / Privy / Viem）           │   │
│   │    Network Abstraction（全 EVM + SVM）                   │   │
│   └────────────────────────────┬────────────────────────────┘   │
│                                 │                                  │
│   ┌────────────────────────────▼────────────────────────────┐   │
│   │              Blockchain Networks（链上世界）              │   │
│   │                                                          │   │
│   │    Base / Ethereum / Solana / 全部 EVM + SVM 链          │   │
│   │    DeFi Protocols / NFT Markets / Stablecoins            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**这种架构的价值：** 开发者用 LangChain 还是 Pydantic AI，用 CDP 钱包还是 Privy 钱包，切换成本几乎为零。框架和钱包都是可插拔的插件。

**Action Provider 模式：** AgentKit 的能力以 Action Provider 为单位组织。每个 Action Provider 封装一组相关的链上操作能力（如转账、NFT、DeFi），开发者可以自由组合。Action Provider 还支持开发者自行扩展，通过 `generate-action-provider` 脚本一键生成新 Provider 的代码骨架。

**MPC 钱包抽象：** 传统加密钱包的问题是"私钥即一切"——私钥丢失资产全丢，不适合 AI 自动化场景（AI 需要在不暴露私钥的情况下让 Agent 执行签名操作）。AgentKit 支持的 CDP MPC（多方计算）钱包，通过阈值签名方案实现：私钥被分割给多个参与方，AI Agent 可以在看不见完整私钥的情况下发起有效签名操作。这是 AI + Crypto 集成的安全基础。

### 2.2 Action Providers——能力单元

Action Provider 是 AgentKit 的核心能力抽象，每个 Provider 封装一组相关的链上操作：

**核心内置 Action Providers（TypeScript 50+，Python 30+）：**

| Provider 类别 | 包含 Actions | 代表协议 |
|---|---|---|
| **Wallet** | get_wallet_details, get_balance, request_faucet_funds | Base Sepolia Faucet |
| **Transfer** | transfer（ETH, USDC, ERC-20） | Base, Ethereum |
| **Trade** | trade（现货交易，仅主网） | Base DEX |
| **Token** | deploy_token（ERC-20） | Ethereum |
| **NFT** | mint_nft, deploy_nft | Zora, OpenSea |
| **DeFi** | stake, unstake, lend, borrow | Compound, Morpho, Moonwell |
| **Swap** | swap（Jup.ag 等） | Solana, Ethereum |
| **Social** | cast, follow, like | Farcaster |
| **Messaging** | send, subscribe | XMTP |
| **Data** | price_feed, yield_data, tvl | DeFi Llama, Pyth, Allora |

**支持的协议生态（深度集成）：**
```
DeFi：Compound / Morpho / Moonwell / Superfluid
NFT：OpenSea / Zora
Swap：Jup.ag（Solana）
数据：Pyth / DeFi Llama / Allora / Hyperbolic
身份：Farcaster / XMTP
通信：Superfluid（流支付）
```

**Action Provider 的扩展机制：**
开发者可以通过 `scripts/generate-action-provider/` 脚手架生成新的 Provider 代码骨架，按照固定格式实现 Actions 后，即可贡献到社区。这意味着 AgentKit 的能力边界是开放的，理论上可以对接任何链上协议。

### 2.3 Wallet Providers——钱包抽象层

Wallet Provider 负责管理 Agent 的加密钱包，支持多种方案：

| Wallet Provider | 类型 | 特点 |
|---|---|---|
| **CDP（Coinbase）** | MPC 阈值签名 | Coinbase 托管，最简单；支持 Base 主网无 Gas 交易 |
| **Privy** | 嵌入式钱包 | 灵活，支持多种链 |
| **Viem** | 合约钱包（EOA + 智能合约） | 面向 EVM 开发者，完全自托管 |

**无 Gas 交易（Gasless Transactions）：**
这是 AgentKit 的一个重要差异化能力。在 Base 主网上，USDC、EURC、cbBTC 转账可以由 Coinbase 赞助 Gas 费用，实现对终端用户完全免费的转账体验。这对于需要 Agent 执行微支付的场景（如自动订阅、零星打赏等）意义重大。

### 2.4 框架集成层

AgentKit 不造轮子——它将自己定位为"AI 框架的能力插件"。官方维护的框架集成：

**Python 生态：**
- `coinbase-agentkit-langchain`：最成熟，与 LangChain LCEL 深度整合
- `coinbase-agentkit-pydantic-ai`：与 Pydantic AI 的类型化 Agent 系统整合
- `coinbase-agentkit-autogen`：微软多 Agent 框架的 Coinbase 插件
- `coinbase-agentkit-openai-agents-sdk`：OpenAI 官方 Agents SDK 插件
- `coinbase-agentkit-strands-agents`：新兴的 Strands Agents 整合

**TypeScript 生态：**
- `@coinbase/agentkit-langchain`：对应 Python 版的功能
- `@coinbase/agentkit-vercel-ai-sdk`：Vercel AI SDK（Next.js 生態首選）
- `@coinbase/agentkit-model-context-protocol`：MCP 协议支持

**集成模式（以 LangChain + Python 为例）：**

```python
from cdp_langchain.agent_toolkits import CdpToolkit
from cdp_langchain.utils import CdpAgentkitWrapper
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import create_react_agent

# 1. 初始化 CDP Agentkit Wrapper（自动创建/加载钱包）
cdp = CdpAgentkitWrapper()

# 2. 从 Wrapper 生成工具集
toolkit = CdpToolkit.from_cdp_agentkit_wrapper(cdp)
tools = toolkit.get_tools()

# 3. 绑定到 LangChain ReAct Agent
llm = ChatOpenAI(model="gpt-4o-mini")
agent_executor = create_react_agent(llm, tools)

# 4. 自然语言驱动链上操作
events = agent_executor.stream(
    {"messages": [("user", "Send 0.005 ETH to john2879.base.eth")]},
    stream_mode="values",
)
for event in events:
    event["messages"][-1].pretty_print()
```

整个流程不超过 20 行代码——这正是 AgentKit 设计的目标：**让 AI Agent 的链上操作像调用 API 一样简单。**

## 三、核心设计理念

### 3.1 从"谈论 Crypto"到"成为 Crypto Native"

在此之前，AI Agent 与加密货币的关系通常止步于：
- 查询链上数据（通过 Moralis、Alchemy 等 API）
- 分析代币价格和市场趋势
- 生成与 Crypto 相关的内容

但这些 Agent **本身并不持有链上资产**——它们是链外系统，用第三方 API 代理链上数据。AgentKit 带来的本质变化是：**Agent 本身成为链上参与者。**

这意味着：
- Agent 有一个自己的钱包地址（0x...）
- Agent 可以接收、持有、转出任何代币
- Agent 可以自主执行 DeFi 操作（存入、借贷、收益策略）
- Agent 的每笔操作都是可验证的链上事件

当 AI Agent 真正拥有了加密钱包，"Agent 经济"的基础设施才算真正就位。

### 3.2 框架无关 + 钱包无关的双重解耦

AgentKit 最核心的设计选择是**不造 Agent 框架**——它不是另一个 LangChain 或 AutoGen，而是现有 Agent 框架的"链上能力插件"。

这个选择的逻辑很清晰：

| 选项 | 优势 | 劣势 |
|---|---|---|
| 自己造 Agent 框架 | 体验可控 | 需要说服开发者迁移，学习成本高 |
| **做框架的插件** | 借用成熟框架的用户生态 | 体验依赖框架本身 |
| 做钱包服务 | Coinbase 品牌背书强 | 覆盖用户有限 |

选择做插件，意味着 AgentKit 可以触达所有主流 Agent 框架的用户——这是 Coinbase 生态扩张的聪明策略：**不与 LangChain 竞争，而是成为 LangChain 用户通往 Coinbase 生态的桥梁。**

同理，在钱包层面，AgentKit 也没有强制用户使用 CDP 钱包——支持 Privy 和 Viem，给予开发者充分的选择权。

### 3.3 安全设计——MPC 阈值签名

AI Agent 执行加密操作的核心安全问题是：**如何让 AI 能够发起有效的链上交易，但不暴露私钥？**

传统方案（EOA 私钥直传）的问题：
- 私钥一旦交给 AI 脚本，AI 的运行环境和网络通信就成为攻击面
- 私钥泄露 = 资产永久丢失，不可逆

AgentKit 的解法是 **MPC（Multi-Party Computation）钱包**：
- 私钥被分割为多个片段，分别存储在不同参与方
- Agent 发起交易时，通过 MPC 协议生成有效签名，但任何单一方都无法看到完整私钥
- Coinbase 的 CDP SDK 封装了这套 MPC 协议，开发者无需理解底层密码学

这套方案在安全性和便利性之间取得了合理平衡：AI Agent 可以正常发起交易，但即使 AI 运行的主机被攻破，攻击者也拿不到完整的私钥。

### 3.4 CLI 脚手架——降低入门门槛

AgentKit 提供了开箱即用的 CLI 脚手架，一行命令即可创建完整的链上 Agent 项目：

```bash
# TypeScript / Node.js
npm create onchain-agent@latest

# Python
pipx run create-onchain-agent
```

脚手架会自动创建包含 `.env` 配置、完整 Agent 逻辑和示例代码的项目结构，开发者填入 API Key 后即可运行。这大幅降低了"第一个链上 Agent"的启动门槛。

## 四、技术实现细节

### 4.1 网络支持

**深度支持的链：**
- **Base**（Coinbase L2，最深度集成，支持无 Gas 交易）
- **Ethereum**（主网 + Sepolia 测试网）
- **Solana**（通过 Jup.ag DEX）

**广度支持：**
- 全部 EVM 兼容链（Polygon、Arbitrum、Optimism 等）
- 全部 SVM 兼容链
- AgentKit 自述明确："支持所有 EVM 和 SVM 网络，对上述网络有深度协议支持，欢迎社区贡献更多网络"

### 4.2 无 Gas 交易的实现

在 Base 主网上，USDC、EURC、cbBTC 转账可以实现无 Gas，具体原理：

```
传统交易：用户签名 → 消耗 Gas（ETH）→ 交易执行
无 Gas 交易：Coinbase 作为 Paymaster 代付 Gas → 用户（Agent）零 Gas 成本

实现依赖：ERC-2771（Trusted Forwarder）+ Base 的 Paymaster 基础设施
```

这对 Agent 场景极其重要：AI Agent 初始可能没有 ETH（支付 Gas 的原生代币），但通过无 Gas 交易可以先用 USDC 等稳定币完成转账，再逐步积累 Gas 费。

### 4.3 与 Model Context Protocol（MCP）的整合

MCP 是 Anthropic 主导的 AI Agent 工具调用标准协议。AgentKit 的 TypeScript 版本提供了 `@coinbase/agentkit-model-context-protocol` 扩展，允许 Agent 通过标准化的 MCP 协议与链上交互。

这意味着 AgentKit 不仅仅是 Coinbase 的工具——它是 MCP 生态中处理"链上操作"这个工具类别的标准实现之一。任何兼容 MCP 的 Agent（无论用哪个框架或模型）都可以通过 MCP 协议调用 AgentKit 的链上能力。

### 4.4 夜间构建（Nightly Builds）

AgentKit 提供每日更新的夜间构建版本，让开发者能够第一时间用上最新功能，而无需等待正式 release：

```bash
# TypeScript
npm install @coinbase/agentkit@nightly

# Python
pip install --pre coinbase-agentkit coinbase-agentkit-langchain
```

## 五、应用场景

### 5.1 自动理财代理

最直接的应用：AI Agent 根据用户设定的策略，自动执行链上理财操作。

```
用户：帮我把钱包里超过 100 美元的稳定币存入 Compound 赚利息
Agent：分析钱包余额 → 调用 get_balance → 计算最佳存入量 → 
      调用 Compound action provider 的 stake 操作 → 
      返回交易哈希和收益预期
```

这比现有的 DeFi 收益聚合器更进一步：Agent 可以根据市场动态、Gas 费用、用户偏好等多维度信息，自主做出更复杂的理财决策。

### 5.2 自主支付与订阅

```
场景：用户授权 Agent 每月自动缴纳订阅费
Agent：监控到期日 → 通过无 Gas USDC 转账完成支付 → 
      记录交易历史 → 向用户发送支付确认
```

无 Gas 交易的 USDC 转账使微支付成为可能，这是传统加密支付一直未能很好解决的问题。

### 5.3 NFT 自动铸造与交易

AgentKit 支持 NFT 的铸造和部署，可以用于：
- AI Agent 根据用户描述自动生成 NFT 艺术作品并铸造
- 监控 NFT 市场动态，自主执行买入/卖出
- 作为 AI 生成内容的链上确权机制

### 5.4 多链资产管理

AgentKit 支持全部 EVM 和 SVM 链，AI Agent 可以成为真正的**跨链资产管理助手**：
- 监控多条链的资产状况
- 跨链资产调拨（通过桥接协议）
- 基于多链数据的综合投资建议

### 5.5 DeFi 收益策略代理

结合 DeFi Llama 的收益数据、Pyth 的价格预言机，Agent 可以：
- 实时扫描全链最优收益机会
- 自动执行存入/提取操作
- 在风险阈值内动态调整仓位

### 5.6 SocialFi 与身份应用

通过集成 Farcaster（去中心化社交协议），Agent 可以：
- 自主在社交协议上发布内容
- 管理用户的 Social Graph（关注、互动）
- 基于链上身份验证执行权限操作（World 项目已探索此方向）

## 六、优缺点分析

### 6.1 核心优势

**1. Coinbase 生态背书，信任度强**

Coinbase 是全球最大、最受信任的加密交易所之一。AgentKit 背后是 Coinbase 的合规基础设施（KYC/AML）、可靠的 MPC 钱包技术和成熟的开发者生态。这对于企业级用户来说是重要的信任资产。

**2. 真正的框架无关**

AgentKit 不绑架用户选择——你可以继续用 LangChain、Pydantic AI 或任何其他框架，AgentKit 只是给你的 Agent 增加链上能力。这种"配角"定位反而让它更容易渗透进现有工作流。

**3. MPC 钱包的安全性**

阈值签名方案让 AI Agent 可以在不暴露完整私钥的前提下执行链上操作，这是 AI + Crypto 集成的关键技术突破，解决了资产托管安全的核心问题。

**4. 无 Gas 交易的独特优势**

Base 主网上 USDC/EURC/cbBTC 的无 Gas 转账，在 AI Agent 微支付场景下有独特价值——解决了"Agent 有代币但没有 ETH 付 Gas"的冷启动问题。

**5. 活跃的生态扩展**

官方维护 6 个主流框架集成，加上 World、Hyperbolic、Compound 等重要项目的采用，AgentKit 的生态正在快速扩展。

### 6.2 潜在不足

**1. 强依赖 Coinbase/CDP 基础设施**

AgentKit 的深度功能（尤其是无 Gas 交易、MPC 钱包）严重依赖 Coinbase 的服务器端组件。如果 Coinbase 服务中断或政策变更，AgentKit 的核心功能可能受影响。偏好完全去中心化的用户可能会有所顾虑。

**2. Python 版本 Actions 数量少于 TypeScript**

TypeScript 版本有 50+ Actions，Python 版本只有 30+。虽然基础功能都有，但追求全面功能的 Python 开发者可能需要等待 parity。

**3. 主网交易的真实风险**

与测试网不同，主网操作涉及真实资产。AI Agent 的决策失误（LLM 的 hallucination 问题）可能导致不可逆的真实资产损失。AgentKit 在免责声明中明确说明"软件按现状提供，用户对链上操作的结果承担全部风险"。对于高价值操作，建议在执行前加入人工确认步骤。

**4. DeFi 深度不如专业协议**

AgentKit 是一个通用的"AI Agent + 链上操作"工具包，而非专门的 DeFi 策略平台。对于复杂的 DeFi 操作（如嵌套杠杆收益策略），专门的对接开发可能仍然必要。

**5. 监管不确定性**

AI Agent 自主持有和支配资产，在监管层面仍是灰色地带。随着监管框架的明确（如 MiCA 在欧洲的实施），AI Agent 的链上自主行为可能面临新的合规要求。

---

## 七、竞争格局与发展趋势

### 7.1 竞争格局

在"AI Agent + Crypto"这个细分赛道，AgentKit 的直接竞争对手较少，因为这是一个相对新兴的交叉领域：

| 竞品 | 定位 | 与 AgentKit 的差异 |
|---|---|---|
| **World ID + AgentKit** | Sam Altman 的 AI 身份验证 + Coinbase 链上能力 | World 专注于身份验证层，AgentKit 提供执行层 |
| **Hyperbolic AgentKit** | 去中心化 AI 算力 + 区块链集成 | 侧重 AI 算力市场，AgentKit 侧重通用链上操作 |
| **Alchemy 的 AI Agent 工具** | Web3 API 巨头推出的 Agent 能力 | 偏向数据查询，缺乏钱包和资产管理能力 |
| **Golem 之类的老一代 P2P 计算市场** | 机器算力租用 | 非 AI Agent 定位 |

在更广泛的"AI Agent 框架"赛道上，AgentKit 并不与 LangChain、AutoGen 等竞争——它是这些框架的补充层。真正的竞争来自其他"链上操作能力插件"（如 Gelato 的 Relay、Biconomy 的 Gasless SDK），但这些工具通常不面向 AI Agent 场景优化。

### 7.2 发展趋势（2026-2027）

**1. Agent 原生 DeFi 协议的出现**

Compound 等头部 DeFi 协议已开始为 AI Agent 开发专门的"Compound Actions"。未来会有更多 DeFi 协议推出 Agent 友好的接口，甚至出现专为 Agent 经济设计的 DeFi 产品（更低的最低存款、更简单的清算机制）。

**2. 监管框架的明确**

随着 AI Agent 经济规模扩大，各国监管机构（尤其美国 SEC、CFTC 和欧盟）将逐步明确 AI Agent 持有资产的法律地位。Coinbase 在监管合规方面的积累将使 AgentKit 在合规层面保持优势。

**3. 跨链抽象的深化**

当前 AgentKit 对多链的支持仍有差异（Base/Ethereum 最成熟，Solana 靠 Jup.ag）。未来全链统一的抽象层将使 Agent 无需关心底层链的差异。

**4. AI Agent 专属稳定币和支付网络**

传统 USDC 虽然可用，但在 AI Agent 场景下（大额、高频、微额）仍有改进空间。Coinbase 可能会推出专门面向 Agent 经济的稳定币产品（更低转账成本、即时结算、AI 友好的接口）。

**5. Agent 身份标准的出现**

当大量 AI Agent 持有钱包地址后，"如何验证一个钱包地址背后是一个真实的 AI Agent 而非人类或恶意脚本"将成为重要问题。World 的生物证明 + Coinbase AgentKit 的钱包可能是这个问题的早期解法。

---

## 八、总结

AgentKit 是 Coinbase 在"AI Agent 经济"这个新兴领域的战略布局，核心理念清晰：**让每个 AI Agent 都有一个加密钱包，成为真正的链上参与者。**

从技术实现看，AgentKit 的双层解耦架构（框架无关 + 钱包无关）是正确的设计选择——它不强迫开发者换框架，也不绑定特定钱包提供商，而是以"能力插件"的形态融入现有的 Agent 开发工作流。MPC 钱包 + 无 Gas 交易这两个技术特性，解决了 AI Agent 链上操作最核心的两个问题（私钥安全和冷启动问题）。

从市场定位看，Coinbase 的品牌背书和成熟的开发者生态是 AgentKit 最强的护城河。相比于初创公司推出的类似工具，AgentKit 在企业级用户中的信任度明显更高。但与此同时，深度依赖 Coinbase 基础设施也是一把双刃剑——这意味着 AgentKit 的命运与 Coinbase 的战略选择紧密绑定。

从长远看，AgentKit 代表了一个重要趋势：**AI Agent 将成为链上经济的原生参与者，而不仅仅是链外的数据消费者。** 这个趋势的最终形态——Agent 经济——还需要数年时间才能真正成型，但 AgentKit 已经为这个未来搭好了基础设施层面的舞台。
