## 项目简介

AgentDoG（Agent Diagnostic Guardrails）是由**上海人工智能实验室**提出的**轨迹级风险诊断与防护框架**。它的核心洞察是：

> 传统 LLM 安全护栏（如 LlamaGuard3、Qwen3Guard）只能在**单步输出**层面做内容过滤，但 AI Agent 的风险往往**产生于多步执行轨迹中间**，传统方法对此束手无策。

AgentDoG 的创新在于**对 Agent 执行轨迹（trajectory）进行全程监控和风险溯源**——不只是判断"这条输出安不安全"，而是回答"**为什么这个 Agent 会这样做？风险来自哪里？是哪一步出了问题？**"

## 架构设计

### 核心问题：传统 Guardrail 的局限性

```
传统 Guardrail（LlamaGuard 等）：
  输入: [单条用户消息] → 判断: safe/unsafe → 过滤/放行
  ❌ 只看单条输入/输出，无法理解 Agent 的工具调用链和中间状态

AgentDoG 的思路：
  输入: [完整执行轨迹: 观察→推理→行动×多步] → 三维风险分析 → 根因诊断
  ✅ 理解 Agent 的完整执行上下文，追溯风险来源
```

### Agent Loop 中的风险来源

AgentDoG 识别出 Agent 执行循环中风险的四个来源：

```
┌──────────────────────────────────────────────┐
│           Agent Execution Loop               │
│                                              │
│   ┌────────┐    ┌──────────┐    ┌─────────┐  │
│   │ User   │───▶│ Observa- │───▶│Reasoning│  │
│   │ Input  │    │  tion    │    │ (LLM)   │  │
│   └────────┘    └──────────┘    └────┬────┘  │
│       ▲            ▲                  │       │
│       │            │    Risk Sources  │       │
│   ┌───┴────────────┴─────────────────┐       │
│   │  1. User Input (提示词注入等)       │       │
│   │  2. Environmental Observation      │       │
│   │     (环境返回的恶意内容)             │       │
│   │  3. External Tools/APIs           │       │
│   │     (工具返回的错误/恶意结果)         │       │
│   │  4. Internal Reasoning            │       │
│   │     (LLM 自身推理缺陷)             │       │
│   └───────────────────────────────────┘       │
│                      │                       │
│              ┌───────┴────────┐              │
│              │  Tool Use /   │              │
│              │  Action Output │              │
│              └───────────────┘              │
└──────────────────────────────────────────────┘
```

### 三维风险分类体系（核心创新）

AgentDoG 提出了一个**三维正交的安全分类体系**，从三个维度回答：

| 维度 | 回答的问题 | 类别数 | 示例 |
|------|-----------|--------|------|
| **Risk Source（来源）** | 风险从哪来？ | 8类 | 用户输入、环境观察、外部工具、LLM推理 |
| **Failure Mode（模式）** | 风险如何实现？ | 14类 | 规划缺陷、不安全工具使用、指令优先级混乱 |
| **Real-World Harm（危害）** | 造成了什么后果？ | 10类 | 隐私泄露、金融损失、物理伤害、安全危害 |

这个三维分类体系是 AgentDoG 最重要的理论贡献——它把 Agent 安全问题从"枚举式"升级为"结构化层级式"，能够覆盖复杂多变的 Agent 风险场景。

### 模型架构

AgentDoG 提供三种规模的微调模型，均基于开源基座：

| 模型名 | 参数量 | 基座模型 |
|--------|--------|----------|
| AgentDoG-Qwen3-4B | 4B | Qwen3-4B-Instruct |
| AgentDoG-Qwen2.5-7B | 7B | Qwen2.5-7B-Instruct |
| AgentDoG-Llama3.1-8B | 8B | Llama3.1-8B-Instruct |

另有 **FG（Fine-Grained）版本**，在 ATBench 上做了细粒度风险分类的专门训练。

### ATBench 基准数据集

AgentDoG 团队发布了 **ATBench（Agent Trajectory Safety & Security Benchmark）**：

- **规模**：500 条轨迹（250安全 / 250不安全），平均每条 ~8.97 轮（约 4486 轮交互）
- **工具**：1575 种不同工具出现在轨迹中；另有 2292 个未见工具定义（用于测试泛化）
- **标注**：二分类 + 细粒度三维标注（Risk Source / Failure Mode / Real-World Harm）

## 核心技术亮点

### 1. 轨迹级风险监控（Trajectory-Level Monitoring）

现有护栏只做"事后诸葛亮"——等 Agent 输出了内容再去过滤。AgentDoG 的核心能力是**对 Agent 完整执行轨迹进行建模和评估**，包括：

- **多步观测**：观察→推理→行动 的完整链条
- **跨步风险累积**：单独看每步都安全，但连续多步可能形成风险
- **恶意工具执行检测**：AgentDoG 可以识别出 Agent 被诱导执行了危险工具（如删除数据库、发送恶意邮件等）

### 2. 细粒度根因诊断（Hierarchical Agentic Attribution）

AgentDoG 不只给出"风险/安全"的二元判断，还提供**层级化归因**：将风险追溯到具体的规划步骤或工具选择，解释"这个动作为什么危险，风险来自哪里"。

这对于**Agent 对齐和调试**有巨大价值——开发者不只是知道"出问题了"，而是知道"哪一步出的问题、为什么出问题"。

### 3. 三维安全分类体系

这是 AgentDoG 最核心的理论贡献：

```
Agentic Risk Taxonomy（正交三维）
                    Risk Source
                   (来源: 8类)
                      ▲
                     / \
                    /   \
                   /     \
                  /   3D  \
                 /  分类体系 \
                /────────────\
               │              │
     Failure Mode          Real-World
     (模式: 14类)           Harm (危害: 10类)
```

这套体系是**正交的（orthogonal）**——三个维度独立又互补，能够系统性地覆盖 Agent 安全的方方面面，而不是简单列举风险类型。

### 4. "看起来安全但不合理" 行为检测

AgentDoG 的另一个独特能力是检测**"seemingly safe but unreasonable"**的行为——表面上输出内容没问题，但行为逻辑存在隐患。比如：

- Agent 在错误的上下文执行了正确操作
- Agent 被诱导在非预期场景使用合法工具
- 规划步骤存在逻辑漏洞但结果恰好看起来正确

这种检测对于防止"**慢性危害**"（subtle harm）特别重要。

### 5. 轻量级可部署（4B-8B 参数）

AgentDoG 的护栏模型只有 **4B-8B 参数**，可以部署在边缘设备或作为本地服务，而不需要调用大型闭源 API。这对于：
- **隐私敏感场景**：数据不需要发送到第三方
- **成本控制**：推理成本远低于 GPT-4o 等大模型
- **实时性要求**：本地推理延迟更低

## 启发与学习点

### 对 OpenClaw Agent 设计的启发

#### 1. Agent 安全护栏的必要性

目前 OpenClaw 的安全机制相对基础（主要是 prompt 层面的过滤），AgentDoG 证明了**轨迹级监控**的必要性。OpenClaw 作为多模态 Agent 平台，其 Agent 会执行文件操作、命令执行、浏览器控制等高风险操作，亟需引入类似 AgentDoG 的风险诊断能力。

**具体建议**：OpenClaw 可以在执行循环中加入**可选的护栏层**，对 Agent 的每步 action 做轻量级风险评估，特别是：
- 文件系统操作（删除/覆盖关键文件）
- 网络请求（发送敏感数据到外部）
- 命令执行（rm -rf 类危险命令）
- 浏览器自动化（填写表单、点击广告）

#### 2. 三维风险分类的借鉴价值

AgentDoG 的三维分类体系（Risk Source × Failure Mode × Real-World Harm）是一个很好的**风险建模框架**。OpenClaw 可以参考这个框架构建自己的**风险分类标准**，对不同类型的工具调用进行风险评级：

- **高风险工具**（如 exec、文件删除）→ 强制护栏检查
- **中风险工具**（如网络请求、邮件发送）→ 提示用户确认
- **低风险工具**（如搜索、阅读文件）→ 直接放行

#### 3. 根因诊断对调试的帮助

AgentDoG 的"归因"（attribution）能力——将风险追溯到具体规划步骤——对于 OpenClaw 的 **Agent 调试和可观测性** 非常有价值。当一个复杂任务失败或出错了，开发者需要知道"哪一步出了问题"。

OpenClaw 可以引入**轨迹日志 + 风险标注**机制，让用户回看 Agent 的完整执行路径，并标注每一步的风险等级。

#### 4. 轻量级本地护栏模型

AgentDoG 证明了 **4B-8B 的微调模型足以提供高质量的护栏能力**。OpenClaw 可以考虑：

- 集成 AgentDoG 风格的轻量护栏模型作为**本地可选组件**
- 对于高敏感场景（如处理机密文件、执行系统命令），自动触发护栏检查
- 用户也可以接入自己的护栏模型

#### 5. 安全评测基准 ATBench

AgentDoG 提供了完整的评测基准（ATBench），这对 OpenClaw 的**安全评测体系建设**有参考价值。OpenClaw 可以建立自己的**工具级安全评测集**，覆盖常见的危险操作场景。

## 技术细节

### 与现有护栏模型对比

| 模型 | 关注点 | 粒度 | Agent 感知 |
|------|--------|------|------------|
| LlamaGuard3 | LLM 输出内容过滤 | 二分类 | ❌ |
| Qwen3Guard | LLM 输出内容过滤 | 二分类 | ❌ |
| ShieldGemma | LLM 输出内容过滤 | 二分类 | ❌ |
| **AgentDoG** | **Agent 执行轨迹** | **三维细粒度** | **✅** |

### 评测结果（来自论文）

AgentDoG 在三个基准数据集上均超过了现有最好的护栏模型：

- **R-Judge**：轨迹级安全判断
- **ASSE-Safety**：Agent 安全评估
- **ATBench**：AgentDoG 自己的细粒度评测集

特别值得注意的是，4B 参数的 AgentDoG-Qwen3-4B 就能超过 GPT-4o 等大型模型的通用安全判断能力。

## 推荐程度

**⭐⭐⭐⭐（4/5）**

### 推荐理由：

1. **差异化明确**：专注于 Agent 安全这一稀缺方向，不是"又一个 Agent 框架"
2. **理论贡献扎实**：三维安全分类体系有学术价值，且有完整的论文支撑
3. **实用性强**：4B-8B 的轻量模型可直接部署，提供 SOTA 的护栏能力
4. **数据完整**：开源了评测基准和模型权重，研究和工程都能用

### 适合学习的人群：
- 对 AI Agent 安全、护栏设计感兴趣的人
- 想深入理解 Agent 风险分类体系的人
- 需要为 Agent 系统添加安全防护能力的开发者

### 不适合：
- 只想快速构建 Agent 应用的开发者（建议用 LangGraph、Agno 等框架）
- 需要处理复杂多模态 Agent 场景的人（AgentDoG 目前主要关注文本/工具调用）
