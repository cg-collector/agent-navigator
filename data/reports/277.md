## 2. 项目简介

OS-Copilot 是由上海人工智能实验室、华东师范大学、普林斯顿大学和香港大学联合开发的开源框架，旨在构建能够与操作系统全面交互的通用型计算机智能体。不同于大多数局限于单一领域（如特定网站或软件）的 Agent 方案，OS-Copilot 的设计目标是让 Agent 具备跨域操作能力——涵盖网页浏览、代码终端、文件系统、多媒体处理以及各类第三方应用程序的控制。

基于该框架实现的 FRIDAY（Full-featured Resident Intelligent Digital Assistant for You）是其核心演示 Agent，最大的技术特色是**自我改进（Self-Improvement）**和**自指导学习（Self-Directed Learning）**能力。FRIDAY 不依赖预定义的完整工具集，而是在执行任务过程中动态生成工具、评估工具质量并持久化高价值工具到长期记忆中，从而实现"越用越强"的进化效果。在 GAIA 通用 AI 助手基准测试中，FRIDAY 以 35% 的性能优势超越此前所有方法，证明了其跨应用泛化能力。

该项目于 2024 年 2 月发布论文，同年 3 月被 ICLR 2024 LLM Agents Workshop 接收，目前仍在活跃开发中——已支持视觉感知（friday_vision）、前端交互界面，路线图包含 Windows 支持、多轮对话、开源模型适配等方向。对于 GUI Agent 和 Computer Use 研究者而言，OS-Copilot 提供了一个从架构设计到自学习机制的完整参考实现。

## 3. 架构深度剖析

### 3.1 核心架构

OS-Copilot 采用经典的 **Planner → Configurator → Actor** 三阶段流水线架构，同时引入了受认知科学启发的记忆系统设计：

```
用户请求 → Planner（规划器）→ Configurator（配置器）→ Actor（执行器）→ 操作系统
                                    ↑                              ↓
                              长期记忆系统 ←←← Critic（评价器）反馈循环
```

**Planner（规划器）：** 负责理解用户自然语言请求，将复杂任务分解为可执行的子任务序列。关键设计在于 Planner 必须具备对自身能力的元认知（Meta-cognition）——它需要检索 Agent 可用的工具集和系统信息，以确保生成的计划粒度合理、可执行。这避免了过度分解（效率低）或粒度过粗（无法执行）的问题。

**Configurator（配置器）：** 这是 OS-Copilot 最具创新性的模块，其设计灵感来自人脑的三种记忆类型：
- **工作记忆（Working Memory）：** 当前子任务的上下文、中间状态
- **陈述性记忆（Declarative Memory）：** 用户画像、操作系统版本、工作目录等静态知识
- **程序性记忆（Procedural Memory）：** 已积累的可复用工具库（核心！）

Configuration Tracker 通过稠密检索（Dense Retrieval）从长期记忆中召回相关信息构建 prompt。当检索不到合适工具时，激活 Tool Generator 动态生成针对当前应用的定制化工具。

**Actor（执行器）：** 分为两个子阶段：
1. **Executor（执行器）：** 基于配置 prompt 生成可执行动作（如 bash 命令、AppleScript 脚本），并在真实 OS 环境中执行
2. **Critic + Refiner（评价与精炼）：** Critic 评估执行结果是否达成目标；若失败则 Refiner 收集反馈进行自我纠正，最多迭代 3 次

### 3.2 关键模块

**Tool Generator（工具生成器）：** 当现有工具库无法覆盖当前需求时，FRIDAY 会调用 LLM 生成新的 Python 类形式的工具。例如面对"切换系统暗色模式"的子任务，它会生成一个利用 AppleScript 调用 macOS 系统 API 的 Python 工具类。这种"按需生成"机制使 Agent 无需预装大量工具即可应对新场景。

**Self-Directed Learning（自指导学习）：** 给定学习目标（如"掌握电子表格操作"），FRIDAY 自主生成由易到难的任务课程表（Curriculum），通过试错方式逐个解决，过程中积累的工具和语义知识自动沉淀到程序性记忆中。在 SheetCopilot-20 数据集上的实验表明，经过自学习后 FRIDAY 在电子表格控制任务上的 Pass@1 率显著提升。

**Critic 评分机制：** 每次成功执行后，Critic 用 LLM 对生成的工具打分（0-10 分）。得分 ≥8 的工具被视为高质量，自动更新到长期工具库供未来复用。这一机制保证了工具库的质量而非数量，避免垃圾工具污染记忆空间。

## 4. 技术亮点

### 亮点一：受认知科学启发的三层记忆架构

OS-Copilot 将人脑的记忆系统映射到 Agent 设计中：工作记忆处理实时上下文，陈述性记忆存储静态知识，程序性记忆存储可执行技能。这种设计不是简单的工程类比——它解决了 GUI Agent 领域的一个核心矛盾：**预定义工具集无法覆盖长尾应用 vs. 每次从头生成工具效率低下**。通过程序性记忆的累积效应，Agent 在使用过程中逐步建立起个人化的"技能树"，越用越熟练。这与人类学习新软件的过程高度一致：第一次用时笨手笨脚地摸索，之后越来越顺畅。

### 亮点二：工具质量驱动的自改进闭环

多数 Agent 框架的工具管理是静态的（开发者预设或一次性生成），而 OS-Copilot 引入了动态工具生命周期：**生成 → 执行 → 评分 → 存储/丢弃**。Critic 评分机制确保只有真正有效的工具才能进入长期记忆，形成正反馈循环。实验数据显示，在 GAIA 基准上 FRIDAY 比此前最佳方法提升 35%，作者归因于跨任务积累的技能迁移——在某个任务中学到的工具可能在完全不同的任务中被复用。这种"技能复用带来泛化能力提升"的发现，对整个 Agent 领域都有启发意义。

### 亮点三：自指导学习实现零样本应用掌控

FRIDAY 的自指导学习使其能在极少人工监督下掌握全新应用的操作方法。以 PowerPoint 为例：未学习前，FRIDAY 无法正确控制字体、字号、图片位置等；经过一轮自指导学习后，它自主积累了文本框颜色修改、行距调整、图片尺寸定位等工具；进一步学习后能完成完整的 PPT 制作任务。这意味着部署 OS-Copilot 后，用户只需指定学习目标，Agent 就能通过自主探索掌握新软件——大幅降低了 GUI Agent 的定制成本。

## 5. 基准测试表现

| 基准 | 指标 | FRIDAY 表现 | 对比基线 |
|------|------|------------|---------|
| **GAIA**（通用AI助手，466题） | 总体准确率 | 显著优于基线 | 超越前 SOTA **35%** |
| **SheetCopilot-20**（电子表格控制20任务） | Pass@1 | 经过自学习后显著提升 | 未学习时表现较差 |
| **PowerPoint 定性测试** | 幻灯片制作质量 | 学习后可完成复杂排版 | 学习前无法控制字体/图片 |

需要客观指出的是，OS-Copilot 目前主要在 macOS 环境下验证，GAIA 基准虽然覆盖面广但偏重问答类任务而非纯 GUI 操作。此外，论文发表至今（2024年2月）GUI Agent 领域发展迅速，OSWorld、WebVoyager 等更专业的 GUI 基准陆续出现，FRIDAY 在这些新基准上的表现尚无公开数据。

## 6. 与同类项目横向对比

| 维度 | OS-Copilot / FRIDAY | OpenHands | UI-TARS | Agent S (gui-agents) |
|------|---------------------|-----------|---------|---------------------|
| **核心定位** | OS级通用Agent + 自改进 | 软件开发Agent | GUI理解+操作统一模型 | 多平台GUI Agent |
| **自学习能力** | ✅ 核心特色（工具积累） | ❌ 无 | ❌ 无 | ⚠️ 有限 |
| **记忆系统** | 三层认知架构（工作/陈述/程序性） | 简单历史窗口 | 无显式记忆 | 轨迹记忆 |
| **操作范围** | Web+终端+文件+第三方App | 代码编辑器+终端 | 任意GUI界面 | 移动+桌面+Web |
| **输入模态** | 文本为主，新增Vision | 文本 | 截图（视觉为主） | 截图+文本 |
| **跨平台支持** | 主要macOS | Linux容器 | 平台无关 | Android+iOS+桌面+Web |
| **开源程度** | 完全开源 | 完全开源 | 模型/数据集开源 | 完全开源 |
| **成熟度** | 学术原型，活跃开发 | 生产可用 | 学术领先 | 学术原型 |

**关键差异总结：** OS-Copilot 的独特价值在于"自改进"这一维度——其他框架大多聚焦于单次任务的执行能力，而 OS-Copilot 关注 Agent 如何在使用过程中变得越来越强。这在生产环境中极具吸引力：一个会越用越好用的 Agent，比一个每次都从零开始的 Agent 有本质区别。但在纯 GUI 理解精度和多平台覆盖方面，UI-TARS 和 Agent S 更具优势。

## 7. 对我们的启发

1. **工具即记忆，记忆即能力：** OpenClaw 的 Skill 系统可以借鉴 OS-Copilot 的程序性记忆思路——当楠楠通过试错完成了某个新类型的任务（比如首次操作某个飞书多维表格），应该将成功的方法论沉淀为可复用的 Skill 或模板，而不是下次再从头摸索。
2. **自指导学习的落地路径：** 可以给楠楠设计"探索性任务"模式——学长指定一个想让她掌握的能力域（比如"学会用 Feishu Bitable API 做数据分析"），楠楠自主生成练习任务并通过试错学习，最终产出一份可复用的操作指南。
3. **Critic 评分机制的质量控制思想：** 楠楠的自我改进流程（self-improving skill）可以引入类似机制——每次修正错误后，让独立评审判断这次修正是否真的有效，有效的方法才写入长期记忆。
4. **Configurator 的上下文组装艺术：** OS-Copilot 在执行每个子任务前都会精心组装 prompt（工具信息+用户画像+系统状态+历史经验），这种"充分上下文化"的策略值得在我们的 Agent 设计中强化——特别是在调用复杂工具前，确保 prompt 包含了所有必要的环境信息。

## 8. 综合评分：7.5/10

**评分理由：**
- **(+)** 架构设计有深度，三层记忆系统兼具理论优雅和实用价值
- **(+)** 自改进和自指导学习是真正的差异化创新，不是工程堆砌
- **(+)** 论文质量高，实验设计扎实（GAIA + SheetCopilot + PPT 定性）
- **(+)** 完全开源，代码结构清晰，文档完善
- **(-)** 主要支持 macOS，Windows/Linux 支持不完善
- **(-)** 单轮对话限制，不支持多轮交互式任务
- **(-)** 缺乏在 OSWorld/WebVoyager 等 GUI 专业基准上的评测数据
- **(-)** 项目更新节奏偏慢，2024年后社区活跃度有所下降
- **(-)** 工具生成依赖闭源 LLM（OpenAI），开源模型适配仍在路线图阶段

**一句话总结：** OS-Copilot 是 GUI Agent 领域少数具备"进化能力"的框架，其自改进机制的设计思想和实验验证都具有标杆意义，尽管工程成熟度和平台覆盖还有差距，但其核心理念——**让 Agent 在使用中变强**——代表了 Computer Use Agent 的重要发展方向。
