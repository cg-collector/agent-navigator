# 学习报告 #064 — Strands Agents SDK

## 项目名称
Strands Agents（AWS 开源 AI Agent SDK）

## GitHub 链接
https://github.com/strands-agents/sdk-python

## 一句话简介
AWS 开源的模型驱动（model-driven）AI Agent SDK，用最少的代码构建从简单对话到复杂自主工作流的生产级 Agent。

## 核心架构

**技术栈：** Python + TypeScript 双语言 SDK，支持 Amazon Bedrock、Anthropic、OpenAI 等多家模型提供商。

**设计模式：**
- **模型驱动循环（Model-Driven Loop）**：不硬编码工作流，而是让模型自主规划、调用工具、反思迭代。Agent Loop = 推理 → 工具调用 → 反思 → 继续或结束
- **Prompt + Tools = Agent**：只需定义一个系统提示和一组工具函数即可构建 Agent，极其简洁
- **可插拔组件**：工具选择策略、上下文管理、会话状态存储、记忆系统均可自定义
- **多 Agent 协作**：支持 multi-agent 编排模式

**已在 AWS 内部生产验证：** Amazon Q Developer、AWS Glue、VPC Reachability Analyzer 都在用。

## 技术亮点

1. **极简 API 设计**：几行代码就能跑一个完整 Agent，上手门槛极低
2. **模型驱动 vs 硬编码工作流**：与 LangGraph 等需要显式定义 DAG 的框架不同，Strands 把编排权交给 LLM，利用模型的推理能力自动规划下一步
3. **生产就绪**：内置可观测性（observability）、session 管理、错误恢复，不是玩具框架
4. **双语言支持**：Python 和 TypeScript SDK 并行维护，前后端都能用
5. **Provider 无关**：不绑定 AWS Bedrock，可以自由切换模型提供商

## 对楠楠/学长的启发

- **设计哲学值得学习**：把复杂度从开发者转移到模型——OpenClaw 的 heartbeat/cron 设计其实也有类似的"让 AI 自己决定做什么"的思路
- **工具即能力**：Strands 的核心理念是"模型 + 工具 = Agent"，和 OpenClaw 的 skill 系统高度一致
- **多 Agent 协作**：Strands 的 multi-agent 模式可以参考，用于楠楠/桐桐/其他角色协作的场景
- **生产级思维**：从 Amazon Q 的实战中提炼的设计，比学术项目更接地气

## 评分
⭐⭐⭐⭐ （4/5）

扣一星是因为目前生态还比较新（2025年5月才发布），社区规模和第三方工具集成还不如 LangChain/CrewAI 成熟。但设计理念非常先进，值得持续关注。
