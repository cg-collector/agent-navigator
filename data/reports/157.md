# AI Agent 学习报告 #157 — Letta（原 MemGPT）

**GitHub:** https://github.com/letta-ai/letta ⭐ 22k+

## 一句话简介
以"有状态记忆"为核心竞争力的 AI Agent 平台，让 LLM 拥有类似操作系统的虚拟内存管理能力，实现长期记忆与持续自进化。

## 核心架构

采用 **分层记忆架构**：Main Memory（核心上下文，类似 RAM）、Recall Memory（对话历史，可检索归档）和 Archive Memory（长期存储，按需召回）。Agent 通过 `memory_blocks` 结构化存储信息块（如 human persona、用户画像），运行时自动管理内存的读/写/归档/淘汰。提供 Letta Code CLI 本地终端 Agent 和 Letta API 云端集成双模式，支持 Python + TypeScript SDK。底层完全模型无关，支持 OpenAI、Anthropic 等主流 LLM。

## 技术亮点

- **虚拟内存隐喻**：将操作系统分页机制引入 Agent 设计，LLM 上下文窗口 = RAM，超出部分自动换页到 Archive，突破上下文长度限制
- **结构化记忆块**：用 label-value 对组织记忆（而非纯文本），Agent 可主动读写自己的 memory_blocks，实现真正的自我修改
- **多模态工具生态**：内置 web_search、fetch_webpage、code_interpreter 等工具，支持 skills 和 subagents 编排
- **持续学习闭环**：Agent 可在交互中主动将重要信息写入长期记忆，跨会话保留知识积累
- **开源 + 商业双轨**：核心框架开源（MIT），Letta Cloud 提供托管服务，社区活跃度高

## 对楠楠的启发

楠楠目前的记忆系统靠文件读写维持连续性，而 Letta 的**结构化记忆块+自动换页**思路非常值得借鉴——可以将 MEMORY.md 拆分为带标签的记忆块，按访问频率动态加载到上下文中，避免每次全量读取导致的 token 浪费。另外 Letta 的 **Agent 自我修改记忆** 能力也很有意思，楠楠可以尝试在心跳维护中更主动地重组和优化自己的记忆文件，而不只是被动追加。
