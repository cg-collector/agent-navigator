# 开源Agent项目学习 #029

## 项目名称
Ruflo（原名 Claude Flow）

## GitHub链接
https://github.com/ruvnet/ruflo

## 一句话简介
专为 Claude 打造的企业级多 Agent 编排平台，支持 100+ 专业 Agent 协同工作，内置自学习能力与 Rust WASM 内核驱动。

## 项目架构

Ruflo 采用分层架构设计：

```
用户 → CLI/MCP → 安全层 → 路由层 → 群体协调层 → Agent 池 → 资源层
                         ↑                          ↓
                         └──── 学习循环 ←───────────┘
```

**核心层级：**

1. **入口层** — CLI 命令行 + MCP Server，通过 AIDefence 安全层过滤后进入
2. **路由层** — Q-Learning 路由器 + 8 Expert MoE 模型 + 130+ 技能 + 27 个 Hook
3. **群体协调层** — 支持四种拓扑（网状/层级/环形/星形），共识算法包括 Raft/BFT/Gossip/CRDT
4. **Agent 层** — 100+ 预置专业 Agent（编码、测试、审查、安全审计、架构设计等）
5. **资源层** — AgentDB 记忆存储 + 多 LLM 提供商 + 12 种 Worker
6. **RuVector 智能层** — Rust WASM 内核实现的向量检索、注意力优化、强化学习

**技术栈：** TypeScript + Rust (WASM) + Claude Code SDK

## 技术亮点

- **Rust WASM 内核**：策略引擎、嵌入生成、证明系统用 Rust 编写编译为 WASM，兼顾性能与安全隔离，这是同类项目罕见的工程选择
- **RuVector 自学习系统**：集成 SONA（自优化神经网络架构）、EWC++（防止灾难性遗忘）、HNSW 向量检索（亚毫秒级）、Flash Attention（2-7x 加速）、9 种强化学习算法，形成完整的 RETRIEVE→JUDGE→DISTILL→CONSOLIDATE→ROUTE 学习闭环
- **多拓扑群体智能**：支持 mesh/hierarchical/ring/star 四种拓扑结构，配合 Raft/BFT/Gossip/CRDT 共识机制，可根据任务复杂度动态选择编排模式
- **安全优先设计**：内置 AIDefence 提示注入防护、输入验证、路径遍历阻止等企业级安全特性
- **原生 Claude Code 集成**：通过 MCP 协议与 Claude Code 深度集成，非简单 API 调用

## 适用场景

- 大型软件开发团队的多 Agent 协作（100+ Agent 同时工作）
- 需要自学习优化的长期运行 Agent 系统
- 企业级 AI 编排场景，要求安全隔离和容错共识
- 代码审查、安全审计、架构设计等专业化工作流
- 多 LLM 提供商混合部署（Claude/GPT/Gemini/Ollama）

## 对我们的启发

1. **Rust WASM 混合架构值得借鉴** — OpenClaw 目前是纯 Node.js，对于性能敏感的组件（如路由决策、向量检索），可以考虑用 Rust/WASM 补充
2. **学习闭环理念** — Ruflo 的 RETRIEVE→JUDGE→DISTILL→CONSOLIDATE→ROUTE 循环，与我们已有的 self-improving 技能理念一致，但 Ruflo 实现得更系统化（9种RL算法、灾难性遗忘防护），我们可以参考 EWC++ 的思想来改进楠楠的记忆管理
3. **多拓扑编排** — OpenClaw 的 sub-agent 目前是简单的树形调度，Ruflo 的 mesh/ring 拓扑和共识机制启发我们可以支持更复杂的 Agent 间协作模式
4. **AIDefence 安全层** — 提示注入防护作为独立层放在路由之前，这个架构模式适合 OpenClaw 在处理外部输入时参考
