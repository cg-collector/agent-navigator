## 项目简介

Agno 是一个用于构建、运行和管理规模化 Agentic 软件的开源 Python 框架。它由三大核心层组成：

1. **Framework（框架层）**：构建具有记忆、知识库、护栏和 100+ 集成的 Agent、团队和工作流
2. **Runtime（运行时）**：基于 FastAPI 的无状态、会话级后端，将 Agent 系统作为可扩展的生产服务运行
3. **Control Plane（控制平面）**：通过 AgentOS UI 进行测试、监控和管理生产环境中的 Agent 系统

Agno 的核心理念是将流式执行和长时运行作为一等公民对待，原生支持人类审批工作流、运行时护栏、审计日志和全链路追踪。项目使用 Apache 2.0 许可证，活跃维护中。

> **历史背景**：Agno 前身为 Phidata（`phidatahq/phidata`），2025 年底更名为 Agno 并迁移至 `agno-agi/agno` 组织，定位从"AI 助手框架"升级为"Agentic 软件运行时"。

### 三大核心抽象

**1. Agent（智能体）**
```python
agent = Agent(
    name="my-agent",
    model=Claude(id="claude-sonnet-4-6"),
    tools=[MCPTools(url="...")],
    db=SqliteDb(db_file="agent.db"),
    add_history_to_context=True,
    num_history_runs=3,
)
```
- 支持流式响应、工具调用、记忆持久化
- 可配置审批策略（自动批准 / 人工审批 / 分级授权）

**2. Team（团队）**
```python
team = Team(
    model=Claude(...),
    members=[researcher_agent, writer_agent],
    add_history_to_context=True,
)
```
- 多 Agent 协作模式
- 共享协调模型负责任务分配和结果整合

**3. Workflow（工作流）**
- 结构化的多步骤流水线
- 每个 step 是一个 Agent，顺序或条件执行

## 技术亮点

### 1. "Framework + Runtime + Control Plane" 三位一体架构

大多数 Agent 框架只解决"怎么构建 Agent"的问题，而 Agno 同时回答了三个问题：
- **怎么构建** → Framework 层的 Agent/Team/Workflow 抽象
- **怎么运行** → Runtime 层的无状态 FastAPI 服务 + 会话隔离
- **怎么管理** → Control Plane 层的 AgentOS UI + 追踪 + 审计

这种分层设计使得从原型到生产的路径极短——同一份代码既是开发时的交互式 Agent，又是部署后的生产级 API。对比如 LangGraph 只偏重编排、OpenAI Agents SDK 只偏轻量封装，Agno 的全栈思维更完整。

### 2. 生产级的信任与治理机制

Agno 将"信任"内建到引擎层面，而非事后补丁：

- **Guardrails（护栏）**：在 Agent 执行循环中内嵌安全检查，不是简单的输出过滤
- **Approval Workflows（审批流）**：细粒度控制哪些操作需要人工批准（如发邮件、删除数据），支持分级权限
- **Human-in-the-Loop**：Agent 可以暂停等待审批，恢复后继续执行
- **Audit Logs（审计日志）**：所有决策和操作全程记录
- **Evaluations（评估）**：将评估集成到 Agent 循环中，持续验证输出质量

这套机制对企业场景至关重要，也是区别于大多数"玩具级"Agent 框架的关键差异。

### 3. MCP 原生集成 + 100+ 工具生态

Agno 对 MCP（Model Context Protocol）是一等公民支持：

```python
from agno.tools.mcp import MCPTools
tools = [MCPTools(url="https://docs.agno.com/mcp")]
```

这意味着任何 MCP 服务器都可以直接作为 Agent 工具使用。加上内置的 100+ 工具（编程、搜索、文件操作、数据库、API 调用等），形成了丰富的工具生态。这种"协议优先"的设计让 Agno 能自然融入 MCP 生态系统，而不是造自己的工具标准。

## 对楠楠的启发

### 1. 从"构建"到"运维"的思维升级

我们目前的 Agent 系统更多关注"怎么让 Agent 更聪明"，但 Agno 提醒我们：**生产环境的 Agent 需要 Framework + Runtime + Ops 的完整闭环**。特别是：
- 会话隔离机制（per-user, per-session）— 多用户场景必备
- 审计日志 — 出问题时能回溯
- 控制平面 UI — 非开发者也能管理和测试 Agent

这些对我们未来将 Agent 产品化很有参考价值。

### 2. 护栏即代码（Guardrails as Code）的思想

Agno 将安全检查嵌入执行循环而非事后过滤的做法值得学习。我们在做 Agent 时也可以考虑：
- 在工具调用前加一层意图分类
- 对高风险操作设置审批门控
- 记录完整的决策链路用于复盘

### 3. MCP 协议优先的工具集成策略

Agno 用 MCP 统一所有工具接入的方式很优雅。我们在设计 Agent 工具体系时，也应该优先考虑兼容 MCP 标准，这样可以：
- 复用整个 MCP 生态的工具服务器
- 降低自定义工具的开发成本
- 与主流 Agent 框架保持互操作性
