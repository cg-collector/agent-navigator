# 开源Agent项目学习 #040

## 项目名称
Mastra

## GitHub链接
https://github.com/mastra-ai/mastra

## 一句话简介
由 Gatsby 团队打造的 TypeScript AI Agent 框架，集 Agent、Workflow、RAG、Memory、MCP、Evals、可观测性于一体，专为 TypeScript 生态设计。

## 项目架构

Mastra 采用模块化 monorepo 架构，核心包包括：

- **@mastra/core** — 核心运行时，提供 Agent、Tool、Workflow 抽象
- **@mastra/memory** — 多层记忆系统（对话历史、工作记忆、语义回忆）
- **@mastra/rag** — RAG 管道，支持从 API/数据库/文件检索上下文
- **@mastra/evals** — 内置评估框架（模型评分、规则评分、统计方法）
- **@mastra/telemetry** — 可观测性层，追踪 agent 调用和 token 用量
- **ee/** — 企业特性（认证、审计等），独立许可证

架构层次：
```
用户应用 (React/Next.js/Node)
    ↓
Mastra Runtime (Agents + Workflows)
    ↓
Model Routing (40+ LLM 提供商统一接口)
    ↓
Storage Layer (执行状态持久化)
    ↓
Tool/MCP Layer (工具 + MCP Server)
```

Workflow 引擎采用图式设计，支持 `.then()`、`.branch()`、`.parallel()` 等直观的控制流语法，同时支持 human-in-the-loop（暂停/恢复）。

## 技术亮点

- **TypeScript 原生设计**：不像大多数框架用 Python 写再用 TS 封装，Mastra 从零用 TS 构建，类型安全贯穿始终，与 Next.js/React 生态无缝集成
- **Model Routing 统一接口**：一个标准接口接入 40+ LLM 提供商（OpenAI、Anthropic、Gemini 等），切换模型无需改动业务代码
- **多层记忆系统**：区分对话历史（conversation history）、工作记忆（working memory）和语义回忆（semantic recall），让 Agent 在不同时间尺度上保持上下文连贯性
- **图式 Workflow 引擎**：用 `.then().branch().parallel()` 的链式语法编排复杂流程，比 LangGraph 的状态机更直观，同时支持暂停/恢复的 human-in-the-loop
- **内置 Evals + 可观测性**：框架自带评估和追踪能力，不需要额外接入第三方工具就能在生产环境持续优化 Agent
- **MCP Server 原生支持**：既能作为 MCP Client 调用工具，也能将自身 Agent/Tool 暴露为 MCP Server，与其他 Agent 生态互通

## 适用场景

- 需要 TypeScript 全栈 AI 应用的团队（尤其 Next.js 项目）
- 需要明确控制流编排的 multi-step AI 流程（客服机器人、数据处理管道）
- 需要 human-in-the-loop 的审批/协作场景
- 需要从原型快速迭代到生产级部署的 AI 产品
- 需要内置 eval 和可观测性的生产环境

## 对我们的启发

1. **Workflow 的链式语法值得借鉴**：Mastra 的 `.then().branch().parallel()` 比状态机更易读。OpenClaw 的 subagent pipeline 如果能用类似语法编排会直观很多。

2. **多层记忆是方向**：Mastra 把记忆分为短期对话、工作记忆、语义回忆三层，这种分层思路和楠楠的 MEMORY.md + 日志文件 + 心跳整理的机制本质一致，但更系统化。

3. **Evals 内置于框架**：Mastra 把评估作为一等公民而非附加组件，这说明成熟的 Agent 框架必须内置质量保障机制。OpenClaw 可以考虑加入类似的自动 eval 流程。

4. **TypeScript 生态的优势**：对于有前端的项目，TypeScript Agent 框架可以前后端共享类型定义，减少开发成本。"Python trains, TypeScript ships" 这句话很有道理——快速原型用 Python，生产部署用 TypeScript。

5. **MCP 双向支持**：既能消费 MCP 工具，也能作为 MCP Server 暴露能力。这种双向互通是 Agent 互操作的未来方向。
