## 一、项目概述

Agent S 是一个开源的**计算机使用智能体框架**（Computer-Use Agent Framework），其核心理念是构建能够像人类一样操作计算机的自主智能体，即**Agent-Computer Interface（ACI）**——一种将 GUI 操作抽象为智能体可理解接口的新范式。

该项目的演进路径极具说服力：

| 版本 | 时间 | OSWorld 准确率 | 亮点 |
|------|------|----------------|------|
| Agent S1 | 2024.10 | 20.6% | 起点，奠定基础架构 |
| Agent S2 | 2025.03 | 48.8% | 引入 Manager-Worker 层级架构 |
| Agent S2.5 | 2025.08 | ~54% | 简化设计，性能优化 |
| Agent S3 | 2025.10 | 62.6%（单智能体）| 移除层级，引入原生编码智能体 |
| Agent S3 + bBoN | 2025.12 | **72.6%** | 超越人类水平（72%） |

从 20.6% 到 72.6%，仅用一年零两个月便超越了人类基准水平，堪称 2025 年 AI Agent 领域最激动人心的技术突破之一。

## 二、架构概述

### 2.1 整体架构

Agent S3 采用**双模型协同架构**，将推理能力与视觉 grounding 能力解耦：

```
┌─────────────────────────────────────────────────┐
│                  Agent S3                        │
│                                                  │
│  ┌──────────────┐        ┌──────────────────┐  │
│  │  主推理模型    │        │  Grounding 模型   │  │
│  │ (Reasoning   │◄──────►│  (如 UI-TARS     │  │
│  │   Model)     │  屏幕截图 │   1.5-7B)        │  │
│  │  GPT-5 /     │        │                  │  │
│  │  Claude 等   │        │  输出精确坐标      │  │
│  └──────┬───────┘        └────────┬─────────┘  │
│         │                          │            │
│         ▼                          ▼            │
│  ┌──────────────────────────────────────────┐   │
│  │           行为决策层                      │   │
│  │  • 决定操作类型（点击/输入/滚动/调用代码）   │   │
│  │  • 生成行为叙事（Behavior Narrative）     │   │
│  │  • 支持本地代码执行环境（Python/Bash）      │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 2.2 双模型分工

| 模型角色 | 职责 | 技术要求 |
|---------|------|---------|
| **主推理模型** | 决策规划、任务分解、GUI 指令生成 | 强大的推理能力（GPT-5、Claude 3.7 等） |
| **Grounding 模型** | 将 GUI 指令映射为屏幕精确坐标 | 视觉理解 + 定位能力（UI-TARS-1.5-7B） |

这种解耦设计的核心洞察是：GUI 操作的核心难点在于**"知道做什么"**和**"知道点哪里"**是两个不同性质的问题，应该由不同专长的模型分别处理。

### 2.3 操作原语（Action Primitives）

Agent S3 定义了一套丰富的操作原语，智能体可以选择最适合当前任务的行动：

- `click(x, y)` — 精确坐标点击
- `type(text)` — 文本输入
- `scroll(direction)` — 滚动页面
- `hotkey(keys)` — 组合键操作
- `call_code_agent()` — 调用本地编码环境执行 Python/Bash 代码

本地编码环境的引入意义重大：它让智能体不再局限于 GUI 操作，当任务可以用代码更快完成时（数据处理、文件操作、系统配置），智能体会主动选择编程方式——这解决了纯 GUI 智能体的根本局限。

### 2.4 Agent S2 → S3 的架构演进

S2 采用了层级化的 Manager-Worker 设计：Manager 负责任务规划，Worker 负责具体执行。但 S3 发现这种设计**带来了不必要的开销**，于是彻底改为扁平架构：

```
S2: Manager ──► Worker(s)     [层级，有协调开销]
S3: 单原生编码智能体           [扁平，更高效]
```

这次简化带来了 **13% 的性能提升和 52% 的 LLM 调用减少**，是一个很好的"less is more"案例。

## 三、技术亮点

### 3.1 Behavior Best-of-N（bBoN）—— 多 rollout  scaling

这是 S3 最核心的创新。核心洞察是：**计算机使用任务的长时域特性导致单次执行高方差**——一个小错误会级联放大，最终导致完全失败。

bBoN 的解决思路：

```
1. 并行运行 N 个独立 rollout（每个都有随机温度/Top-P采样）
2. 对每个 rollout 生成"事实"（Facts）—— 将冗长的步骤日志压缩为关键事实陈述
3. 将 Facts 拼接为"行为叙事"（Behavior Narrative）—— 可读、可比较的行为轨迹描述
4. 由 Judge 模型对比各行为叙事，选择最佳的一次执行作为最终输出
```

技术细节：
- **Fact 生成**：不是记录每个像素级操作，而是提取"在任务成功层面重要的信息"，大幅压缩信息量
- **Judge 选择**：基于行为叙事而非原始输出来评判，降低了上下文长度要求，也提高了评判质量
- **对齐验证**：论文中 judge 与人类偏好对齐度达 **92.8%**，说明行为叙事作为评判介质的有效性

效果：bBoN 将 S3 单次执行准确率从 62.6% 提升至 69.9%（10 runs with GPT-5）。

### 3.2 超越人类基准

在 OSWorld（业界最权威的计算机使用智能体评测基准）上：

- Agent S3 + bBoN：**72.6%** 准确率
- 人类基准：**72.36%**

这是首次有 AI 智能体在真实计算机操作任务上超越人类平均水平。

跨平台泛化能力同样令人印象深刻：
- WindowsAgentArena：50.2% → 56.6%（bBoN）
- AndroidWorld：68.1% → 71.6%（bBoN）

### 3.3 零样本泛化（Zero-Shot Generalization）

bBoN 的另一个关键发现是：跨多个 rollouts 的 scaling 对未见过的任务同样有效。这意味着**增加 rollouts 数量可以直接换取性能提升**，而不需要针对每个任务微调。

### 3.4 支持多平台

- **Linux**、**macOS**、**Windows** 全平台支持
- 评测覆盖 OSWorld（Linux）、WindowsAgentArena（Windows）、AndroidWorld（Android）

| OpenAI CUA/Operator | 闭源，端到端 GUI 操作 | Agent S3 是开源替代，性能更强 |

## 五、启发与思考

### 1. 任务级 Scaling vs 模型级 Scaling

过去行业普遍认为更强的模型 = 更好的智能体。但 S3 证明了**对同一任务的多次独立执行，通过行为级评判选择最优解**同样能带来巨大提升。这是一种全新的 scaling 维度——**behavioral scaling**，独立于 model scaling 而存在。

这给我们的启发是：当我们受限于模型能力时，可以通过增加执行多样性 + 智能选择来逼近上限。

### 2. 架构简化有时比增加复杂度更有效

从 S2 的 Manager-Worker 层级架构到 S3 的扁平原生编码智能体，**删掉协调层反而提升了性能**。这提醒我们：在构建 Agent 系统时，不要假设层级分解总是有益的——有时扁平 + 能力强的单一智能体 + 代码执行能力反而更高效。

### 3. Fact/Behavior Narrative 的信息压缩价值

S3 中 fact 生成的过程实际上是一个**有损但任务相关的压缩**：将所有步骤压缩为"对任务成功有贡献的关键事实"。这种思路可以借鉴到 Agent 的记忆系统设计中——不需要记住所有步骤，只需要记住"关键的决策节点和结果"。

### 4. Judge 模式是构建可靠 Agent 的重要组件

用单独的 LLM/Judge 来评判多个候选轨迹的质量，而不是依赖单次执行的置信度——这实际上是一种**集思广益（ensemble）思路在轨迹级别的应用**。这对于构建生产级可靠的 AI Agent 系统有直接参考价值。

### 5. 代码执行能力是 GUI 智能体的"第二把剑"

S3 允许智能体在 GUI 操作和代码执行之间自主选择，这本质上扩展了智能体的能力边界——GUI 处理不了的文件处理、数据操作、系统配置等交给代码执行。这是一种**混合操作模式**（hybrid action mode），值得在任何复杂 Agent 系统中借鉴。

## 六、总结

Agent S3 是 2025 年最具技术突破性的开源 AI Agent 项目之一。它通过：
- **双模型解耦架构**解决 GUI grounding 难题
- **Behavior Best-of-N**开辟了行为级 scaling 新维度
- **简化设计**证明了"less is more"在 Agent 架构中的价值

最终在 OSWorld 上以 72.6% 超越人类基准（72.36%），这不仅是数字上的超越，更是 AI Agent 能否可靠替代人类操作计算机的重要里程碑。

> **一句话总结**：Agent S3 的核心贡献不是"做了一个更好的 GUI Agent"，而是证明了**通过对多次执行的行为级评判和选择，可以系统性地消除长时域任务的高方差问题**——这是一种通用性很强的思路，适用于任何需要可靠执行的生产级 Agent 系统。
