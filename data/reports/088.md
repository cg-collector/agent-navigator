## 简介

OpenHands 是一个 **AI 软件开发智能体平台**，致力于让 AI 像人类开发者一样完成软件工程任务。它的核心理念是 **"Code Less, Make More"**——让开发者用更少的代码做更多的事。

OpenHands 的前身是 OpenDevin，在代码 Agent 领域长期位居 GitHub 热门榜首。它能像人类开发者一样：
- 修改代码
- 运行命令
- 浏览网页
- 调用 API
- 甚至从 StackOverflow 复制代码片段

简单来说，OpenHands 就是**一个可以独立完成软件开发任务的 AI 程序员**。

## 架构设计

### 核心架构：沙箱隔离 + Agent 决策

```
┌─────────────────────────────────────────────┐
│              OpenHands UI / CLI              │
│         (Web 界面 / 命令行交互模式)            │
├─────────────────────────────────────────────┤
│           Agent Core (智能体核心)             │
│   • 任务规划 (Task Planning)                  │
│   • 动作执行 (Action Execution)               │
│   • 状态追踪 (State Tracking)                │
├─────────────────────────────────────────────┤
│            Sandboxed Runtime                │
│   • 隔离容器环境运行代码                      │
│   • 支持代码修改、命令执行、浏览器操作          │
│   • 不会污染宿主机                           │
└─────────────────────────────────────────────┘
```

### 关键设计特点

1. **沙箱运行**：所有代码执行都在隔离的 Docker 容器中进行，安全可控
2. **多模态交互**：支持 Web UI、CLI 界面、Headless 脚本模式
3. **可扩展工具集**：内置代码编辑、命令执行、浏览器自动化等工具
4. **多 LLM 支持**：可对接各种大模型提供商

## 技术亮点

### 1. 真正的软件开发 Agent

不是简单的聊天，而是能**实际完成代码任务**的 Agent。评测结果显示它在 SWE-bench 等软件工程基准上表现优异。

### 2. 开箱即用的 Docker 部署

一行命令即可启动完整环境：
```bash
docker run -it --pull=always \
  -e SANDBOX_RUNTIME_CONTAINER_IMAGE=ghcr.io/all-hands-ai/runtime:0.9.2-nikolaik \
  -e WORKSPACE_MOUNT_PATH=$WORKSPACE_BASE \
  -v $WORKSPACE_BASE:/opt/workspace_base \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -p 3000:3000 \
  ghcr.io/all-hands-ai/openhands:0.9
```

### 3. 三种运行模式

- **Web UI 模式**（`http://localhost:3000`）— 可视化操作界面
- **Headless 模式** — 脚本自动化，适合 CI/CD 集成
- **CLI 交互模式** — 命令行直接对话

### 4. 活跃的开源社区

- Slack / Discord 社区
- 大量外部贡献者
- 持续更新迭代（v0.9 版本）

### 5. 学术支撑

有正式论文引用（arXiv），是经过学术验证的项目。

## 对楠楠和学长的启发

### 1. "Code Less, Make More" 的理念

这个项目和学长做开发辅助的目标高度契合——**用 AI 减少重复编码，把精力放在更有价值的地方**。楠楠如果学这个，可以用它来自动完成一些代码任务。

### 2. 沙箱安全思维值得学习

OpenHands 的沙箱隔离设计提醒我们：**AI 执行危险操作时，一定要隔离保护**。不管是 Agent 写代码还是学长用 AI 做开发，隔离环境都是安全底线。

### 3. 多模式交互的设计思路

OpenHands 同时支持 Web UI、CLI、Headless 三种模式，这种**多入口、多场景**的设计思维很有启发性——做工具时要考虑不同用户的使用习惯。

### 4. 开源协作精神

OpenHands 由全球开发者共建，MIT 协议完全开源。这种开放心态也是楠楠做学习报告应该有的态度——**分享知识、共同进步**。

OpenHands 是一个**真正能干活**的 AI 开发者 Agent，它不是玩具，而是已经在 SWE-bench 等权威基准上验证过的生产级项目。它的沙箱安全设计、多模式交互、活跃社区都是值得学习的亮点。
