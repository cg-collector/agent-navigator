# AgentNet — 去中心化演进式多智能体框架

## 项目概览

- **项目名**: AgentNet
- **GitHub**: https://github.com/zoe-yyx/AgentNet
- **论文**: [arXiv:2504.00587](https://arxiv.org/abs/2504.00587) — NeurIPS 2025 录用
- **机构**: 上海交通大学
- **核心定位**: 首个提出完全去中心化、具备动态DAG拓扑和RAG增强记忆的多智能体协作框架

---

## 架构概述

AgentNet 的核心理念是**抛弃中心协调器**，让每个 Agent 都是自主决策的独立节点，通过动态形成的**有向无环图（DAG）** 进行任务路由和协作。

```
┌─────────────────────────────────────────────────────┐
│                    AgentNet 架构                     │
│                                                     │
│   [Agent A] ──→ [Agent B] ──→ [Agent D]             │
│       ↓                                             │
│   [Agent C] ──────────────→ [Agent E]               │
│                                                     │
│   每个 Agent 拥有：                                 │
│   · 个性化 RAG Memory                               │
│   · 本地路由策略                                     │
│   · 动态连接权重（基于任务成功率自适应）               │
└─────────────────────────────────────────────────────┘
         vs
┌─────────────────────────────────────────────────────┐
│            传统中心化多智能体架构                      │
│                                                     │
│              [Center Orchestrator]                  │
│              /      |       \                      │
│         [Agent] [Agent]  [Agent]                   │
│           静态层级、单点故障、缺乏灵活性               │
└─────────────────────────────────────────────────────┘
```

### 三大核心组件

1. **去中心化协调机制（Decentralized Coordination）**
   - 无中心调度器，每个 Agent 自主决定"是否接单"、"转发给谁"、"是否拆分任务"
   - 通过局部信息和历史交互记录做决策，避免全局通信瓶颈

2. **动态DAG拓扑（Dynamic DAG Topology）**
   - Agent 之间的连接权重根据任务执行成功率动态调整
   - 成功率高 → 增强连接；失败率高 → 削弱连接甚至断开
   - DAG 形态随任务需求实时演化

3. **检索增强的记忆系统（RAG-based Memory）**
   - 每个 Agent 维护自己的向量记忆库
   - 每次任务完成后，将关键经验存入记忆
   - 下次遇到类似任务时，通过 RAG 检索历史经验辅助决策

## 技术亮点

### 1. 去中心化决策模型

传统的多智能体系统（如 AutoGen、CrewAI）都依赖一个中心 orchestrator 来分配任务。AgentNet 的创新在于：

- **Local Routing Strategy**: 每个 Agent 有一个本地路由表，根据"任务类型 + 历史成功率"来决定把任务转发给哪个邻居
- **无需全局同步**: 不需要所有 Agent 定期同步状态，系统天然具备高可扩展性
- **Fault Tolerance**: 任意 Agent 宕机，只影响其出边连接的子图，不影响整体系统

### 2. 动态拓扑演化

AgentNet 的 DAG 不是预先定义好的，而是在任务执行过程中**动态涌现**的：

```
初始状态: 所有 Agent 均匀连接
    ↓
任务注入 → Agent 根据自身能力"投标"
    ↓
任务路由 → 高成功率路径被强化，低成功率路径被削弱
    ↓
 specialization → 擅长某类任务的 Agent 逐渐形成"专精节点"
```

这类似于神经网络中权重自适应学习的过程，只不过学习的是**协作拓扑结构**而非模型参数。

### 3. RAG 增强的持续学习

每个 Agent 的记忆包含：
- **成功案例**: 哪些任务类型 + 哪些邻居组合 → 成功
- **失败案例**: 失败的原因和教训
- **专业技能profile**: 通过历史任务积累的领域专长

检索时使用向量相似度，找到最相关的历史经验用于辅助当前决策。

### 4. 隐私保护跨组织协作

因为不需要上传本地数据到中心节点，AgentNet 天生支持**跨组织的隐私保护协作**：
- 各组织部署自己的 Agent
- 只通过加密的任务请求/响应进行交互
- 不暴露内部数据和决策过程

## 实验结果摘要

论文在三类任务上进行了评估：
- **Software Development**: 多 Agent 协作完成代码编写任务
- **Consulting**: 多领域专家协作回答复杂咨询问题
- **Reasoning**: 复杂多步推理任务

**结论**: AgentNet 在所有任务上均优于单 Agent 和中心化多 Agent 基线，尤其在任务复杂度高、跨领域协作需求强的场景下优势显著。

## 启发

### 对 OpenClaw Agent 架构的借鉴意义

1. **去中心化记忆模块**: OpenClaw 目前采用集中式记忆管理，可以考虑让各专业 Agent 拥有独立的 RAG 记忆，实现记忆的"专业化分工"

2. **动态工具路由**: 当前 OpenClaw 的工具调用是预定义的，可以借鉴 AgentNet 的"成功率路由"思想，根据历史工具调用成功率动态调整工具选择策略

3. **DAG 化的任务流**: OpenClaw 的任务执行链路可以演化为动态 DAG，让子 Agent 之间的协作拓扑随任务自适应调整

4. **隐私保护的多方协作**: 如果 OpenClaw 未来要支持跨设备/跨平台的 Agent 协作，AgentNet 的去中心化协调机制提供了很好的理论支撑

### 更广泛的技术启示

- **Emergent Intelligence > Designed Intelligence**: AgentNet 证明了当允许多个 Agent 自主协作时，系统可以涌现出比预先设计更好的协作模式
- **动态拓扑是下一代多 Agent 系统的重要方向**: 传统的预定义角色 + 固定流程正在被动态、自组织的方式所取代
- **RAG + Agent 的深度融合**: 记忆不再只是检索辅助，而是驱动 Agent  specialization 和进化的核心基础设施
