> **研究日期：** 2026-04-16

> **文档地址：** https://humansignal.github.io/Adala/

> **首次发布：** 2023 年 11 月

## 一、项目概述

Adala（Autonomous DAta Labeling Agent）是由 HumanSignal 公司开源的自主数据处理智能体框架。作为知名标注工具 Label Studio 的母公司，HumanSignal 将其在数据标注领域多年积累的经验注入到 Adala 的设计中，打造了一个专注于**数据标注与处理**场景的 Agent 框架。

与 CrewAI、AutoGen 等通用多智能体编排框架不同，Adala 从诞生之初就有着明确的垂直定位——**让 AI Agent 自主完成数据标注、分类、摘要生成等数据处理任务**。它的核心理念是：Agent 不应只是被动执行指令的工具，而应具备通过迭代学习不断精进技能的能力。

Adala 的名称本身就揭示了其定位：Autonomous DAta（Labeling）Agent，强调"自主性"和"数据"两个关键词。在 LLM 驱动的自动化标注工具层出不穷的背景下，Adala 通过引入**技能学习（Skill Learning）、记忆系统（Memory）和人机协同（Human-in-the-Loop）**三大机制，试图解决纯 LLM 标注中常见的可靠性不足、不可控输出等痛点问题。

## 二、核心架构

Adala 的架构设计围绕六个核心组件展开，形成了一个清晰的分层体系：

### 2.1 Agent（智能体）

Agent 是用户直接交互的主入口，负责协调所有其他组件的工作。一个 Agent 实例可以拥有多个 Skill，连接一个 Environment，配置多个 Runtime，并维护自己的 Memory。Agent 提供了 `learn()` 和 `run()` 两个核心方法——前者用于从 ground truth 数据中学习技能，后者用于对新数据进行推理预测。

```python
agent = Agent(
    environment=StaticEnvironment(df=train_df),
    skills=ClassificationSkill(
        name='sentiment',
        instructions="Label text as positive, negative or neutral.",
        labels={'sentiment': ["Positive", "Negative", "Neutral"]},
    ),
    runtimes={'openai': OpenAIChatRuntime(model='gpt-3.5-turbo')},
)
agent.learn(learning_iterations=3, accuracy_threshold=0.95)
predictions = agent.run(test_df)
```

### 2.2 Skill（技能）

Skill 是 Adala 最核心的抽象概念，定义了 Agent 能够执行的特定数据处理能力。每个 Skill 包含：
- **instructions**：自然语言描述的任务指令
- **input_template / output_template**：输入输出的结构化模板
- **labels**：约束输出范围的标签集合

框架内置了 ClassificationSkill（分类）、SummarizationSkill（摘要）等常用技能，同时也支持用户自定义 Skill。Skill 的设计理念是**可组合、可复用**——复杂任务可以通过组合多个简单 Skill 来实现。

### 2.3 Runtime（运行时）

Runtime 是 Agent 执行技能的实际计算后端，本质上是对 LLM API 的抽象封装。Adala 初始支持 OpenAI Chat Runtime 作为默认运行时，其设计允许可扩展地接入更多 LLM 提供商。Runtime 的抽象层使得同一个 Skill 可以在不同模型上运行和切换，为成本优化和效果对比提供了便利。

值得一提的是，Adala 引入了 **Teacher Runtime** 的概念——允许使用更强的模型（如 GPT-4）作为"老师"来指导使用较弱模型（如 GPT-3.5-turbo）的学生 Agent 进行学习，这种师生分离的设计在成本敏感的场景下具有实际价值。

### 2.4 Environment（环境）

Environment 定义了 Agent 的操作环境，主要提供 ground truth 数据集作为学习的监督信号。内置的 StaticEnvironment 接受 pandas DataFrame 作为输入，用户也可以自定义 Environment 来对接外部数据源（如数据库、文件系统或 Label Studio 的标注结果）。Environment 本质上回答了"Agent 在什么上下文中学习"这个问题。

### 2.5 Memory（记忆）

Memory 组件是 Adala 区别于许多简单 LLM 包装器的关键差异点。它为 Agent 提供了持久化的知识存储空间，记录：
- 历次学习的错误样本和纠正反馈
- 人类审核者提供的修正信号
- 技能演化的中间状态

当 Agent 再次遇到类似情况时，可以从 Memory 中检索历史经验作为起点，避免重复犯错。这种设计借鉴了人类"从经验中学习"的认知模式。

### 2.6 可靠性保障机制

为确保 Agent 输出的可靠性，Adala 设计了两道防线：

1. **监督集成（Supervision Integration）**：通过 ground truth 数据集设定学习边界，确保 Agent 的输出不会偏离预期范围。
2. **约束生成（Constrained Generation）**：通过 labels 定义强制约束输出空间，Agent 只能在预定义的标签集中选择，杜绝幻觉输出。

## 三、技术亮点

### 3.1 迭代式技能学习

Adala 最突出的技术创新是其**自主技能习得机制**。不同于传统的 few-shot prompt engineering 或 fine-tuning 方案，Adala 采用了一种轻量级的在线学习范式：Agent 在 Environment 中反复观察 ground truth 数据，根据预测误差调整自身行为，经过若干轮迭代后逐步收敛到满意的准确率。整个过程无需梯度更新，完全基于 LLM 的 in-context learning 能力实现。

`learn()` 方法支持设置 `learning_iterations`（学习轮数）和 `accuracy_threshold`（准确率阈值），Agent 会自动在学习达到阈值或耗尽迭代次数后停止。这种**自终止学习循环**的设计使得非专业用户也能安全地使用 Agent，无需手动调参。

### 3.2 人机协同（Human-in-the-Loop）

Adala 将人类反馈深度嵌入到 Agent 的工作流中。在每一轮学习中，Agent 的预测都会与 ground truth 对比，差异部分自动标记供人工审核。人类审核者可以选择接受/拒绝预测结果，甚至提供详细的修改理由。这些反馈信号会写入 Memory，影响后续的学习过程。

这种人机协同模式特别适合需要高精度标注的场景（如医疗文本分类、金融合规审查），既利用了 LLM 的效率优势，又保留了人类的最终把关能力。

### 3.3 与 Label Studio 生态的无缝集成

作为 HumanSignal 产品家族的一员，Adala 天然支持与 Label Studio 的集成。用户可以用 Label Studio 创建高质量的 ground truth 数据集，然后直接导入 Adala 作为 Environment 使用；反之，Adala 的预测结果也可以回传到 Label Studio 进行人工审核。这种闭环工作流在企业级数据标注场景中极具吸引力。

### 3.4 灵活的输出约束

通过 output_template 和 labels 的配合，Adala 实现了**结构化约束生成**。这在实际应用中极为重要——它从根本上消除了 LLM 随机输出格式错误或产生不在标签集中的"幻觉"答案的可能性。对于生产环境中的数据处理流水线来说，这种确定性保证是不可妥协的需求。

### 3.5 模块化与可扩展设计

Adala 的六大组件均为可插拔模块。开发者可以：
- 自定义 Skill 处理特定领域的标注任务
- 编写新的 Runtime 接入不同的 LLM 提供商
- 实现 Environment 接口对接各类数据源
- 扩展 Memory 后端以支持向量数据库或图数据库

这种模块化设计使 Adala 既适合快速原型验证，也能适应企业级的定制化需求。

## 四、适用场景

### 4.1 自动化数据标注

这是 Adala 的核心应用场景。对于需要大量标注数据的 ML 项目，Adala 可以显著降低人工标注成本。典型流程包括：先用少量人工标注数据训练 Agent → Agent 批量预标注新数据 → 人工审核校正 → Agent 从反馈中继续改进。

### 4.2 文本分类与情感分析

Adala 内置的 ClassificationSkill 使其开箱即用地适用于各种文本分类任务，包括情感分析、意图识别、话题分类、垃圾邮件检测等。约束生成机制保证了分类结果的规范性。

### 4.3 文本摘要与信息抽取

通过 SummarizationSkill 和自定义 Skill，Adala 可以处理长文档摘要、关键信息提取、实体识别等 NLP 任务。对于需要将非结构化文本转化为结构化信息的知识管理场景尤其适用。

### 4.4 数据质量审核与清洗

利用 Agent 的自主学习能力，Adala 可以被训练来检测数据集中的异常值、不一致性和潜在错误，充当自动化的数据质量守门人角色。

### 4.5 教育与研究

由于代码简洁、概念清晰，Adala 也适合作为教学工具帮助学生理解 Agent 框架的基本原理，以及作为研究平台探索自主 Agent 学习的新方法。

## 五、局限性

### 5.1 领域聚焦的双刃剑

Adala 明确定位于数据处理/标注场景，这意味着它**不适合**构建通用对话机器人、代码生成助手、多步推理 Agent 等其他类型的 AI 应用。如果你需要一个全能型 Agent 框架，CrewAI 或 LangGraph 可能更合适。这种专注既是优势也是限制。

### 5.2 Runtime 支持有限

截至研究时，Adala 主要支持 OpenAI 系列 Runtime。虽然架构上支持扩展，但社区贡献的其他 Runtime（如 Anthropic、本地模型等）相对较少。对于希望使用开源模型或特定 LLM 提供商的用户来说，可能需要自行编写 Runtime 适配器。

### 5.3 缺乏多 Agent 协作能力

Adala 的设计围绕**单 Agent + 多 Skill** 模式展开，不支持多 Agent 之间的通信、协商或分工协作。对于需要多个专业化 Agent 配合完成的复杂任务（如一个 Agent 负责搜索、另一个负责分析、第三个负责撰写报告），Adala 无法原生支持。

### 5.4 学习效率依赖 LLM 能力

Adala 的迭代学习完全依赖底层 LLM 的 in-context learning 能力。对于复杂的细粒度分类任务（如数十个类别的多标签分类），弱模型可能需要大量迭代才能收敛，而每次迭代都意味着 API 调用成本。在某些场景下，传统的 fine-tuning 方案可能在成本和效果上都更有优势。

### 5.5 社区活跃度下降

Adala 于 2023 年 11 月发布初期获得了一定关注，但随着 2024-2025 年 Agent 框架赛道竞争白热化（CrewAI、LangGraph、AutoGen 等快速迭代），Adala 的社区活跃度和更新频率有所放缓。对于寻求长期维护保障的企业用户而言，这是一个需要考量的风险因素。

### 5.6 文档与示例不够丰富

相比 CrewAI 或 LangGraph 拥有详尽的文档、教程和社区案例，Adala 的文档相对精简，高级用法和生产部署指南较为缺乏。新用户在上手过程中可能会遇到资料不足的问题。

## 七、与同类框架对比

### 7.1 vs. Label Studio 内置的 LLM 辅助标注

Label Studio 本身也在产品中集成了 LLM 辅助标注功能。两者的区别在于：Label Studio 的集成偏向**交互式辅助**（在 UI 中调用 LLM 预标注，人工逐条确认），而 Adala 是一个**独立的编程框架**，支持批量自动化处理和编程化控制。Adala 更适合需要嵌入到数据流水线中的自动化场景。

### 7.2 vs. Smolagents（HuggingFace）

Smolagents 是 HuggingFace 出品的极简 Agent 框架，采用"代码优先"的设计哲学（#352 已详细分析）。两者都强调简洁易用，但定位不同：Smolagents 是通用型框架，Agent 通过生成和执行 Python 代码完成任务；Adala 则专注于数据处理，通过 Skill + 约束生成确保输出可靠性。如果你的任务是数据标注，Adala 的约束生成机制比 Smolagents 的自由代码执行更安全可控。

### 7.3 vs. AutoGen（Microsoft）

AutoGen（现整合进 Microsoft Agent Framework，#355 已分析）专注于多 Agent 对话协作，支持复杂的 Agent 间协商和多步推理。Adala 则是单 Agent + 多 Skill 模式，不涉及多 Agent 通信。在数据处理这个垂直领域，Adala 的 Skill 抽象和约束生成比 AutoGen 的对话范式更加直观和高效。

### 7.4 vs. 手动 Prompt Engineering

最直接的替代方案是不使用任何框架，直接编写 prompt 调用 LLM API 进行批量标注。Adala 相对此方案的优势在于：（1）提供了结构化的学习循环，无需手动管理 few-shot 示例；（2）Memory 机制避免了重复犯错；（3）约束生成消除了格式错误；（4）人机协同流程标准化。劣势则是引入了额外的框架依赖和学习成本。

## 八、总结评价

Adala 是一个**定位精准、设计清晰**的垂直领域 Agent 框架。它在数据处理/标注这一细分场景中提供了一个完整的解决方案——从技能定义、迭代学习、记忆积累到人机协同，形成了一套完整的方法论。

其最大的价值在于将 Agent 的"自主性"落到了实处：不是空泛地宣称"AI 自主工作"，而是通过具体的 Skill Learning 机制让 Agent 真正能够从数据和反馈中改进。约束生成和 Memory 两大机制则为生产环境的可靠性提供了工程保障。

然而，Adala 也面临着来自通用型 Agent 框架的挤压——随着 LangGraph、CrewAI 等框架不断丰富其生态插件，许多原本需要专用框架才能完成的任务现在可以用通用框架 + 自定义组件来实现。在这种趋势下，Adala 的生存空间取决于能否在数据处理的深度上持续建立壁垒，以及 HumanSignal 是否愿意持续投入资源进行维护和推广。

**推荐指数：★★★☆☆** — 如果你正在寻找一个专门用于数据标注/处理的 Agent 框架，Adala 值得尝试；但对于通用 Agent 开发需求，建议优先考虑生态更丰富的通用型框架。
