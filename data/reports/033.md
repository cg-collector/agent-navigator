# 开源Agent项目学习 #033

## 项目名称
Cline（原名 Claude Dev）

## GitHub链接
https://github.com/cline/cline

## 一句话简介
运行在 VS Code / JetBrains IDE 内的自主编程 AI Agent，具备文件编辑、终端执行、浏览器操作、MCP 扩展等能力，累计安装量超 500 万，是目前最受欢迎的开源编程 Agent 插件。

## 项目架构

Cline 的核心是一个 **Human-in-the-Loop（人在回路）** 的 Agent Loop：

```
用户任务 → 读取代码/AST → 制定计划 → 逐步执行 → 用户审批 → 完成
                ↑                          ↓
                └── linter/compiler 反馈 ←──┘
```

### 分层架构：

1. **IDE 扩展层（VS Code / JetBrains）** — 提供聊天面板、文件差异预览、终端集成、审批按钮等 GUI
2. **Agent Loop 核心层** — 基于 Claude 等 LLM 的自主决策循环，包含 Plan/Act 两种模式
   - **Plan 模式**：先输出完整计划，用户确认后再执行
   - **Act 模式**：边思考边执行，适合熟悉后的快速迭代
3. **工具集层** — Agent 可调用的核心能力：
   - 文件读写与编辑（diff-based，精确到行）
   - 终端命令执行（用户审批后）
   - 浏览器自动化（headless，点击/输入/截图/日志）
   - 代码搜索（AST 分析 + 正则搜索）
   - MCP 协议扩展（可接入任意 MCP Server）
4. **模型适配层** — 支持 OpenAI、Anthropic、Google Gemini、AWS Bedrock、Azure、OpenRouter、Ollama 等多种后端

### 关键设计：
- **上下文管理**：对大项目不会简单塞入全部代码，而是通过 AST 分析、正则搜索、按需读取来精准选择相关文件
- **实时反馈循环**：执行命令后监控 linter/compiler 输出，自动修复语法错误、缺少 import 等问题
- **成本追踪**：逐请求和总任务统计 token 用量和费用

## 技术亮点

- **Human-in-the-Loop GUI 审批**：不同于纯终端的 Agent（如 Aider、Codex CLI），Cline 将审批机制深度融入 IDE，每个文件修改和终端命令都需用户确认，在安全性和自主性之间找到平衡点
- **MCP 原生集成**：通过 Model Context Protocol 扩展自身能力，可接入数据库、API、自定义工具等，实现"Agent 自我进化"
- **浏览器自动化闭环**：对于 Web 开发任务，能启动 headless 浏览器实际运行页面、截图、读控制台日志，形成"改代码→跑起来→看效果→修bug"的完整闭环
- **精准上下文选择**：不是暴力塞代码，而是通过 AST + 搜索智能选择相关文件，在上下文窗口有限时尤其关键
- **多模型多后端**：不绑定单一供应商，从 Anthropic 到本地 Ollama 都支持

## 适用场景

- 日常软件开发：新功能开发、bug 修复、代码重构
- 从设计稿/截图生成前端页面（支持图片输入）
- 遗留代码理解和迁移
- 自动化测试编写
- 多文件跨模块的重构任务
- 任何需要在 IDE 内"跟AI结对编程"的场景

## 对我们的启发

1. **审批机制的借鉴**：OpenClaw 已经有 `/approve` 机制，但 Cline 的 diff-based 审批（展示具体改动再确认）更精细。可以借鉴到 OpenClaw 的文件编辑审批流程中
2. **上下文管理策略**：Cline 对大项目的"按需读取"策略（AST + 搜索 → 精准选择文件）对楠楠处理复杂任务很有参考价值，避免上下文爆掉
3. **Plan/Act 双模式**：先规划再执行 vs 边想边做，这种灵活度值得在 coding-agent skill 中借鉴
4. **实时反馈循环**：执行命令后自动检查错误并修复，这个模式可以应用到楠楠帮学长写代码时的工作流中
5. **浏览器闭环测试**：改完代码自动跑起来看效果，这个思路可以用在楠楠帮学长做 Web 开发任务时
