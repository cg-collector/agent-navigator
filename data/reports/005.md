## 项目简介

Letta 前身是 **MemGPT**（MemoryGPT），是一个专注于**有状态持久记忆AI Agent**的开源平台。它的核心理念是：传统LLM是"无状态的"——每次对话都是孤立的，而真正的AI Agent应该能够**跨会话学习、记忆和自我改进**。

Letta 提供了：
- **Letta API**：将状态持久化Agent嵌入任何应用
- **Letta Code**：终端里的记忆优先编程助手，类似Claude Code但有持久记忆
- **Letta Platform**：托管服务，也支持完全自托管（Docker）

特点：**模型无关**（支持GPT-5.2、Claude 4.5 Sonnet、Gemini 3 Pro、GLM-4.7等），有开源SDK（Python/TypeScript），超过100位社区贡献者。

## 架构设计

### 整体定位

Letta 把自己定位为**AI Infrastructure**，而不是另一个"让LLM调用工具"的框架。它的核心差异化在于**记忆系统**，而非工具调用。

### Agent Loop 架构演进

Letta 团队深入分析了三种Agent循环架构：

**1. ReAct（早期）**
- LLM生成 thought → action → (stop token)
- 通过格式化的prompt实现，出现在tool calling和native reasoning之前

**2. MemGPT（Letta前身，第一代有状态Agent）**
- 所有操作都是tool call，包括assistant message（通过特殊的send_message工具）
- 通过tool参数控制：
  - `thinking`：模型的推理/思维链
  - `request_heartbeat`：请求继续执行
- 优点：工具规则（tool rules）可以强制约束调用模式
- 缺点：依赖模型可靠的tool calling能力

**3. Letta V1（当前最新架构）**
- **废弃heartbeats和send_message工具**
- 只支持native reasoning和直接assistant message生成
- 底层使用 OpenAI Responses API，支持加密的跨提供商推理
- 优势：
  - 充分利用模型的原生推理能力
  - **任何LLM都能接入**（不再要求tool calling能力）
  - system prompt更简洁
- 局限：
  - 非推理模型（GPT-4o mini等）不再产生推理
  - 无法跨模型传递/修改推理trace
  - 定时/后台任务需要自定义prompt实现

### 记忆系统架构（核心亮点）

Letta的记忆系统受到**操作系统内存管理**的启发，将记忆分为多个层级：

```
┌─────────────────────────────────────────┐
│         Context Window (类似CPU缓存)       │
│  ┌─────────────┐  ┌────────────────────┐  │
│  │ Message     │  │ Core Memory Blocks │  │
│  │ Buffer      │  │ (in-context, pinned) │  │
│  │ (最近消息)   │  │ - human block       │  │
│  └─────────────┘  │ - persona block     │  │
│                    └────────────────────┘  │
├─────────────────────────────────────────┤
│         Recall Memory (类似磁盘存储)        │
│  完整对话历史 → 可搜索 → 需要时召回         │
├─────────────────────────────────────────┤
│         Archival Memory (外部数据库)        │
│  向量数据库 / 图数据库 → 嵌入存储 → RAG召回 │
└─────────────────────────────────────────┘
```

**记忆管理技术**：
- **Message Eviction**：上下文窗口满时，智能驱逐消息（通常驱逐70%）
- **Recursive Summarization**：被驱逐的消息递归摘要，越老的摘要越精简
- **Memory Blocks**：结构化、可编辑的上下文单元，Agent可以自己修改
- **Sleep-Time Compute**：后台异步的记忆Agent，不阻塞主对话线程

### 两种记忆类型对比

| 类型 | 位置 | 持久性 | 管理者 |
|------|------|--------|--------|
| Short-term | 消息缓冲区 | 会话结束丢失 | 自动 |
| Long-term | Recall + Archival | 持久化到磁盘/数据库 | 自动+手动 |

## 核心技术亮点（≥3个）

### 1. OS风格的记忆层级管理（MemGPT遗产）

Letta将操作系统的内存管理思想引入AI Agent：
- **核心记忆（RAM）**：固定在上下文窗口中，Agent随时可访问
- **召回记忆（磁盘）**：完整历史，搜索召回
- **归档记忆（外部存储）**：向量/图数据库，长期知识

这解决了LLM上下文窗口有限但需要长期记忆的矛盾。

### 2. Sleep-Time Compute（睡眠计算）

Letta创新性地引入了**异步后台Agent**专门处理记忆：
- 主Agent负责对话和任务
- 专门的sleep-time Agent在后台异步整理、总结、更新记忆
- **不阻塞主对话响应**，同时提高记忆质量
- 记忆在"空闲时间"被主动重组优化，而非被动驱逐

### 3. 记忆作为上下文工程（Memory as Context Engineering）

Letta团队提出一个重要观点：Agent记忆本质上是**上下文工程**——决定哪些token进入上下文窗口、如何组织。Agent的"记忆"不是复制人类记忆结构，而是精心设计的信息呈现方式。

这一理念指导下的实现：
- Memory Blocks是**可编辑**的抽象单元
- Agent可以通过工具**自己修改**自己的记忆block
- 支持多Agent协作修改同一个Agent的记忆

### 4. Letta Code：记忆优先的编程Agent

Letta Code是面向编程场景的杀手级应用，对比Session-based Agent（Claude Code/Codex）：

| 维度 | Claude Code / Codex | Letta Code |
|------|---------------------|------------|
| 会话独立性 | 每次会话独立 | 同一Agent跨会话 |
| 学习能力 | 会话间不学习 | 持久记忆，持续进化 |
| 上下文 | 当前消息+AGENTS.md | +记忆block+历史 |
| 关系定位 | 每次像雇新承包商 | 像带一个记住一切的同事 |

Letta Code支持 `/init`（初始化记忆系统）、`/remember`（主动记忆）、`/skill`（从轨迹中学新技能）等命令。

### 5. 完全模型无关的Agent架构

Letta V1新架构让**任何LLM都能作为Agent后端**，不再要求tool calling能力：
- 支持的模型：GPT全系列、Claude系列、Gemini、GLM、Moonshot等
- 通过Responses API支持native reasoning
- 有模型排行榜（leaderboard.letta.com）供参考

## 启发与学习点

### 对OpenClaw Agent设计的启发

1. **记忆系统优先级**：目前OpenClaw的Agent主要是无状态的（每次会话fresh），但Letta证明了**持久记忆是下一代Agent的核心差异点**。OpenClaw可以考虑引入轻量级的记忆block机制。

2. **Sleep-Time Compute的思路**：后台异步处理非紧急任务（如记忆整理）而不阻塞主流程，这个设计模式可以迁移到OpenClaw的sub-agent管理中。

3. **上下文窗口作为稀缺资源管理**：Letta将上下文窗口类比为OS内存，这给Agent设计提供了很好的心智模型。Agent应该有意识地管理"工作内存"vs"长期记忆"。

4. **Agent as Infrastructure**：Letta把自己定位为AI基础设施（Platform+SDK），而不只是一个框架。这种定位让它可以做托管服务、API商业化，同时保持开源。OpenClaw也是"平台级"定位，可以参考。

5. **模型演进敏感度**：Letta随着LLM的演进（从ReAct到MemGPT到V1）不断迭代架构，说明**Agent架构必须跟着模型能力进化**。OpenClaw也应该持续关注模型新能力（如o3/gemini 3的推理能力）并适配。

### 技术细节亮点

- **多语言SDK**：Python（核心）+ TypeScript（Letta Code CLI）+ 社区AUR包
- **自托管支持**：Docker一键部署，保护隐私
- **Skills系统**：用SKILL.md标准定义的技能，兼容Claude Code/Codex等
- **MCP+A2A协议**：支持MCP服务器和Agent间通信协议

## 推荐程度

**⭐⭐⭐⭐⭐（5/5）**

### 推荐理由：

1. **差异化明确**：不像大多数Agent框架聚焦在"工具调用"，Letta聚焦在"记忆和学习"，是真正稀缺的技术方向
2. **工程深度**：从OS内存管理获得灵感，记忆层级设计非常系统化，有学术论文支撑（MemGPT）
3. **实用性强**：Letta Code直接可用于日常编程，比Claude Code更适合长期项目伴侣
4. **社区活跃**：21k+ stars，100+贡献者，持续更新
5. **架构清晰**：V1架构对模型演进有清晰的认识和适配策略

- 想深入理解Agent记忆系统的人
