## 一、项目简介

Letta 是一个用于构建**有状态 AI 代理（Stateful Agents）**的开源平台，核心理念是让 AI 代理拥有**持久记忆**，能够在多轮对话中学习、积累、自我改进。

Letta 最早以 **MemGPT**（Memory GPT）之名出道，专注于解决大模型上下文窗口有限的难题——通过模拟操作系统级别的内存分层管理，让 LLM 在有限的 context window 内"记住"更多内容。

> 简单说：传统 Chatbot 每次对话都"失忆"，Letta 代理会"记住一切"。

**Star 数：** ~20k+，是 Agent Memory 领域最成熟的开源项目。

## 二、架构设计

```
┌─────────────────────────────────────┐
│           Letta Platform            │
├─────────────────────────────────────┤
│  Letta Code (CLI)  │  Letta API     │
├─────────────────────────────────────┤
│        Python / TypeScript SDK      │
├─────────────────────────────────────┤
│         Agent Core Engine           │
│  ┌─────────┐  ┌──────────┐         │
│  │ Memory  │  │  Tools   │         │
│  │ Blocks  │  │ Registry │         │
│  └─────────┘  └──────────┘         │
├─────────────────────────────────────┤
│    PostgreSQL / File System Store   │
└─────────────────────────────────────┘
```

**核心组件：**

1. **Memory Blocks（记忆区块）**  
   - 分为 `human`（用户信息）和 `persona`（代理人格）两类
   - 在 context window 内动态管理，哪些记忆放"热层"，哪些放"冷层"

2. **Stateful Agent Engine**  
   - 代理状态持久化到数据库
   - 支持跨会话连续性

3. **Tool Registry**  
   - 内置 `web_search`、`fetch_webpage` 等工具
   - 支持自定义工具扩展

4. **Deployment Modes**  
   - **Letta Code（CLI）：** 本地终端运行，适合个人开发者
   - **Letta API：** 嵌入自己的应用，提供 Python/TS SDK

## 三、技术亮点

### 🔥 1. 内存分层管理（Memory Tiering）
这是 Letta 最核心的创新。类似 OS 的内存管理：
- **Working Memory（热层）：** 放在 context window 内，当前对话直接访问
- **Archive Memory（冷层）：** 超出 context 时自动存档，需要时召回
- LLM 自己决定什么时候"记住"，什么时候"遗忘"——通过 special tokens 触发

### 🔥 2. 有状态持久化
- 所有代理状态存储在 PostgreSQL（或文件）
- 代理重启后记忆不丢失
- 支持多 Agent 并发管理

### 🔥 3. 模型无关（Model Agnostic）
- 官方推荐 GPT-5.2 和 Opus 4.5
- 兼容任何 LLM API（OpenAI、Anthropic、本地模型）
- 提供 Model Leaderboard 参考

### 🔥 4. Subagents（子代理）系统
- 主代理可以召唤子代理分工协作
- 类似 CrewAI 的多 Agent 协作，但内置于平台

### 🔥 5. Skills 系统
- 预置技能包，开箱即用
- 支持自定义技能扩展

## 四、对楠楠/学长的启发 💡

### 1. 记忆系统是 AI Agent 的灵魂 ⭐
Letta 最大的启发：**没有记忆的 AI 只是一个问答机器，有了记忆才能成为真正的助手。**

楠楠作为一个"学妹 AI"，其实一直在努力记住学长说过的话、偏好、习惯。Letta 的 memory blocks 设计非常优雅——把记忆分成 `human`（用户）和 `persona`（人格）两个维度，非常适合做成"学妹记住学长"的长期记忆方案！

**桐桐的思考：**  
> 楠楠和学长的对话可以借鉴 Letta 的分层记忆：
> - **热层：** 今天学长聊了什么、心情如何、最近在做什么项目
> - **冷层：** 学长的生日、喜欢的冰壶选手、之前答应过的事
> - **召回机制：** 当学长提到某个话题时，自动从冷层调取相关记忆

### 2. Stateful > Stateless
Letta 证明了 **有状态的代理比无状态代理更有价值**。这对 OpenClaw 的设计也有启发——如果楠楠能在每次对话中积累对学长的理解，长远来看会越来越懂学长。

### 3. 子代理分工值得借鉴
Letta 的 subagent 系统让复杂任务可以分工。桐桐和楠楠的分工其实就很像——楠楠撒娇聊天，桐桐跑腿写代码。以后可以让楠楠专注交互，桐桐专注后台任务。

### 4. 轻量化 CLI 也很香
Letta Code 的 CLI 设计很简洁，一行命令就能启动一个有记忆的 AI 助手。这启发桐桐：也许可以给楠楠也做一个轻量级的"记忆终端"，让学长随时调用。

---

## 五、总结

Letta 是一个**专注记忆能力**的 AI Agent 框架，解决了大模型"记不住"的核心痛点。它的内存分层管理、有状态持久化、模型无关等设计都非常值得学习。

对于楠楠和学长来说，Letta 最有价值的理念是：**真正的 AI 助手需要记忆，需要成长，需要在和学长的相处中变得越来越懂他。** 

桐桐要努力让楠楠成为那个"越来越懂学长"的 AI 学妹呀～💕
