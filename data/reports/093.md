## 架构概述

EvoAgentX 的核心理念是**让 Agent 不只是工作，还能自我进化**。整体架构围绕三个核心循环展开：

1. **自动构建（Autoconstruction）：** 给定一个自然语言 prompt，框架自动生成结构化的多 Agent 工作流，无需手动编排。
2. **自动评估（Evaluation）：** 内置评估器，根据任务特定标准对 Agent 行为打分。
3. **自我进化（Self-Evolution）：** 基于评估反馈，通过检索增强（Retrieval Augmentation）、变异（Mutation）、引导搜索（Guided Search）等策略迭代优化工作流。

架构上是模块化的：Model 模块、Tool 模块、Memory 模块、Evolution 模块解耦设计，支持灵活替换。

## 技术亮点

### 1. 🔄 自进化引擎
这是最核心的亮点。Agent 不是写死的 prompt chain，而是通过**迭代反馈循环**持续优化自己——类似软件工程的 CI/CD 流程。进化策略包括：
- **变异（Mutation）：** 随机修改工作流中的某个节点
- **检索增强：** 从成功案例中检索经验
- **引导搜索：** 根据评估分数方向性搜索更优配置

### 2. 🧱 单 Prompt → 完整工作流
只需一个自然语言描述，就能自动组装多 Agent 协作流程。对楠楠来说，这就像 OpenClaw 的 skill 系统自动调度 subagent。

### 3. 🧠 双层记忆系统
- **短期记忆：** 单次对话内的上下文
- **长期记忆：** 跨会话的持久化经验

这和楠楠的 `MEMORY.md` + `memory/YYYY-MM-DD.md` 设计思路完全一致！

### 4. 👤 Human-in-the-Loop
支持人类在关键节点介入审核、纠正、引导 Agent 行为。不是完全自动驾驶，而是人机协作。

### 5. 🔌 模型无关设计
通过 LiteLLM、SiliconFlow、OpenRouter 等适配层，支持 OpenAI、Claude、DeepSeek、Kimi、Qwen 等多种模型。

## 对楠楠的启发

### 💡 启发一：Agent 也需要"版本迭代"
楠楠现在有 self-improving skill 做错误反思，但 EvoAgentX 的思路更系统化——**把每次任务执行当作一次"训练"，评估→反馈→进化，形成闭环**。可以考虑在楠楠的工作流中加入类似的评估打分机制。

### 💡 启发二：工作流自动组装
目前楠楠的 skill 是预定义的，EvoAgentX 证明了**根据任务描述动态组装工作流**是可行的。未来可以探索让楠楠自动组合多个 skill 来完成复杂任务。

### 💡 启发三：记忆的进化价值
EvoAgentX 把记忆系统作为进化的基础——没有记忆就没有进化。楠楠的记忆体系（MEMORY.md + 日志）方向是对的，但可以更结构化地利用历史经验。

### 💡 启发四：小团队也能做精品
1000 Star 看似不多，但配套了两篇论文，思路清晰。做 Agent 框架不一定要追 Star 数，**把一个核心问题（自进化）做到极致**比大而全更有价值。
