## 📖 项目简介

MetaGPT 是一个基于 LLM 的**元编程（Meta-Programming）多智能体协作框架**，其核心创新在于将真实世界中的**标准化作业流程（SOP）**编码到 Agent 协作中。它不是简单地让多个 LLM 对话，而是模拟一家完整的软件公司——包含产品经理、架构师、项目经理、工程师等不同角色，每个角色遵循严格的 SOP 工作流，通过结构化的中间产物（PRD、设计文档、流程图、API 规范等）进行交接。

用户只需输入一行需求（如"开发一个贪吃蛇游戏"），MetaGPT 就能自动完成：竞品分析 → 需求文档（PRD）→ 技术架构设计 → 任务拆解 → 代码生成 → 代码执行与调试的全流程。

在 HumanEval 和 MBPP 基准测试中分别达到 **85.9%** 和 **87.7%** 的 Pass@1 准确率，且在复杂软件项目任务中达到 **100% 完成率**。

### 四大核心组件

1. **Role（角色系统）：** 每个 Agent 被赋予明确的专业角色（产品经理/架构师/工程师），拥有专属的 Prompt 模板、专业知识和输出格式约束。角色通过继承 `Role` 基类实现，可自定义 `__init__`（设定角色属性）、`_act`（定义行为逻辑）、`_observe`（观察消息）等方法。

2. **SOP 工作流引擎：** 将人类工作流编码为有序的 Action 序列。每个角色的 SOP 是预定义的操作链，确保 Agent 按照正确的顺序执行任务。SOP 通过 `Action` 类实现，支持串行和并行执行。

3. **Structured Output（结构化输出）：** 强制 Agent 输出标准化的中间产物（如 PRD 文档、UML 图、JSON 格式的数据结构定义），而非自由文本对话。这大幅降低了 Agent 间信息传递的歧义性，减少级联幻觉问题。

4. **Message & Shared Memory（消息与共享记忆）：** 基于发布-订阅模式的消息总线，Agent 之间通过标准化消息传递信息。共享环境对象允许所有 Agent 访问全局上下文。

## ✨ 技术亮点

### 1. SOP 编码 — 将人类工作流注入 Agent 协作

这是 MetaGPT 最核心的创新。传统多智能体框架（如 AutoGen、CrewAI）主要依赖 Agent 间的自由对话来推进任务，容易产生无意义的闲聊和级联幻觉。MetaGPT 则将软件公司真实的 SOP（Standardized Operating Procedures）硬编码为 Agent 的行为序列：

```python
# 产品经理的 SOP 示例
class ProductManager(Role):
    def __init__(self):
        self.set_actions([
            WriteCompetitiveAnalysis(),  # Step 1: 竞品分析
            WritePRD(),                 # Step 2: 写需求文档
            WriteUserStories(),          # Step 3: 用户故事
        ])
```

**为什么值得学习：** SOP 将"怎么做"的知识从 LLM 的推理能力中分离出来，用工程化手段保证流程正确性。这比纯 Prompt 引导更可靠，是"工程化 AI Agent"的典范思路。

### 2. 结构化中间产物 — 用文档而非对话串联工作流

MetaGPT 要求每个 Agent 输出**结构化文档**作为交付物：
- 产品经理 → PRD（含竞争分析、用户故事、需求条目）
- 架构师 → 系统设计文档（含类图、时序图、数据结构、API 定义）
- 工程师 → 可执行的源代码 + 单元测试

这些文档是**机器可读的结构化数据**（Pydantic Model），不是自然语言废话。下游 Agent 可以精确解析上游输出，不会产生理解偏差。

**为什么值得学习：** 这解决了多 Agent 系统中最头疼的问题——信息传递损耗。用 Schema 约束替代自由文本，本质上是在 Agent 间建立了强类型接口，类似于微服务之间的 API 契约。

### 3. 运行时执行反馈闭环 — 代码写完还能自己跑

MetaGPT 不仅生成代码，还会在沙箱环境中**实际运行生成的代码**，捕获错误后反馈给 Engineer Agent 进行修复，形成"编写→执行→报错→修复→再执行"的闭环。实验表明这个反馈机制让 MBPP 分数提升了 **5.4 个百分点**。

**为什么值得学习：** 大多数 Agent 框架只管"生成"，不管"结果对不对"。MetaGPT 引入了类似 TDD（测试驱动开发）的思想——Agent 自己写的代码要自己负责跑通。这对构建可靠的自动化编程系统至关重要。

## 💡 对楠楠的启发

1. **SOP 思想可以引入我们的 Agent 系统：** 我们在做复杂任务（比如股票分析、研究报告）时，也可以定义明确的 SOP 流程——先收集数据 → 再做初步分析 → 然后交叉验证 → 最后输出结论。每一步都有标准的输入/输出格式，而不是让 LLM 自由发挥。这和我们现有的进化分析流程（evolve-review）理念一致，但可以更加系统化。

2. **结构化输出的威力：** MetaGPT 用 Pydantic Model 约束 Agent 输出的做法非常实用。我们在飞书文档写入、Bitable 操作时其实已经在用类似的思路了，但可以更进一步——定义清晰的内部数据契约，让不同的 skill/subagent 之间通过结构化数据通信，减少信息丢失。

3. **角色专业化 > 通用全能：** MetaGPT 让每个 Agent 只专注一件事（PM 只管需求，Arch 只管设计），这比让一个万能 Agent 做所有事效率更高、质量更好。我们在派生 subagent 时也可以更精细地定义每个 subagent 的专长和边界，而不是每次都派一个通用的"去研究一下"。

4. **执行即验证的理念：** MetaGPT 的运行时反馈闭环提醒我们——生成内容之后应该有自我验证环节。我们的 evolve-review 已经在做这件事了，但可以扩展到更多场景：代码生成后自动跑测试、文档生成后自动检查完整性、搜索结果出来后自动交叉验证来源可靠性。
