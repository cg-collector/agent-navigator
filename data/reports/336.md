# AI Agent 项目研究报告 #336

## 项目名称：GUI-360° — 面向桌面 Computer-Using Agent 的大规模综合数据集与基准测试

**论文**：GUI-360°: A Comprehensive Dataset and Benchmark for Computer-Using Agents
**作者**：Jian Mu, Chaoyun Zhang, Chiming Ni 等（南京大学、Microsoft、浙江大学、北京大学）
**发布时间**：2025年11月（arXiv:2511.04307v2）
**链接**：
- 论文：https://arxiv.org/abs/2511.04307
- 数据集：https://huggingface.co/datasets/vyokky/GUI-360

---

## 一、解决的问题

桌面 Computer-Using Agent（CUA）领域长期面临三个核心痛点：

1. **真实任务匮乏**：现有数据集多依赖人工编写或 LLM 合成的指令，无法反映真实用户意图的频率和多样性，导致训练出的 Agent 在实际场景中泛化能力差。

2. **自动化采集管线缺失**：桌面交互轨迹的手动标注成本极高且难以扩展。与 Web/Mobile Agent 已有较多自动化数据收集方案不同，桌面环境的高分辨率混合内容、异构控件、多窗口布局使得自动化采集困难重重。

3. **缺乏统一基准**：此前没有一个大规模基准能同时评估 GUI 定位（Grounding）、屏幕解析（Screen Parsing）和动作预测（Action Prediction）。现有数据集各管一段——有的只做元素检测，有的只覆盖单一模态，有的只针对窄范围应用。

GUI-360° 正是为同时解决这三个问题而生。

---

## 二、核心架构深度剖析

### 2.1 三阶段数据收集管线

整个数据集的构建围绕**可扩展性**和**最小化人工干预**设计，分为三个阶段：

**阶段一：Query Acquisition（查询获取）**

四步流水线：
- **原型查询源采集**：从三个真实渠道获取——应用内帮助内容（In-App）、在线社区论坛（Online）、搜索引擎日志（Search）。共收集约 78,000 条原始查询（Word 27,903 / Excel 28,552 / PowerPoint 21,698）。
- **环境模板构建**：为每个查询设计最小但充分的软件环境模板，编码任务执行所需的上下文。
- **任务实例化**：将查询匹配到最佳环境模板，用 LLM 将通用查询改写为针对具体环境可执行的指令。
- **质量过滤**：用 LLM 过滤低质量或模糊查询，仅保留可执行、语义明确的条目。

**阶段二：Automatic Trajectory Collection（自动轨迹收集）**

设计专门的 CUA — **TrajAgent**，在 Windows Office 应用中自动执行任务并收集完整轨迹。TrajAgent 的一次执行同时产出：
- 全分辨率截图
- 无障碍元数据（a11y metadata，可用时）
- 自然语言目标
- 中间推理链（agent "thoughts"）
- 逐步动作轨迹（包括成功和失败的步骤）

**阶段三：Evaluation & Post-processing（评估与后处理）**

使用自动化评估器验证轨迹正确性，过滤噪声数据，增强整体数据质量。

### 2.2 三个评测任务

| 任务 | 输入 | 输出 | 评估维度 |
|------|------|------|----------|
| **GUI Grounding** | 步骤级计划 + 截图 | 屏幕坐标 / UI 元素 | 定位精度 |
| **Screen Parsing** | 截图 | 可交互元素集合及其属性 | 元素覆盖与分类 |
| **Action Prediction** | 当前状态 + 用户意图 | 下一步动作（click/type/select/API call） | 动作正确率 |

### 2.3 混合 GUI+API 动作空间

这是 GUI-360° 的一个独特设计。动作空间不仅包含 GUI 操作（点击、输入、选择），还包含高层 API 调用。这反映了现代 CUA 架构（如 UFO）的实际设计——当有可靠的 API 可用时，Agent 应优先使用 API 而非模拟 GUI 操作，这样既高效又可靠。

## 三、技术亮点

### 亮点一：LLM 增强的全自动采集管线

传统 GUI 数据集的瓶颈在于"数据收集本身需要大量人工"。GUI-360° 的管线将 LLM 作为"数据工程师"贯穿全流程：从查询源筛选、环境模板设计、任务实例化到质量过滤，LLM 承担了几乎所有原本需要人工完成的决策环节。最终产出 **120 万+ 执行动作步骤**，覆盖数千条轨迹，覆盖 Word/Excel/PowerPoint 三大办公应用。这种"用 AI 收集 AI 训练数据"的范式值得借鉴——关键在于**质量过滤层**的设计，确保 LLM 生成/筛选的数据确实可用。

### 亮点二：首次同时支持三种任务 + 失败案例

在 GUI-360° 之前，没有一个数据集同时支持 Grounding、Parsing 和 Action Prediction 三种任务。更重要的是，数据集**刻意保留了失败的轨迹**。这对 Agent 训练至关重要——Agent 不仅需要知道什么动作是对的，更需要从失败案例中学习"什么不能做"。这与 RLHF 中的负样本采样理念一致。

### 亮点三：混合动作空间设计

将 GUI 操作和 API 调用统一到同一动作空间，使基准测试能评估 Agent 在"什么时候用 GUI、什么时候用 API"上的策略选择能力。这直接对应了真实场景中 Agent 的核心决策：用可靠的 API 高效完成子任务，还是用 GUI 操作覆盖 API 不支持的长尾需求。

## 四、基准测试表现

论文对多种 SOTA 视觉-语言模型进行了基准测试，揭示了以下关键发现：

**Off-the-shelf 模型的表现**：
- 现有模型在桌面异构布局的 GUI Grounding 上表现显著不足
- 步骤级 Action Prediction 频繁失败，且错误会级联（cascade）
- 屏幕解析在高分辨率混合内容场景下准确率较低

**微调后的改善**：
- 在 GUI-360° 上进行监督微调（SFT）后，各任务指标有显著提升
- 强化学习（RL）进一步带来增益
- 但即使经过训练，与人类级可靠性仍有明显差距

这说明 GUI-360° 作为一个基准，其难度足够区分不同模型的能力层级，同时数据集本身也具有训练价值——微调即有效果提升。

## 五、横向对比

| 维度 | GUI-360° | ScreenSpot | DeskVision | UI-Vision |
|------|----------|------------|------------|-----------|
| 查询来源 | 真实多渠道 | 人工设计 | 在线 | 人工设计 |
| 任务覆盖 | Grounding + Parsing + Action Pred. | 仅 Grounding | 仅 Region Captioning | Grounding + Action |
| 样本规模 | 1,225,177 | 1,200+ | 54,855 | 8,227 |
| 数据收集 | 自动化 | 人工 | 自动化 | 人工 |
| 动作空间 | GUI + API | 无 | 无 | 仅 GUI |
| 推理链 | ✓ | ✗ | ✗ | ✗ |
| 无障碍信息 | ✓ | ✗ | ✗ | ✗ |
| 失败案例 | ✓ | ✗ | ✗ | ✗ |

**优势**：
- 规模领先：120 万步 vs 竞品的千/万级别
- 任务覆盖最全：唯一同时支持三种评测任务的数据集
- 真实性最强：查询来自真实用户行为而非人工构造
- 包含推理链和失败案例，训练价值更高

**劣势**：
- 目前仅覆盖 Windows Office 三件套，应用覆盖面较窄
- 虽然管线可扩展，但向其他桌面应用迁移需要重新构建环境模板
- 主要面向 Windows 桌面，缺乏跨平台（macOS/Linux）覆盖

## 六、对我们的实际启发

### 1. "AI 收集 AI 数据"的范式成熟了

GUI-360° 证明了 LLM 可以作为高质量数据收集管线中的核心组件。对于我们的 Agent 开发，当需要训练数据时，不必执着于人工标注——用 LLM 构建自动化管线 + 严格的质量过滤层，可以高效产出可用数据。

### 2. 失败案例是训练的隐藏金矿

刻意保留并利用失败轨迹进行训练，这一设计思路可以迁移到任何 Agent 开发场景。在 OpenClaw 中记录 Agent 执行失败的案例（工具调用失败、规划错误等），定期用于 SFT 或 Few-shot 示例，能显著提升 Agent 的鲁棒性。

### 3. 混合动作空间是 Agent 架构的正确方向

GUI-360° 的 GUI+API 混合动作空间验证了 OpenClaw 等现代 Agent 框架的设计理念——当有可靠 API 时用 API，没有时退回到 GUI/CLI 操作。这种分层策略在工程实践中已被证明是最优的。

### 4. 基准测试的"三角验证"思路

同时评估 Grounding（感知）、Parsing（理解）和 Action Prediction（决策）三个维度，比单一指标更能反映 Agent 的真实能力。在评估我们自己的 Agent 时，也应从感知、理解、执行三个角度分别打分，避免被单一指标误导。

---

## 总结

GUI-360° 是桌面 CUA 领域迄今最全面的数据集和基准测试。它通过 LLM 增强的自动化管线解决了数据规模和质量问题，首次统一了三种核心评测任务，并引入了混合动作空间和失败案例等创新设计。虽然应用覆盖面目前限于 Office 三件套，但其管线设计和评测框架具有通用性，对整个 GUI Agent 社区具有重要参考价值。对于 Agent 开发者而言，GUI-360° 提供了从数据收集、模型训练到效果评估的完整参考范本。
